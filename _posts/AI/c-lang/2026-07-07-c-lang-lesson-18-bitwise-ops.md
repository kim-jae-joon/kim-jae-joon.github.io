# 18강: C 언어 활용: 비트 연산과 마스킹

## 1. 비트 연산, 대체 왜 하는 걸까요?

우리는 보통 숫자를 10진수(0~9)로 생각하죠. 하지만 컴퓨터는 바보라서 0과 1밖에 모릅니다. 우리가 `int a = 5;`라고 쓰면, 컴퓨터는 내부적으로 `00000101`이라는 2진수로 저장합니다.

비트 연산은 이렇게 쪼개진 0과 1을 하나하나 직접 건드리는 작업입니다. 비유를 들어볼까요?

> 여러분이 거대한 스위치 박스 앞에 서 있다고 생각해보세요. 스위치가 32개가 나란히 붙어 있습니다. 어떤 스위치는 켜져 있고(1), 어떤 스위치는 꺼져 있죠(0). 비트 연산은 이 스위치 하나하나를 직접 껐다 켰다 하는 정밀 작업입니다. 

왜 이렇게까지 하냐고요? **속도가 압도적으로 빠르고 메모리를 극도로 아낄 수 있기 때문**입니다. 예를 들어, '전등 켜짐', '에어컨 켜짐', 'TV 켜짐'이라는 상태를 저장할 때, `bool` 변수 3개를 쓰면 메모리가 낭비되지만, 비트 3개만 쓰면 `int` 하나로 수십 가지 상태를 한 번에 관리할 수 있거든요.

## 2. 비트 연산자의 종류와 정체

비트 연산자는 크게 5가지가 있습니다. 이 녀석들의 성격을 파악하는 게 제일 중요합니다.

### (1) AND 연산자 (&) : "둘 다 1이어야만 1이다!"
아주 까다로운 상사 같은 녀석입니다. 양쪽 비트가 모두 1일 때만 결과가 1이 됩니다. 하나라도 0이면 가차 없이 0으로 만들어버립니다.
- `1 & 1 = 1`
- `1 & 0 = 0`
- `0 & 0 = 0`

### (2) OR 연산자 (|) : "하나만 1이어도 좋아!"
성격 좋은 친구 같은 녀석입니다. 둘 중 하나만 1이면 결과는 1이 됩니다.
- `1 | 0 = 1`
- `0 | 0 = 0`
- `1 | 1 = 1`

### (3) XOR 연산자 (^) : "우린 달라야 해!"
청개구리 같은 녀석입니다. 두 비트가 서로 다를 때만 1을 뱉어냅니다. 같으면 0이 됩니다.
- `1 ^ 0 = 1`
- `0 ^ 1 = 1`
- `1 ^ 1 = 0`
- `0 ^ 0 = 0`

### (4) NOT 연산자 (~) : "반대로 해!"
청개구리의 끝판왕입니다. 0은 1로, 1은 0으로 싹 다 뒤집어버립니다. (단, 부호 비트 때문에 결과값이 생각보다 이상하게 나올 수 있는데, 이건 나중에 같이 살펴봅시다.)

### (5) Shift 연산자 (<<, >>) : "옆으로 밀어!"
비트들을 왼쪽이나 오른쪽으로 통째로 밀어버리는 연산입니다. 
- `<<` (Left Shift): 비트를 왼쪽으로 밉니다. 빈자리는 0으로 채워집니다. (수학적으로는 2를 곱하는 효과가 있습니다.)
- `>>` (Right Shift): 비트를 오른쪽으로 밉니다. (수학적으로는 2로 나누는 효과가 있습니다.)

## 3. [코드 실습 1] 비트 연산자 종합 선물 세트

자, 이제 이론은 됐고 실제로 코드를 보면서 뜯어봅시다. 어떻게 돌아가는지 한 줄 한 줄 설명해 드릴게요.

```c
#include <stdio.h>

int main() {
    // 2진수로 0000 1010 (10진수 10)
    unsigned char a = 10; 
    // 2진수로 0000 0101 (10진수 5)
    unsigned char b = 5;  

    // 1. AND 연산: 둘 다 1인 것만 추출
    // 1010 & 0101 = 0000 (결과: 0)
    unsigned char res_and = a & b;
    printf("AND 결과: %d\n", res_and);

    // 2. OR 연산: 하나라도 1이면 1
    // 1010 | 0101 = 1111 (결과: 15)
    unsigned char res_or = a | b;
    printf("OR 결과: %d\n", res_or);

    // 3. XOR 연산: 서로 다른 것만 1
    // 1010 ^ 0101 = 1111 (결과: 15)
    unsigned char res_xor = a ^ b;
    printf("XOR 결과: %d\n", res_xor);

    // 4. NOT 연산: 0은 1로, 1은 0으로
    // ~1010 = 1111 0101 (unsigned char 기준)
    unsigned char res_not = ~a;
    printf("NOT 결과: %d\n", res_not);

    return 0;
}
```

**코드 뜯어보기:**
- `unsigned char`를 쓴 이유는 1바이트(8비트)만 사용해서 비트 변화를 쉽게 관찰하기 위해서입니다. `int`를 쓰면 32비트라 0이 너무 많아 헷갈리거든요.
- `a & b`: 10(1010)과 5(0101)를 비교했는데 겹치는 1이 하나도 없죠? 그래서 결과는 0입니다.
- `a | b`: 10(1010)과 5(0101)를 합치면 1111이 됩니다. 2진수 1111은 10진수로 15입니다.
- `a ^ b`: 이번 예제에서는 두 숫자의 비트가 완전히 달라서 결과가 OR와 같게 나왔지만, 겹치는 부분이 있다면 0이 되었을 겁니다.
- `~a`: 1010의 모든 비트를 반전시켜 1111 0101이 되었습니다.

## 4. [코드 실습 2] Shift 연산의 마법

시프트 연산은 특히 숫자를 빠르게 곱하거나 나눌 때, 혹은 특정 위치의 비트를 확인하고 싶을 때 씁니다.

```c
#include <stdio.h>

int main() {
    unsigned char val = 3; // 2진수로 0000 0011

    // 1. 왼쪽 시프트 (Left Shift)
    // 0000 0011 << 2 => 0000 1100 (3 * 2^2 = 12)
    unsigned char left = val << 2;
    printf("3을 왼쪽으로 2칸 밀면: %d\n", left);

    // 2. 오른쪽 시프트 (Right Shift)
    // 0000 1100 >> 1 => 0000 0110 (12 / 2^1 = 6)
    unsigned char right = left >> 1;
    printf("12를 오른쪽으로 1칸 밀면: %d\n", right);

    // 3. 비트 위치 이동을 통한 값 생성
    // 1을 3번 밀면 0000 1000 이 됨 (8의 자리수 생성)
    unsigned char pos = 1 << 3;
    printf("1을 왼쪽으로 3칸 밀면: %d\n", pos);

    return 0;
}
```

**코드 뜯어보기:**
- `val << 2`: 비트를 왼쪽으로 두 칸 밀었습니다. 0011이 1100이 되었죠. 결과는 12입니다. 정말 신기하죠? 2의 거듭제곱을 곱하는 것과 같습니다.
- `left >> 1`: 반대로 오른쪽으로 밀면 2로 나누는 효과가 납니다. 12가 6이 되었습니다.
- `1 << 3`: 이게 실무에서 정말 많이 쓰입니다! "4번째 비트만 1로 만들고 싶어!" 할 때 이렇게 씁니다.

## 5. 비트 마스킹(Bit Masking) : 원하는 부분만 도려내기

자, 이제 오늘의 하이라이트 **'마스킹'**입니다. 마스킹이라는 말, 들어보셨나요? 포토샵에서 특정 영역만 색칠하거나 가리는 '마스크' 기능과 똑같습니다.

컴퓨터에서 마스킹이란, **특정 비트만 1로 만들거나 0으로 만들어서 내가 원하는 정보만 쏙 뽑아내는 기술**을 말합니다.

여기서 핵심은 **마스크(Mask)**라는 특수한 변수를 만드는 것입니다. 예를 들어, 3번째 비트만 확인하고 싶다면 `0000 0100`이라는 마스크를 만드는 거죠.

### 마스킹의 3대 핵심 기법

실무에서는 딱 이 3가지만 알면 끝납니다.

1. **비트 확인 (Check)**: 특정 비트가 0인지 1인지 알아내기 $\rightarrow$ `AND(&)` 사용
2. **비트 설정 (Set)**: 특정 비트를 강제로 1로 만들기 $\rightarrow$ `OR(|)` 사용
3. **비트 해제 (Clear)**: 특정 비트를 강제로 0으로 만들기 $\rightarrow$ `AND(&)`와 `NOT(~)` 사용

---

## 6. [코드 실습 3] 마스킹 실전 적용 (Check, Set, Clear)

이 코드가 오늘 강의의 정점입니다. 아주 자세히 설명할 테니 집중하세요!

```c
#include <stdio.h>

int main() {
    // 현재 시스템 상태 (예: 0000 1010)
    // 2번 비트(전등), 3번 비트(에어컨), 1번 비트(TV)가 켜져 있다고 가정
    unsigned char status = 0x0A; // 10진수 10, 2진수 0000 1010
    
    // 마스크 정의: 3번째 비트(값: 4)를 타겟으로 설정
    unsigned char mask = (1 << 2); // 0000 0100

    printf("초기 상태: 0x%02X\n", status);

    // [1] 비트 확인 (Check)
    // status & mask 결과가 0이 아니면 그 비트는 1인 것!
    if (status & mask) {
        printf("결과: 3번째 비트가 켜져 있습니다!\n");
    } else {
        printf("결과: 3번째 비트가 꺼져 있습니다!\n");
    }

    // [2] 비트 설정 (Set)
    // 4번째 비트(값: 8)를 강제로 켜보자! (마스크: 1 << 3)
    unsigned char set_mask = (1 << 3); // 0000 1000
    status = status | set_mask; 
    printf("4번째 비트 설정 후: 0x%02X\n", status);

    // [3] 비트 해제 (Clear)
    // 다시 2번째 비트(값: 2)를 강제로 꺼보자! (마스크: 1 << 1)
    unsigned char clear_mask = (1 << 1); // 0000 0010
    // ~clear_mask는 1111 1101 이 됨. 이것과 AND를 하면 2번째만 0이 됨.
    status = status & (~clear_mask);
    printf("2번째 비트 해제 후: 0x%02X\n", status);

    return 0;
}
```

**코드 뜯어보기 (매우 중요!):**
- **Check**: `status & mask`를 하면, 마스크가 0인 자리는 무조건 0이 되고, 마스크가 1인 자리만 `status`의 원래 값이 남습니다. 그래서 결과가 0이 아니면 "아, 여기 1이 있었구나!"라고 판단하는 겁니다.
- **Set**: `status | set_mask`를 하면, 마스크의 1인 부분이 `status`에 강제로 합쳐집니다. 다른 비트들은 OR 연산의 성질 덕분에 원래 값을 유지하고, 타겟 비트만 1이 됩니다.
- **Clear**: 이게 제일 어렵습니다. `~clear_mask`를 하면 타겟 비트만 0이고 나머지는 전부 1인 상태가 됩니다. 이걸 `AND` 연산하면, 타겟 비트는 0과 만나서 무조건 0이 되고, 나머지는 1과 만나서 원래 값을 유지합니다. 마치 정밀하게 그 부분만 도려내는 조각칼 같죠.

## 7. 특별 코너

### ❓ 초보자 폭풍 질문!
**Q: 재준봇님! `unsigned char` 같은 거 말고 그냥 `int` 쓰면 안 되나요? 왜 굳이 복잡하게 타입을 바꾸죠?**

**재준봇의 답변:**
아주 좋은 질문입니다! 사실 `int`를 써도 연산은 똑같이 됩니다. 하지만 비트 연산을 공부할 때는 '눈에 보이는 것'이 중요해요. `int`는 보통 32비트입니다. 그러면 앞에 0이 20개 넘게 붙어 있겠죠? `unsigned char`는 딱 8비트라서 우리가 2진수로 계산할 때 훨씬 직관적입니다. 그리고 실무에서 하드웨어 제어를 할 때는 메모리 1바이트가 아쉬운 상황이 많아서 정확한 크기의 타입을 지정하는 습관을 들이는 게 훨씬 좋습니다!

### ⚠️ 실무 주의보
**"부호 있는 정수(signed int)로 비트 연산 하지 마세요!"**

비트 연산을 할 때 `int`나 `char`처럼 부호가 있는 타입을 쓰면, 가장 왼쪽의 비트(MSB)가 '부호 비트'로 작동합니다. 여기서 `NOT` 연산이나 `Shift` 연산을 하면, 컴퓨터가 부호를 유지하려고 예상치 못한 동작(Arithmetic Shift)을 수행할 수 있습니다. 결과값이 갑자기 `-1`이 되거나 이상한 숫자가 튀어나오는 끔찍한 경험을 하고 싶지 않다면, **비트 연산에는 반드시 `unsigned` 키워드를 붙인 타입을 사용하세요!**
