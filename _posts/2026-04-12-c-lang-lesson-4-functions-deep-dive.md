### 1. 함수란 무엇일까요? - '작은 기능 단위' 만들기

여러분, 우리가 주방에서 요리를 할 때를 생각해볼까요? 밥을 지을 때는 밥솥을 쓰고, 국을 끓일 때는 냄비를 사용하죠? 이처럼 각자의 역할에 특화된 도구들이 있습니다. 프로그램에서도 마찬가지입니다.

**함수(Function)**란, **특정 작업을 수행하는 코드들의 집합**을 의미합니다. 자주 사용되는 코드 블록을 하나의 '기능 단위'로 묶어 이름을 붙여두는 것이죠. 필요할 때마다 그 이름을 불러서 해당 기능을 실행시킬 수 있습니다.

예를 들어, "화면에 인사말을 출력하는" 기능이나, "두 숫자를 더하는" 기능, "가장 큰 값을 찾는" 기능 등, 각각의 기능을 함수로 만들 수 있습니다.

**간단한 예제:** "안녕하세요!"를 출력하는 함수

```c
#include <stdio.h> // 표준 입출력 함수를 사용하기 위해 포함

// say_hello 라는 이름의 함수를 '정의'합니다.
// 이 함수는 어떤 값도 반환하지 않고, 어떤 값도 입력받지 않습니다.
void say_hello() {
    printf("안녕하세요, C 언어 초보자 여러분!\n");
    printf("함수를 배우러 오신 것을 환영합니다!\n");
}

int main() { // 모든 C 프로그램의 시작점인 main 함수입니다.
    // 위에서 정의한 say_hello 함수를 '호출'합니다.
    say_hello();

    printf("메인 함수 실행 중...\n");

    // 다시 say_hello 함수를 호출할 수도 있습니다.
    say_hello();

    return 0; // 프로그램이 성공적으로 종료되었음을 운영체제에 알립니다.
}
```

**[컴파일 및 실행 결과]**
```
안녕하세요, C 언어 초보자 여러분!
함수를 배우러 오신 것을 환영합니다!
메인 함수 실행 중...
안녕하세요, C 언어 초보자 여러분!
함수를 배우러 오신 것을 환영합니다!
```

어때요? `say_hello()`라는 이름만 부르면, 그 안에 있는 두 줄의 `printf` 문이 한꺼번에 실행되는 것을 볼 수 있습니다. 이처럼 함수는 코드를 깔끔하게 정리하고 재사용할 수 있게 해줍니다!

### 2. 왜 함수를 사용해야 할까요? - 코딩의 효율성 극대화!

함수를 사용하는 이유는 단순히 코드를 묶어두는 것 이상의 강력한 장점들을 제공하기 때문입니다. 마치 잘 정리된 도구 상자처럼 말이죠!

| 장점        | 설명                                                                                                                                                                          |
| :---------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **재사용성**  | 한 번 정의한 함수는 프로그램의 여러 곳에서 필요할 때마다 다시 호출하여 사용할 수 있습니다. 똑같은 코드를 여러 번 작성할 필요가 없어집니다.                                       |
| **가독성**    | 복잡한 로직을 작은 기능 단위의 함수로 나누면, `main` 함수나 다른 함수들이 훨씬 깔끔해지고, 전체 프로그램의 흐름을 이해하기 쉬워집니다. (마치 소제목처럼요!)                          |
| **유지보수**  | 특정 기능에 문제가 발생하거나 기능을 개선해야 할 때, 해당 기능이 구현된 함수만 수정하면 됩니다. 전체 코드를 뒤질 필요가 없어 에러 수정 및 기능 업데이트가 용이합니다.             |
| **모듈화**    | 프로그램을 논리적인 단위(모듈)로 분리하여 개발할 수 있습니다. 각 팀원이 각자의 함수를 개발하고 나중에 합칠 수 있어, 대규모 프로젝트에 필수적입니다.                           |
| **디버깅 용이** | 버그가 발생했을 때, 문제가 발생한 함수 내에서만 원인을 찾아보면 되므로 디버깅 시간을 크게 단축할 수 있습니다.                                                                  |

이러러한 장점들 때문에 실제 복잡한 프로그램을 만들 때는 함수를 적극적으로 활용합니다.

### 3. 함수의 세 가지 요소: 선언, 정의, 호출

함수가 어떻게 동작하는지 이해하려면, 이 세 가지 핵심 요소를 정확히 알아야 합니다.

1.  **함수 선언 (Declaration) 또는 함수 원형 (Prototype):**
    *   컴파일러에게 "이러이러한 함수가 곧 나올 거야!"라고 미리 알려주는 역할을 합니다.
    *   함수의 이름, 입력받는 값(매개변수)의 종류와 개수, 그리고 반환하는 값의 종류를 명시합니다.
    *   `반환타입 함수이름(매개변수타입1 매개변수이름1, 매개변수타입2 매개변수이름2, ...);` 형태입니다. 마지막에 `;`가 붙는다는 것이 중요합니다!

2.  **함수 정의 (Definition):**
    *   실제로 함수가 어떤 작업을 수행할지 코드로 작성하는 부분입니다.
    *   함수 선언과 동일한 형태의 헤더와 함께 중괄호 `{}` 안에 실행될 코드 블록이 들어갑니다.
    *   `반환타입 함수이름(매개변수타입1 매개변수이름1, ...) { // 실행될 코드; return 반환값; }`

3.  **함수 호출 (Call):**
    *   정의된 함수를 실제로 실행시키는 행위입니다.
    *   함수의 이름과 함께 필요한 값을 괄호 `()` 안에 넣어 전달합니다.
    *   `함수이름(인자1, 인자2, ...);`

아직 '함수 선언(원형)'은 좀 낯설게 느껴질 수 있습니다. 일단은 선언과 정의를 합쳐서 생각하고, 뒤에서 '함수 원형'의 필요성을 다시 설명해 드릴게요!

### 4. 함수에게 '입력' 주기: 매개변수 (Parameter)

때로는 함수가 어떤 작업을 수행하기 위해 외부로부터 정보(데이터)를 받아야 할 때가 있습니다. 이때 사용하는 것이 바로 **매개변수(Parameter)**입니다.

매개변수는 함수 정의 시 괄호 `()` 안에 선언하며, 함수 내부에서 일반 변수처럼 사용됩니다. 함수를 호출할 때 전달하는 값을 **인자(Argument)**라고 부르며, 이 인자들이 매개변수로 전달됩니다.

**예제: 두 수를 더하는 함수**

```c
#include <stdio.h>

// add_numbers 함수 정의:
// int 타입의 num1과 num2 두 개의 매개변수를 받아서,
// int 타입의 결과(두 수의 합)를 반환합니다.
int add_numbers(int num1, int num2) {
    int sum = num1 + num2; // 매개변수 num1과 num2를 사용하여 연산
    return sum;            // 계산 결과를 반환
}

int main() {
    int a = 10;
    int b = 20;

    // add_numbers 함수를 호출하며, a와 b의 값을 '인자'로 전달합니다.
    int result1 = add_numbers(a, b);
    printf("%d + %d = %d\n", a, b, result1); // 출력: 10 + 20 = 30

    // 다른 값으로도 다시 호출할 수 있습니다.
    int result2 = add_numbers(5, 3); // 5와 3을 직접 인자로 전달
    printf("5 + 3 = %d\n", result2);   // 출력: 5 + 3 = 8

    return 0;
}
```

**[컴파일 및 실행 결과]**
```
10 + 20 = 30
5 + 3 = 8
```

여기서 `num1`, `num2`가 **매개변수**이고, `add_numbers(a, b)`에서 `a`와 `b` (또는 `add_numbers(5, 3)`에서 `5`와 `3`)가 **인자**입니다. 함수가 호출될 때 `a`의 값은 `num1`으로, `b`의 값은 `num2`로 복사되어 전달됩니다.

### 5. 함수에게 '결과' 받기: 반환 값 (Return Value)

함수가 어떤 작업을 수행한 후에 그 결과를 다시 함수를 호출한 쪽으로 돌려줄 필요가 있을 때가 있습니다. 이때 사용하는 것이 **반환 값(Return Value)**입니다.

함수는 작업을 마치고 `return` 키워드를 사용하여 하나의 값을 반환할 수 있습니다. 함수의 정의 앞부분에 `int`, `double`, `char` 등 어떤 타입의 값을 반환할지 명시합니다.

**예제: 두 수 중 큰 값을 반환하는 함수**

```c
#include <stdio.h>

// get_max 함수 정의:
// int 타입의 두 매개변수 (a, b)를 받아,
// int 타입의 값 (두 수 중 큰 값)을 반환합니다.
int get_max(int a, int b) {
    if (a > b) {
        return a; // a가 더 크면 a를 반환
    } else {
        return b; // b가 더 크거나 같으면 b를 반환
    }
}

int main() {
    int num1 = 100;
    int num2 = 50;

    // get_max 함수를 호출하고, 반환된 값을 max_val 변수에 저장합니다.
    int max_val = get_max(num1, num2);
    printf("%d와 %d 중 더 큰 값은: %d\n", num1, num2, max_val); // 출력: 100와 50 중 더 큰 값은: 100

    // 직접 숫자를 넣어 호출할 수도 있습니다.
    printf("123과 456 중 더 큰 값은: %d\n", get_max(123, 456)); // 출력: 123과 456 중 더 큰 값은: 456

    return 0;
}
```

**[컴파일 및 실행 결과]**
```
100와 50 중 더 큰 값은: 100
123과 456 중 더 큰 값은: 456
```

`return` 문은 함수를 종료시키고, `return` 뒤에 오는 값을 호출한 곳으로 돌려줍니다. **하나의 함수는 오직 하나의 값만 반환할 수 있습니다.** (나중에 구조체나 포인터를 배우면 여러 값을 묶어서 반환하는 것처럼 보이게 할 수도 있습니다.)

### 6. 아무것도 반환하지 않을 때: `void` 키워드

만약 함수가 어떤 작업을 수행하기만 하고, 특별히 호출한 쪽으로 돌려줄 결과 값이 없을 때는 어떻게 할까요? 이때는 **`void`** 키워드를 사용합니다.

`void`는 "없음" 또는 "비어있음"을 의미합니다.
함수 정의에서 반환 타입 자리에 `void`를 사용하면, 이 함수는 값을 반환하지 않는다는 뜻이 됩니다.
마찬가지로, 매개변수가 없는 함수라면 괄호 `()` 안에 `void`를 적어줄 수도 있습니다 (생략해도 무방합니다).

**예제: 메시지를 출력하고, 아무것도 반환하지 않는 함수**

```c
#include <stdio.h>

// print_message 함수 정의:
// const char* 타입의 message 매개변수를 받지만,
// 어떤 값도 반환하지 않습니다 (void).
void print_message(const char* message) {
    printf("[메시지]: %s\n", message);
}

// 매개변수도 없고, 반환 값도 없는 함수
void print_separator() {
    printf("--------------------\n");
}

int main() {
    print_message("프로그램 시작!"); // 문자열을 인자로 전달
    print_separator();              // 구분선 출력 함수 호출
    print_message("함수는 정말 편리해요!");
    print_separator();
    print_message("4강 끝!");

    // 아래처럼 반환 값이 없는 함수를 변수에 저장하려고 하면 에러가 발생합니다!
    // int result = print_separator(); // 컴파일 에러 발생!

    return 0;
}
```

**[컴파일 및 실행 결과]**
```
[메시지]: 프로그램 시작!
--------------------
[메시지]: 함수는 정말 편리해요!
--------------------
[메시지]: 4강 끝!
```

`void` 함수는 특정 동작(출력, 데이터 변경 등)을 수행하는 데 주로 사용됩니다.

### 7. 특별한 함수: `main()` 다시 보기

우리가 여태껏 작성했던 모든 C 프로그램에는 항상 `main()` 함수가 있었습니다. 이제 함수의 개념을 배웠으니 `main()` 함수의 의미를 더 깊이 이해할 수 있습니다.

*   `main()` 함수는 모든 C 프로그램의 **시작점(Entry Point)**입니다. 운영체제가 C 프로그램을 실행할 때 가장 먼저 호출하는 함수가 바로 `main()`입니다.
*   `main()` 함수는 일반적으로 `int` 타입을 반환합니다. 이 반환 값은 프로그램이 성공적으로 종료되었는지 (0), 아니면 어떤 오류와 함께 종료되었는지 (0이 아닌 값)를 운영체제에 알려주는 역할을 합니다. 우리가 `return 0;`를 쓰는 이유가 바로 이것입니다.
*   `main()` 함수도 다른 함수들처럼 매개변수를 가질 수 있습니다. `int main(int argc, char* argv[])` 형태로 사용되며, 프로그램 실행 시 명령줄 인자를 받을 때 사용합니다. 이 부분은 조금 더 고급 개념이므로 다음 기회에 자세히 다루겠습니다!

```c
#include <stdio.h>

// main 함수도 하나의 함수입니다!
// int 타입을 반환하며, 운영체제에 프로그램 종료 상태를 알립니다.
int main() {
    printf("여기가 프로그램의 시작!\n");

    // 다른 함수들을 호출하여 프로그램을 구성합니다.
    // ...

    return 0; // 0을 반환하여 프로그램이 성공적으로 종료되었음을 알립니다.
}
```

### 8. 모듈화의 첫걸음: 함수 원형 (Function Prototype)

자, 이제 **함수 원형(Function Prototype)**의 중요성에 대해 이야기할 시간입니다. 이것이 바로 '모듈화의 시작'이라고 부르는 핵심 개념입니다!

C 언어 컴파일러는 코드를 위에서 아래로 읽어나갑니다. 만약 `main()` 함수에서 `add_numbers()` 함수를 호출했는데, `add_numbers()` 함수가 `main()` 함수보다 **뒤에 정의되어 있다면** 어떻게 될까요?

```c
#include <stdio.h>

int main() {
    // 컴파일러가 여기서 add_numbers를 만나는데...
    // 아직 add_numbers가 무엇인지 모릅니다!
    int result = add_numbers(10, 20); // 에러 발생 가능성!
    printf("10 + 20 = %d\n", result);
    return 0;
}

// add_numbers 함수는 main 함수보다 뒤에 정의되어 있습니다.
int add_numbers(int num1, int num2) {
    return num1 + num2;
}
```

이 코드는 대부분의 컴파일러에서 "암시적 선언(implicit declaration)" 경고를 띄우거나, 오래된 C 표준에서는 에러를 발생시킬 수 있습니다. 컴파일러가 `main` 함수를 처리할 때 `add_numbers`라는 함수를 처음 만나는데, 그 함수의 형태(어떤 값을 받고, 어떤 값을 반환하는지)를 아직 알지 못하기 때문입니다.

이런 문제를 해결하기 위해, 컴파일러에게 미리 "이러이러한 함수가 나중에 나올 거야!"라고 알려주는 것이 바로 **함수 원형(Function Prototype)**입니다.

**함수 원형의 역할:**

*   함수의 **이름**, **매개변수 타입과 개수**, **반환 타입**을 컴파일러에게 미리 알려줍니다.
*   이를 통해 컴파일러는 함수가 호출될 때 인자의 타입과 개수가 올바른지, 반환 값이 올바르게 사용되는지 등을 미리 검사할 수 있습니다.
*   코드의 가독성을 높이고, 여러 파일로 나누어 프로그램을 만들 때 (모듈화) 필수적입니다.

**함수 원형 사용 예제:**

```c
#include <stdio.h>

// ---------- 함수 원형 선언 (Prototype Declaration) ----------
// 컴파일러에게 "두 개의 int를 받고 int를 반환하는 add_numbers라는 함수가 있어!"라고 알려줍니다.
int add_numbers(int num1, int num2);
void print_message(const char* msg); // 문자열을 받고 아무것도 반환하지 않는 함수

// -----------------------------------------------------------

int main() {
    print_message("프로그램 시작!");

    int val1 = 100;
    int val2 = 200;
    int sum = add_numbers(val1, val2); // 이제 컴파일러는 add_numbers가 어떤 함수인지 알고 있습니다!

    printf("%d + %d = %d\n", val1, val2, sum);

    print_message("프로그램 종료!");

    return 0;
}

// ---------- 함수 정의 (Function Definition) ----------
// 함수 정의는 함수 원형 선언이나 main 함수 뒤에 올 수 있습니다.
int add_numbers(int num1, int num2) {
    return num1 + num2;
}

void print_message(const char* msg) {
    printf(">> %s <<\n", msg);
}
// ----------------------------------------------------
```

**[컴파일 및 실행 결과]**
```
>> 프로그램 시작! <<
100 + 200 = 300
>> 프로그램 종료! <<
```

이처럼 함수 원형을 사용하면 `main` 함수가 호출하는 함수들이 `main` 함수보다 뒤에 정의되어 있어도 문제가 없습니다. 이는 앞으로 여러분이 프로그램을 여러 개의 소스 파일(`*.c`)로 나누어 개발할 때 아주아주 중요한 개념이 됩니다. 각 파일에 정의된 함수들을 다른 파일에서 사용하려면 해당 함수의 원형을 미리 알려줘야 하기 때문이죠! 이것이 바로 **모듈화의 핵심적인 시작점**입니다.

### 9. 변수의 유효 범위: 지역 변수와 매개변수 (Scope)

함수 내에서 선언된 변수들은 특별한 성질을 가집니다. 바로 **'유효 범위(Scope)'**입니다.

*   **지역 변수 (Local Variable):**
    *   함수 내부에서 선언된 변수는 해당 함수 안에서만 유효합니다.
    *   함수가 호출될 때 생성되고, 함수가 종료될 때 소멸됩니다.
    *   다른 함수에서는 같은 이름의 변수를 선언해도 서로 다른 변수로 인식됩니다.

*   **매개변수 (Parameter):**
    *   매개변수 또한 함수가 호출될 때 생성되는 지역 변수의 일종입니다.
    *   호출 시 전달된 인자의 값으로 초기화되며, 해당 함수 안에서만 유효합니다.

**예제: 지역 변수와 매개변수의 유효 범위**

```c
#include <stdio.h>

void my_function(int param_a) { // param_a는 my_function의 지역 변수 (매개변수)
    int local_b = 20;            // local_b는 my_function의 지역 변수

    printf("[my_function 내부] param_a: %d, local_b: %d\n", param_a, local_b);

    // main_val은 my_function 외부에서 선언된 변수이므로, 여기서 직접 접근할 수 없습니다!
    // printf("main_val: %d\n", main_val); // 컴파일 에러 발생!
}

int main() {
    int main_val = 100; // main_val은 main 함수의 지역 변수
    int local_b = 50;   // my_function의 local_b와는 다른 변수입니다!

    printf("[main 함수 내부] main_val: %d, local_b: %d\n", main_val, local_b);

    my_function(main_val); // main_val의 값(100)을 param_a로 전달

    printf("[main 함수 내부 (함수 호출 후)] main_val: %d, local_b: %d\n", main_val, local_b);

    return 0;
}
```

**[컴파일 및 실행 결과]**
```
[main 함수 내부] main_val: 100, local_b: 50
[my_function 내부] param_a: 100, local_b: 20
[main 함수 내부 (함수 호출 후)] main_val: 100, local_b: 50
```

`main` 함수와 `my_function` 함수 안에 각각 `local_b`라는 변수가 있지만, 서로 독립적인 변수라는 것을 확인할 수 있습니다. `my_function`에서 `local_b`의 값을 변경해도 `main` 함수의 `local_b`에는 아무런 영향을 주지 않습니다.

이러한 유효 범위 개념 덕분에 함수들이 서로 영향을 주지 않고 독립적으로 동작할 수 있어, 프로그램을 더 견고하게 만들 수 있습니다.

### 10. 종합 예제: 사용자 입력을 받아 계산하는 함수들

이제 배운 함수 개념들을 모두 활용하여 작은 계산기 프로그램을 만들어 볼까요? 사용자로부터 두 개의 숫자를 입력받아 덧셈과 뺄셈을 수행하고 결과를 출력하는 프로그램입니다.

함수 원형, 매개변수, 반환 값, `void` 함수, 지역 변수 등 오늘 배운 모든 것을 총동원합니다!

```c
#include <stdio.h> // 표준 입출력 함수 사용

// ---------------------- 함수 원형 선언 ----------------------
// 프로그램 시작 메시지를 출력하는 함수
void print_intro();

// 사용자로부터 숫자를 하나 입력받아 반환하는 함수
int get_user_number();

// 두 수를 더하고 결과를 반환하는 함수
int add_numbers(int num1, int num2);

// 두 수를 빼고 결과를 반환하는 함수
int subtract_numbers(int num1, int num2);

// 계산 결과를 예쁘게 출력하는 함수
void print_calculation_result(const char* operation_name, int val1, int val2, int result);
// ------------------------------------------------------------

int main() {
    print_intro(); // 소개 메시지 출력

    // 사용자로부터 첫 번째 숫자 입력받기
    int number1 = get_user_number();

    // 사용자로부터 두 번째 숫자 입력받기
    int number2 = get_user_number();

    // 덧셈 수행 및 결과 출력
    int sum = add_numbers(number1, number2);
    print_calculation_result("덧셈", number1, number2, sum);

    // 뺄셈 수행 및 결과 출력
    int difference = subtract_numbers(number1, number2);
    print_calculation_result("뺄셈", number1, number2, difference);

    printf("\n계산기 프로그램을 종료합니다. 감사합니다!\n");

    return 0;
}

// ---------------------- 함수 정의 ----------------------

void print_intro() {
    printf("==================================\n");
    printf("  ✨ 아주 간단한 C 언어 계산기 ✨\n");
    printf("  두 숫자를 입력받아 덧셈과 뺄셈을 합니다.\n");
    printf("==================================\n\n");
}

int get_user_number() {
    int num;
    printf("숫자를 입력해 주세요: ");
    scanf("%d", &num); // 사용자로부터 정수를 입력받아 num에 저장
    return num;        // 입력받은 숫자를 반환
}

int add_numbers(int num1, int num2) {
    return num1 + num2; // 두 수의 합을 반환
}

int subtract_numbers(int num1, int num2) {
    return num1 - num2; // 두 수의 차를 반환
}

void print_calculation_result(const char* operation_name, int val1, int val2, int result) {
    printf("-> [결과] %s: %d와 %d의 연산 결과는 %d입니다.\n", operation_name, val1, val2, result);
}
// -------------------------------------------------------
```

**[컴파일 및 실행 결과]** (사용자가 10과 5를 입력했다고 가정)
```
==================================
  ✨ 아주 간단한 C 언어 계산기 ✨
  두 숫자를 입력받아 덧셈과 뺄셈을 합니다.
==================================

숫자를 입력해 주세요: 10
숫자를 입력해 주세요: 5
-> [결과] 덧셈: 10와 5의 연산 결과는 15입니다.
-> [결과] 뺄셈: 10와 5의 연산 결과는 5입니다.

계산기 프로그램을 종료합니다. 감사합니다!
```

코드가 `main` 함수 하나에 다 들어있을 때보다 훨씬 깔끔하고, 각 기능이 어떤 역할을 하는지 한눈에 알 수 있죠? 이것이 바로 함수가 주는 **모듈화의 힘**입니다!
