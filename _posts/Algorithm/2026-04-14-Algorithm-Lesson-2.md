# 2강: 보물찾기의 달인, **동적 프로그래밍 (Dynamic Programming)**

안녕하세요, 알고리즘의 신 주니어 개발자 여러분! 오늘은 **동적 프로그래밍**이라는 마법 같은 기술을 배워볼 거예요. 동적 프로그래밍은 복잡한 문제를 작은 부분 문제로 나누어 해결하고, 그 결과를 재사용해 효율성을 극대화하는 방법입니다. 이 기술이 없으면 보물찾기 게임에서 헤매기 딱 좋죠!

## 문제 상황: 보물 지도 탐험

**상황**: 섬에 흩어진 여러 보물상자들이 있고, 각 상자마다 얻을 수 있는 금화의 양이 정해져 있습니다. 하지만 일부 상자를 열면 그 다음 상자는 열 수 없는 제약이 있습니다 (예: 상자 2를 열면 상자 5는 열 수 없음). 당신의 임무는 가능한 한 많은 금화를 얻는 경로를 찾아내는 것입니다.

**입력**:
- `treasure_map`: 1차원 리스트로, 각 인덱스가 상자 번호를 나타내고 그 값은 해당 상자에서 얻을 수 있는 금화의 양을 의미합니다.
- `obstacles`: 상자 간의 연결 제약 조건을 나타내는 리스트. 예를 들어, `[(1, 3)]`은 상자 1을 열면 상자 3을 열 수 없다는 의미입니다.

**목표**: 최대 금화를 얻을 수 있는 경로를 찾아 그 금화의 총량을 반환하세요.

### 예제
```python
treasure_map = [0, 5, 10, 15, 20, 25]
obstacles = [(1, 3)]  # 상자 1을 열면 상자 3을 열 수 없음

# 결과: 최대 금화 총량 반환 (예시: 35)
```

---

### 💡 초보자 폭풍 질문!
- **Q**: 동적 프로그래밍이 왜 필요한 건가요?
  - **A**: 동적 프로그래밍은 반복되는 부분 문제를 저장해 재사용함으로써 시간을 크게 줄여줍니다. 이 문제에서는 각 상자를 열 때마다 최적의 선택을 기억해 나중에 사용하는 거죠!

### 코드 설계 및 설명

#### 1. 초기화 부분
```python
def max_treasure(treasure_map, obstacles):
    n = len(treasure_map)
    # 각 상자에서 얻을 수 있는 최대 금화를 저장할 리스트 초기화
    dp = [0] * n
    # 제약 조건을 딕셔너리로 저장
    constraint = {}
    
    # 제약 조건 설정
    for pre, next_box in obstacles:
        constraint[next_box - 1] = pre - 1  # 인덱스는 0부터 시작하므로 조정
```
- **설명**: `dp` 리스트는 각 인덱스에 해당하는 상자를 열었을 때 얻을 수 있는 최대 금화를 저장합니다. `constraint` 딕셔너리는 특정 상자를 열 수 없는 제약 조건을 저장합니다.

#### 2. 핵심 알고리즘 부분
```python
    # 첫 번째 상자는 제약 없이 열 수 있음
    dp[0] = treasure_map[0]
    
    for i in range(1, n):
        # 제약 조건 확인
        if i in constraint:
            prev_box = constraint[i]
            dp[i] = max(dp[i], dp[prev_box])  # 제약 조건에 따른 최대값 설정
        else:
            # 이전 상자들에서의 최대값과 현재 상자의 금화를 더함
            dp[i] = max(dp[i-1], (dp[i-1] if i-1 >= 0 else 0) + treasure_map[i])
    
    # 마지막 상자까지의 최대 금화 반환
    return max(dp)
```
- **설명**: 
  - 각 상자에 대해 이전 상자들의 최대 금화를 참조합니다.
  - 제약 조건이 있는 경우, 그 제약을 고려해 최적의 값을 결정합니다.
  - 만약 제약이 없다면, 이전 상자의 최대 금화와 현재 상자의 금화를 합한 값 중 더 큰 값을 선택합니다.

#### 3. 결과 출력 부분
```python
# 예제 실행
treasure_map = [0, 5, 10, 15, 20, 25]
obstacles = [(1, 3)]
print(max_treasure(treasure_map, obstacles))  # 출력 예상: 35
```
- **설명**: 최종적으로 `dp` 리스트에서 가장 큰 값을 반환하여 최대 금화를 얻을 수 있는 경로의 총량을 출력합니다.

### 한계점 및 실무 개선점
- **한계점**:
  - **메모리 사용량**: 큰 데이터셋에서는 `dp` 배열이 많은 메모리를 차지할 수 있습니다.
  - **복잡한 제약 조건**: 제약 조건이 매우 복잡하거나 동적으로 변할 경우, 알고리즘의 효율성이 떨어질 수 있습니다.
  
- **최적화 방향**:
  - **메모리 최적화**: 공간 복잡도를 줄이기 위해 이전 상태만 유지하는 방식 (예: 1차원 배열 사용)을 고려할 수 있습니다.
  - **동적 제약 처리**: 실시간으로 변경되는 제약 조건에 대응하기 위해, 동적으로 업데이트되는 데이터 구조 (예: 트리셋)를 활용할 수 있습니다.
  - **병렬 처리**: 대규모 데이터셋에서는 병렬 처리 기법을 적용하여 계산 시간을 단축할 수 있습니다.
