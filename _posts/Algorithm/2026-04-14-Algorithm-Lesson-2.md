# 2강: 시간여행자의 선택, **동적 계획법 (Dynamic Programming)**

안녕하세요, 여러분! 알고리즘의 신으로서 오늘은 좀 더 신나는 주제로 여러분을 초대합니다. 시간여행이 가능하다면, 같은 문제를 여러 번 풀어보면서 최적의 해답을 찾아가는 게 얼마나 유용할지 상상해 보셨나요? 바로 그 개념이 **동적 계획법 (Dynamic Programming, DP)**입니다! 오늘은 이 멋진 기술을 활용해 **"마법의 보물상자를 찾아서"**라는 재미있는 문제를 풀어보도록 하겠습니다.

## 문제: 마법의 보물상자를 찾아서

**상황:**
- 당신은 마법의 왕국에 있는 신비로운 동굴을 탐험하는 모험가입니다.
- 동굴에는 여러 방이 있고, 각 방에는 다양한 아이템이 있습니다.
- 보물상자는 특정 방에 있으며, 이를 찾기 위해 여러 경로를 거쳐야 합니다.
- 하지만 각 방을 지나갈 때마다 이동 비용이 발생합니다 (예: 체력 소모).
- 목표는 보물상자를 가장 적은 비용으로 찾는 경로를 찾는 것입니다.

**입력:**
- `n`: 동굴의 방 개수 (1 ≤ n ≤ 100)
- `cost`: 2차원 배열 (n x n), `cost[i][j]`는 방 `i`에서 방 `j`로 이동할 때 드는 비용입니다.
- `start`: 시작 방 번호 (1부터 시작)
- `treasure`: 보물상자가 있는 방 번호 (1부터 시작)

**출력:**
- 보물상자까지의 최소 이동 비용을 반환하세요.

**예제:**
```plaintext
n = 4
cost = [
    [0, 2, 5, 1],
    [2, 0, 3, 4],
    [5, 3, 0, 6],
    [1, 4, 6, 0]
]
start = 1
treasure = 4
```
**정답:** 7 (1 → 4 경로가 최소 비용)

---

**초보자 폭풍 질문!**  
> 동적 계획법이 왜 필요할까요? 그냥 모든 경로를 다 계산하면 안 되나요?

**답변:**  
맞아요, 모든 경로를 다 계산하면 될 것 같지만, 방의 수가 많아지면 계산량이 엄청나게 늘어납니다! 동적 계획법은 이미 계산한 결과를 메모이제이션(기억)하여 중복 계산을 피하고 효율성을 극대화합니다. 이는 시간과 자원을 아끼는 데 정말 중요해요!

### 🛠️ 해결 방안 가이드 (알고리즘 설계)
동적 계획법을 활용하여 이 문제를 해결하는 방법은 다음과 같습니다:
1. **메모이제이션 테이블 초기화**: 각 방까지 도달하는 최소 비용을 저장할 2차원 배열 `dp`를 만듭니다.
2. **기저 사례 설정**: 시작 방에서 시작 방으로 이동하는 비용은 0으로 설정합니다.
3. **반복적 업데이트**: 모든 방에 대해, 각 방에서 다른 방으로 이동할 때 최소 비용을 업데이트합니다.
4. **결과 추출**: `treasure` 방까지의 최소 비용을 반환합니다.

### 💻 파이썬 정답 코드 및 상세 해설
```python
def find_treasure_min_cost(n, cost, start, treasure):
    # 시작 방 인덱스는 0부터 시작하도록 조정
    start -= 1
    treasure -= 1
    
    # dp[i]는 i번 방까지 도달하는 최소 비용을 저장
    dp = [float('inf')] * n
    dp[start] = 0  # 시작 방의 비용은 0
    
    # 모든 방을 순회하며 최소 비용 업데이트
    for _ in range(n - 1):  # n-1 번 반복: 모든 노드를 최대한 여러 번 방문하여 최적화
        for i in range(n):
            for j in range(n):
                if dp[i] != float('inf') and cost[i][j] > 0:  # 유효한 이동 경로일 때
                    new_cost = dp[i] + cost[i][j]
                    if new_cost < dp[j]:
                        dp[j] = new_cost
    
    # 보물상자까지의 최소 비용 반환
    return dp[treasure] if dp[treasure] != float('inf') else -1  # 도달 불가능 시 -1 반환

# 예제 입력
n = 4
cost = [
    [0, 2, 5, 1],
    [2, 0, 3, 4],
    [5, 3, 0, 6],
    [1, 4, 6, 0]
]
start = 1
treasure = 4

# 함수 호출 및 결과 출력
print(find_treasure_min_cost(n, cost, start, treasure))  # 출력: 7
```

### 🚀 한계점 및 실무 개선점
- **시간 복잡도**: 기본 구현은 O(n^3)으로, 방의 수가 많아지면 느려질 수 있습니다. 이를 개선하기 위해 **최적 경로를 찾는 순서**를 최적화하거나 (예: 위상 정렬을 활용), **메모이제이션과 함께 그래프의 특성을 활용**하는 방법이 있습니다.
- **메모리 사용**: 2차원 배열 `dp`를 사용하므로 메모리 사용량이 상당할 수 있습니다. **1차원 배열로 메모이제이션**을 최적화하거나, **부분 문제를 더 효율적으로 관리**하는 방법을 고려할 수 있습니다.
- **동적 그래프**: 만약 동굴의 이동 비용이 시간에 따라 변하는 경우, **온라인 동적 계획법** 또는 **이벤트 기반 업데이트** 방식을 도입하여 실시간으로 최적 경로를 유지할 수 있습니다.
