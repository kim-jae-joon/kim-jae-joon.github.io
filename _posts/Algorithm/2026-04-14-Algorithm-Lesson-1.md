# 1강: 좀비 피하기의 영웅, BFS (너비 우선 탐색)

안녕하세요, 여러분! 오늘은 알고리즘의 세계에서 좀비 피하기 미션을 통해 **BFS (너비 우선 탐색)** 알고리즘을 배워볼 거예요. 이건 정말 실생활에서도 유용하게 쓰일 수 있는 꿀팁이니, 꼭 집중하세요!

### 🕵️‍♀️ 문제 상황: 좀비 피하기 미션

**스토리**: 
지구가 좀비 바이러스에 감염되었어요! 당신은 안전한 피난처로 가야 하는데, 도시는 미로처럼 복잡하게 연결된 길로 이루어져 있습니다. 각 위치는 '0' (안전한 곳) 또는 '1' (좀비가 있는 곳)로 표시되어 있어요. 당신의 목표는 시작점에서 피난처까지 가는 가장 안전하고 빠른 경로를 찾아야 합니다. 단, 좀비가 있는 곳은 절대 지나가지 않아야 합니다!

**구체적인 문제**:
- **입력**: 2차원 리스트 `grid` (예: `[[0, 1, 0], [0, 0, 1], [1, 0, 0]]`)
  - `0`: 안전한 곳
  - `1`: 좀비가 있는 곳
- **목표**: 시작점 (0, 0)에서 가장 가까운 피난처 (예: (n-1, m-1) 위치)까지의 안전 경로를 찾으세요.
- **제약 조건**:
  - 이동 가능한 방향: 상하좌우 (동서남북)
  - 좀비가 있는 곳은 건너뛰어야 함.
  - 경로가 없으면 `-1` 반환.

**초보자 폭풍 질문!**:
- **Q**: "왜 BFS를 써야 하나요?"
  - **A**: BFS는 레벨별로 탐색하기 때문에, 최단 경로를 보장해줍니다. 좀비 피하면서 가장 빨리 안전한 곳으로 가는 경로를 찾는 데 이상적이죠!

### 🛠️ 문제 이해하기

이 문제에서는 최단 경로를 찾는 것이 핵심이에요. BFS는 각 레벨(즉, 이동 횟수)마다 인접한 모든 노드를 탐색하므로, 가장 빠르게 목표 지점에 도달할 수 있는 경로를 보장합니다. 시간 복잡도는 대략 \(O(N \times M)\) (N과 M은 각각 행과 열의 수)로, 효율적이죠!

### 🛠️ 해결 방안 가이드 (알고리즘 설계)
1. **초기 설정**:
   - 시작점 (0, 0)을 큐에 넣고, 방문 기록을 위한 `visited` 배열을 준비합니다.
   - 경로 추적을 위한 `parent` 딕셔너리를 사용하여 각 위치의 이전 위치를 저장합니다.

2. **BFS 탐색**:
   - 큐에서 위치를 하나씩 꺼내며, 상하좌우로 이동 가능한 위치를 확인합니다.
   - 이동 가능한 위치가 안전한 곳(`0`)이고 방문하지 않았다면 큐에 추가하고 방문 표시합니다.
   - 목표 위치에 도달하면 경로를 추적합니다.

3. **경로 추적**:
   - 목표 위치에서 `parent` 딕셔너리를 따라 시작점으로 거슬러 올라가 경로를 구성합니다.

### 💻 파이썬 정답 코드 및 상세 해설

```python
from collections import deque

def find_safe_path(grid):
    if not grid or grid[0][0] == 1:
        return -1  # 시작 위치가 안전하지 않으면 바로 실패

    n, m = len(grid), len(grid[0])
    directions = [(0, 1), (1, 0), (0, -1), (-1, 0)]  # 동, 남, 서, 북
    queue = deque([(0, 0)])
    visited = [[False] * m for _ in range(n)]
    parent = {}  # 경로 추적용 딕셔너리

    visited[0][0] = True

    while queue:
        x, y = queue.popleft()
        
        # 목표 위치 도달 시 경로 추적
        if (x, y) == (n-1, m-1):
            path = []
            while (x, y) in parent:
                path.append((x, y))
                x, y = parent[(x, y)]
            path.append((0, 0))  # 시작점 추가
            return list(reversed(path))

        for dx, dy in directions:
            nx, ny = x + dx, y + dy
            if 0 <= nx < n and 0 <= ny < m and not visited[nx][ny] and grid[nx][ny] == 0:
                queue.append((nx, ny))
                visited[nx][ny] = True
                parent[(nx, ny)] = (x, y)  # 경로 추적을 위한 부모 노드 저장

    return -1  # 경로가 없는 경우

# 예시 사용
grid = [[0, 1, 0], [0, 0, 1], [1, 0, 0]]
path = find_safe_path(grid)
print("안전 경로:", path if path != -1 else "경로 없음")
```

### 🚀 한계점 및 실무 개선점
- **한계점**:
  - **메모리 사용**: 큰 그리드에서는 방문 배열과 큐가 많은 메모리를 사용할 수 있습니다.
  - **동적 변경**: 실시간으로 그리드가 변경되는 상황에서는 기존 경로 정보를 빠르게 업데이트하기 어려울 수 있습니다.

- **실무 개선점**:
  - **메모리 최적화**: 필요할 때만 방문 정보를 업데이트하거나, 메모리 효율적인 자료구조를 활용할 수 있습니다.
  - **동적 환경 처리**: 그리드 변경을 감지하고 부분적인 경로 재계산을 수행하는 메커니즘을 도입할 수 있습니다.
  - **병렬 처리**: 대규모 데이터셋에서는 병렬 BFS 알고리즘을 활용하여 처리 시간을 단축할 수 있습니다.
