# 3강: 문자열의 마법사, KMP 알고리즘

안녕하세요, 알고리즘의 신이자 유머러스한 주니어 개발자 여러분! 이번 강의는 **KMP 알고리즘**에 대해 배워볼 거예요. KMP 알고리즘은 문자열 검색 알고리즘으로, 특히 패턴 매칭에서 빠르고 효율적인 해결책을 제공합니다. 이 알고리즘을 알면, 가령 "텍스트 편집기에서 특정 단어나 구문을 빠르게 찾는" 상황에서 정말 큰 도움이 될 거예요.

**상황 설정**: 당신은 비밀 요원이 되었어요! 미션은 기밀 문서에서 숨겨진 암호 패턴을 찾는 거예요. 이 문서에는 불규칙적으로 섞인 여러 개의 'SECRET'이라는 단어가 숨겨져 있습니다. 하지만, 단순 검색으로는 너무 오래 걸려요. KMP 알고리즘을 활용해 빠르게 암호 패턴을 찾아내는 코드를 작성해 보세요!

**문제**: 주어진 텍스트에서 특정 패턴('SECRET')을 효율적으로 찾아, 모든 출현 위치를 리스트로 반환하는 함수를 작성하세요.

### 📚 개념 설명: KMP 알고리즘이 왜 필요한가요?

KMP 알고리즘은 **'부분 일치 테이블'**을 활용하여 패턴 내에서 일치하지 않는 부분을 건너뛰는 방법으로 동작합니다. 일반적인 문자열 검색 알고리즘과 달리, 일치하지 않는 부분을 다시 처음부터 검색하지 않고 효율적으로 이동할 수 있어요. 이렇게 하면 시간 복잡도가 O(n + m)이 되는데, 여기서 `n`은 텍스트의 길이, `m`은 패턴의 길이입니다. 즉, 텍스트가 길어져도 빠르게 검색할 수 있다는 거죠!

**초보자 폭풍 질문!**  
Q: **KMP 알고리즘이 일반적인 문자열 검색과 어떻게 다르죠?**  
A: 일반적인 문자열 검색은 패턴이 일치하지 않으면 텍스트의 처음부터 다시 검사하는 반면, KMP는 이미 검사한 정보를 활용해 부분적으로 건너뛰기 때문에 훨씬 효율적입니다.

### 🛠️ 해결 방안 가이드 (알고리즘 설계)
1. **부분 일치 테이블 생성**: 패턴에 대한 부분 일치 테이블을 먼저 만듭니다. 이 테이블은 패턴 내에서 접두사와 접미사가 일치하는 최대 길이를 저장합니다.
2. **텍스트와 패턴 비교**: 텍스트를 순회하면서 패턴과 비교합니다. 일치하지 않는 경우, 부분 일치 테이블을 참조해 패턴을 얼마나 이동할지 결정합니다.
3. **일치 위치 기록**: 패턴이 완전히 일치하면 해당 위치를 결과 리스트에 추가합니다.

### 💻 파이썬 정답 코드 및 상세 해설
```python
def compute_lps(pattern):
    """부분 일치 테이블 생성 함수"""
    lps = [0] * len(pattern)
    length = 0  # 이전 최대 접두사-접미사 길이
    i = 1
    while i < len(pattern):
        if pattern[i] == pattern[length]:
            length += 1
            lps[i] = length
            i += 1
        else:
            if length != 0:
                length = lps[length - 1]
            else:
                lps[i] = 0
                i += 1
    return lps

def kmp_search(text, pattern):
    """KMP 알고리즘을 이용한 문자열 검색 함수"""
    lps = compute_lps(pattern)
    i = j = 0  # text와 pattern의 인덱스
    result = []
    
    while i < len(text):
        if pattern[j] == text[i]:
            i += 1
            j += 1
        if j == len(pattern):
            result.append(i - j)  # 패턴 일치 위치 저장
            j = lps[j - 1]
        elif i < len(text) and pattern[j] != text[i]:
            if j != 0:
                j = lps[j - 1]
            else:
                i += 1
    return result

# 예제 사용
text = "HISECRETISVERYSECRETBUTALSOMISSINGSECT"
pattern = "SECRET"
print(kmp_search(text, pattern))  # [4, 21] 출력 예상
```

### 🚀 한계점 및 실무 개선점
- **대규모 데이터 처리**: 텍스트와 패턴이 매우 길 경우, 메모리 사용량이 증가할 수 있습니다. 메모리 제한이 있는 환경에서는 부분적으로 처리하거나 스트리밍 방식을 고려할 수 있습니다.
- **다중 패턴 검색**: 여러 패턴을 동시에 검색해야 하는 경우, 각 패턴에 대한 LPS 테이블을 미리 계산하고 효율적으로 관리하는 구조를 설계해야 합니다.
- **병렬 처리**: 텍스트가 엄청나게 길다면, 병렬 처리 기법을 활용하여 여러 부분을 동시에 검색하는 방식으로 성능을 개선할 수 있습니다.
