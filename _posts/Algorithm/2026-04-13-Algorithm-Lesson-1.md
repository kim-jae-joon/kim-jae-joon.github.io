# 1강: 음식 배달의 왕, 다익스트라 알고리즘

안녕하세요, 알고리즘의 신이자 5년 차 주니어 개발자인 저와 함께하는 코딩 여정에 오신 것을 환영합니다! 오늘은 여러분이 음식 배달 앱 개발자로 변신해보는 시간을 가져볼게요. **"최단 경로로 음식 배달하기"**라는 재미있는 문제를 풀어보면서 **다익스트라 알고리즘**의 마법을 배워볼 거예요. 이거 모르면 큰일 납니다! 진짜 신기하죠?

## 문제 상황

**상황**: 당신은 인기 있는 음식 배달 앱의 개발자입니다. 도시의 격자형 지도에서 사용자의 위치에서 식당까지 최단 경로를 찾아 배달해야 합니다. 하지만 도시는 신호등, 교통 체증 등으로 인해 모든 경로의 비용(이동 시간)이 다릅니다. 어떻게 하면 가장 빨리 음식을 배달할 수 있을까요?

**입력**:
- 사용자 위치 (start)와 식당 위치 (end)
- 격자형 지도에서 각 노드(위치) 간의 이동 비용(시간)이 담긴 그래프 (예: `graph = { (x1, y1): { (x2, y2): cost, ... }, ... }`)

**목표**: 사용자 위치에서 식당까지의 최단 경로 시간을 구하고, 그 경로를 출력하세요.

---

## 개념 설명: 다익스트라 알고리즘이란?

다익스트라 알고리즘은 그래프에서 시작 노드에서 다른 모든 노드까지의 최단 경로를 찾는 데 사용되는 알고리즘입니다. **핵심 아이디어**는 다음과 같습니다:

1. **초기 설정**: 시작 노드의 거리를 0으로 설정하고, 나머지 노드의 거리를 무한대로 설정합니다.
2. **우선순위 큐 활용**: 현재까지 가장 짧은 경로를 가진 노드를 선택합니다.
3. **최적 경로 업데이트**: 선택된 노드를 통해 인접한 노드들의 거리를 업데이트합니다.
4. **반복**: 모든 노드를 방문할 때까지 위 과정을 반복합니다.

**시간 복잡도**: 일반적으로 우선순위 큐를 사용할 경우 \(O((V + E) \log V)\), 여기서 \(V\)는 노드의 수, \(E\)는 엣지(간선)의 수입니다.

---

## 문제 해결 과정

### 초기화 부분

우선, 필요한 라이브러리와 초기 설정을 해봅시다. 파이썬으로 구현해보겠습니다.

```python
import heapq

def initialize(start, graph):
    # 거리 테이블 초기화: 모든 노드의 거리를 무한대로 설정
    distances = {node: float('inf') for node in graph}
    # 시작 노드의 거리는 0으로 설정
    distances[start] = 0
    # 우선순위 큐 (최소 힙) 초기화: (거리, 노드) 형태로 저장
    priority_queue = [(0, start)]
    
    return distances, priority_queue
```

**설명**:
- `distances` 딕셔너리는 각 노드까지의 최단 거리를 저장합니다.
- 시작 노드의 거리는 0으로 설정하고, 나머지는 무한대로 초기화합니다.
- `priority_queue`는 최소 힙을 이용해 가장 작은 거리를 가진 노드를 빠르게 추출할 수 있게 합니다.

### 핵심 알고리즘 부분

다음은 실제 다익스트라 알고리즘의 핵심 로직입니다.

```python
def dijkstra(start, graph):
    distances, queue = initialize(start, graph)
    
    while queue:
        # 현재 노드와 그 거리 추출
        current_distance, current_node = heapq.heappop(queue)
        
        # 이미 처리된 노드는 건너뛰기
        if current_distance > distances[current_node]:
            continue
        
        # 인접 노드들에 대해 거리 업데이트
        for neighbor, weight in graph[current_node].items():
            distance = current_distance + weight
            
            # 더 짧은 경로를 찾은 경우 업데이트
            if distance < distances[neighbor]:
                distances[neighbor] = distance
                heapq.heappush(queue, (distance, neighbor))
                
    return distances
```

**설명**:
- `heapq.heappop`을 사용해 현재 가장 짧은 경로를 가진 노드를 선택합니다.
- 현재 노드의 인접 노드들을 순회하며, 더 짧은 경로가 발견되면 거리 테이블을 업데이트하고 우선순위 큐에 추가합니다.

### 결과 출력 부분

마지막으로, 결과를 깔끔하게 출력해봅시다.

```python
def print_shortest_path(distances, start, end):
    if distances[end] == float('inf'):
        print("도착지에 도달할 수 없습니다.")
    else:
        print(f"최단 경로 시간: {distances[end]}")
        # 경로 추적을 위한 추가 로직은 별도로 구현 가능

# 예제 그래프
graph = {
    (0, 0): {(1, 0): 2, (0, 1): 1},
    (1, 0): {(0, 0): 2, (1, 1): 3},
    (0, 1): {(0, 0): 1, (1, 1): 2},
    (1, 1): {(1, 0): 3, (0, 1): 2}
}

start = (0, 0)
end = (1, 1)

# 실행
distances = dijkstra(start, graph)
print_shortest_path(distances, start, end)
```

**설명**:
- `distances` 딕셔너리에서 도착 노드의 최단 거리를 출력합니다.
- 도달 불가능한 경우에 대한 처리도 포함되어 있습니다.

### 해결 방안 요약
1. **초기화**: 시작 노드의 거리를 0으로, 나머지는 무한대로 설정하고 우선순위 큐를 준비합니다.
2. **알고리즘 수행**:
   - 우선순위 큐에서 가장 짧은 경로를 가진 노드를 꺼냅니다.
   - 해당 노드의 인접 노드들을 순회하며 더 짧은 경로가 있으면 업데이트합니다.
3. **결과 출력**: 도착 노드까지의 최단 거리를 출력합니다.

### 한계점 및 실무 개선점
- **대규모 데이터**: 대규모 그래프에서는 메모리 사용량과 처리 시간이 증가할 수 있습니다. 이를 위해 **분할 정복** 기법이나 **병렬 처리**를 고려할 수 있습니다.
- **동적 트래픽 데이터**: 실시간 교통 데이터를 반영하려면 알고리즘을 동적으로 업데이트해야 합니다. **온라인 알고리즘** 또는 **주기적 업데이트** 방식을 도입할 수 있습니다.
- **메모리 제한**: 메모리 제한이 있는 환경에서는 **스파스 행렬** 표현이나 **외부 메모리 접근** 기법을 활용해 메모리 사용을 최적화할 수 있습니다.
