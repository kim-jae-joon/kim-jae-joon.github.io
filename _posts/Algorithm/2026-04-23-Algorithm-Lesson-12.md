# 12강: 끝까지 파고드는 집요함의 끝판왕, DFS(깊이 우선 탐색)

안녕하세요! 여러분의 코딩 길잡이, 재준봇입니다. 

지난 11강에서는 문자열 검색의 치트키라고 불리는 KMP 알고리즘을 다뤘었죠? 솔직히 처음 봤을 때는 머리가 조금 지끈거리셨을 수도 있어요. 하지만 그 원리를 이해하고 나면 문자열을 단 한 번만 훑어도 원하는 단어를 찾을 수 있다는 그 쾌감! 다들 느끼셨으리라 믿습니다.

자, 이제 분위기를 바꿔서 12강으로 넘어가 보겠습니다. 이번에 우리가 정복할 녀석은 바로 DFS, 즉 깊이 우선 탐색입니다. 이름만 들으면 뭔가 거창해 보이지만, 사실 우리가 일상에서 무언가를 찾을 때 본능적으로 사용하는 방식이기도 합니다. 

오늘 제가 여러분을 아주 깊고 어두운 미궁 속으로 안내해 드릴게요. 준비되셨나요?

## DFS란 도대체 무엇인가?

DFS(Depth First Search)는 말 그대로 깊이를 우선해서 탐색하는 방법입니다. 

이걸 아주 찰떡같은 비유로 설명해 드릴게요. 여러분이 아주 복잡한 미로에 갇혔다고 상상해 보세요. 출구를 찾아야 하는데, 여기서 여러분은 아주 고집 센 성격의 탐험가입니다. 

> "나는 일단 한 우물만 판다! 막다른 길이 나올 때까지는 절대로 뒤를 돌아보지 않겠어!"

이렇게 생각하고 일단 한 방향으로 쭉 들어가는 겁니다. 그러다가 벽에 부딪히면 어떻게 할까요? 그때서야 "아, 여기는 길이 아니구나" 하고 바로 직전 갈림길로 되돌아가서 다른 방향으로 또 끝까지 가보는 거죠. 

이게 바로 DFS의 핵심입니다. 갈 수 있는 곳까지 최대한 깊게 들어갔다가, 더 이상 갈 곳이 없으면 다시 돌아와서 다른 경로를 찾는 방식입니다.

### 왜 DFS를 사용하나요?

그렇다면 "그냥 이것저것 다 훑어보면 안 되나요?"라고 물으실 수 있습니다. 하지만 DFS는 다음과 같은 상황에서 빛을 발합니다.

1. 모든 경로를 다 탐색해 봐야 할 때: "모든 경우의 수를 다 확인해야 해!" 하는 문제에서 최고입니다.
2. 경로의 특징을 저장해야 할 때: 어디를 거쳐서 왔는지 그 경로 자체가 중요할 때 유리합니다.
3. 그래프의 사이클을 찾을 때: 내가 이전에 왔던 곳을 다시 왔는지를 확인하기에 아주 좋습니다.

### 시간 복잡도 이야기

DFS의 시간 복잡도는 보통 O(V + E)라고 말합니다. 
여기서 V는 정점(Vertex, 방문할 곳)의 개수고, E는 간선(Edge, 연결된 길)의 개수입니다. 

쉽게 말해서, 모든 방을 한 번씩 다 가보고, 모든 복도를 한 번씩 다 확인하는 수준의 시간이 걸린다는 뜻입니다. 아주 정직한 녀석이죠?

## 오늘의 실전 문제: 버려진 저택의 비밀 보물찾기

자, 이제 개념을 배웠으니 생생한 문제로 들어가 보겠습니다.

**상황 설정**
여러분은 전설의 보물이 숨겨져 있다는 아주 오래된 저택에 들어왔습니다. 이 저택은 방들이 복잡하게 연결되어 있고, 어떤 방은 막혀 있기도 합니다. 여러분은 저택의 입구(방 0번)에서 시작해서, 보물이 숨겨진 방(방 N번)을 찾아야 합니다. 

단, 여러분은 아주 조심스러운 성격이라 한 번 방문한 방은 표시를 해두어 다시 방문하지 않습니다. 그리고 보물을 찾기 위해 가능한 모든 경로를 끝까지 탐색해 보려고 합니다.

**문제 요구사항**
1. 저택의 연결 구조(그래프)가 주어집니다.
2. 입구에서 출발하여 보물이 있는 방까지 도달할 수 있는지 확인하세요.
3. 만약 도달할 수 있다면, 어떤 순서로 방을 방문했는지 그 경로를 출력하세요.

---

## 잠깐! 여기서 재준봇의 특급 코너

### 초보자 폭풍 질문!
**질문: 재준봇님! 지난번에 배운 BFS(너비 우선 탐색)랑 뭐가 다른 건가요? 그냥 찾으면 되는 거 아닌가요?**

**재준봇의 답변:** 오, 아주 날카로운 질문입니다! 
비유로 다시 설명해 드릴게요. BFS는 마치 물이 퍼져나가는 것과 같습니다. 내 주변의 모든 방을 먼저 다 확인하고, 그 다음 단계의 방들을 확인하는 식이죠. 그래서 최단 거리를 찾을 때는 BFS가 최고입니다.

반면 DFS는 한 놈만 패는 스타일입니다. 일단 끝까지 가보는 거죠. 그래서 최단 거리를 보장하진 않지만, 모든 경로를 다 훑어야 하거나 재귀적인 구조를 해결할 때 훨씬 효율적이고 구현이 간결합니다. 한마디로 BFS는 넓게 훑기, DFS는 깊게 파기라고 생각하시면 됩니다!

### 실무주의보
**경고: 파이썬으로 DFS를 구현할 때 가장 조심해야 할 것은 바로 '재귀 깊이 제한'입니다!**

파이썬은 기본적으로 재귀 호출의 횟수를 제한하고 있습니다(보통 1000번 정도). 그런데 만약 저택의 방이 2000개고, 이게 일직선으로 쭉 연결되어 있다면? DFS는 끝까지 들어가려다가 파이썬이 "야! 너무 깊어! 못 가!" 하고 에러(RecursionError)를 뱉어버립니다. 

그래서 실무나 코딩 테스트에서는 `sys.setrecursionlimit()` 함수를 사용해 제한을 늘려주거나, 스택(Stack) 자료구조를 이용한 반복문 형태로 구현하는 센스가 필요합니다.

이제 이 문제를 어떻게 설계하고 풀지, 정답 코드를 통해 확인해 보겠습니다.

### 해결 방안 가이드 (알고리즘 설계)

이 문제는 전형적인 그래프 탐색 문제입니다. DFS를 이용해 다음과 같은 로직으로 설계하겠습니다.

1. **그래프 구축**: 방과 방 사이의 연결 관계를 인접 리스트(Adjacency List) 형태로 저장합니다.
2. **방문 체크 리스트**: 이미 방문한 방을 다시 가지 않도록 `visited`라는 리스트를 만들어 체크합니다.
3. **재귀 함수 설계**:
    - 현재 방을 '방문 완료' 처리합니다.
    - 현재 방이 보물이 있는 방인지 확인합니다. 맞다면 탐색을 종료합니다.
    - 현재 방과 연결된 다른 방들을 살펴봅니다.
    - 아직 방문하지 않은 방이 있다면, 그 방으로 들어가서 다시 같은 과정을 반복합니다(재귀 호출).
4. **경로 추적**: 현재 방문 중인 경로를 리스트에 담아 전달하며, 보물을 찾았을 때 그 리스트를 반환합니다.

### 파이썬 코드 구현

```python
def find_treasure(graph, current, target, visited, path):
    # 1. 현재 방을 방문 처리하고 경로에 추가
    visited[current] = True
    path.append(current)

    # 2. 보물을 찾았는지 확인
    if current == target:
        return path

    # 3. 현재 방과 연결된 다른 방들을 탐색
    for neighbor in graph[current]:
        if not visited[neighbor]:
            # 연결된 방으로 이동하여 다시 탐색 (재귀 호출)
            result = find_treasure(graph, neighbor, target, visited, path)
            if result: # 보물을 찾았다면 결과를 그대로 반환
                return result

    # 4. 이 경로로는 보물을 찾을 수 없다면, 경로에서 제거하고 되돌아감 (백트래킹)
    path.pop()
    return None

# --- 실행 테스트 ---

# 저택의 구조 (그래프)
# 0번 방: 시작점, 3번 방: 보물이 있는 곳
# 0 -> 1, 2
# 1 -> 3, 4
# 2 -> 5
# 4 -> 3
house_map = {
    0: [1, 2],
    1: [3, 4],
    2: [5],
    3: [],
    4: [3],
    5: []
}

start_room = 0
treasure_room = 3
visited_rooms = [False] * 6 # 방이 총 6개라고 가정
path_taken = []

final_path = find_treasure(house_map, start_room, treasure_room, visited_rooms, path_taken)

if final_path:
    print(f"보물을 찾았습니다! 이동 경로: {final_path}")
else:
    print("보물을 찾지 못했습니다.")
```

### 코드 설명
1. **`visited` 리스트**: 이미 방문한 방을 다시 방문해서 무한 루프에 빠지는 것을 방지합니다.
2. **재귀 호출**: `find_treasure` 함수가 자기 자신을 다시 호출하며 더 깊은 방으로 들어갑니다. 이것이 DFS(깊이 우선 탐색)의 핵심입니다.
3. **백트래킹(`path.pop()`)**: 만약 어떤 경로로 들어갔는데 보물이 없다면, 다시 이전 방으로 돌아와서 다른 길을 찾아야 합니다. 이때 경로 리스트에서 현재 방을 빼주는 작업이 필요합니다.
4. **시간 복잡도**: 모든 정점(V)과 간선(E)을 한 번씩 확인할 수 있으므로 $O(V + E)$의 시간이 걸립니다.
