##  22강: 상속 - 코드의 유전자! 🎉

🔥 **안녕하세요, 파이썬 마스터님들이 될 열정찬 학생들!** 🔥  오늘은 파이썬 프로그래밍 세계에 한 층 더 가까워지도록 함께 성장해볼 거예요. 오늘 주제는 바로 **"상속"**이랍니다! 상속이란? 🤔 마치 인간의 유전자처럼 코드를 물려받고 변화시켜 새로운 종류의 클래스를 만들어내는 건축 기술인 셈이죠! 😎

###  🤔 상속, 무엇을 위한 것일까요? 

👨‍💻 개발자들은 여러 기능을 가진 코드들을 재사용하고 싶어하는 마음, 알아요 그러니까 "코드 리팩토링" 같은 건 중요한데, 상속이 바로 이를 가능하게 해주는 기술이에요! 💡

만약 동물들처럼 개와 고양이처럼 다양한 종류의 객체들을 만들고 싶다면? 🤔  개와 고양이는 모두 "동물" 이라는 공통적인 특징을 가지고 있지 않나요? "밥을 먹고, 생명력을 유지한다", 이런 점들이죠!

기존 "동물" 클래스의 코드를 사용해서 개와 고양이 클래스를 만들어보면 어떨까요? 🎉 상속은 바로 이렇게 코드를 효율적으로 재사용하고 변형하는 기술이랍니다!

###  ✨ 상속의 기본 구조 

파이썬에서 상속은 `class` 키워드 다음에 부모 클래스의 이름을 사용하여 진행됩니다. 

> class 자식_클래스(부모_클래스):
>     # 자식 클래스에서 정의하는 코드

예를 들어, "개"와 "고양이"가 각각 상속받는 "동물" 클래스를 만들어보면?


```python
class 동물:  # 부모 클래스 🐶🐱
    def __init__(self, 이름):
        self.이름 = 이름

    def 먹다(self):
        print(f"{self.이름}은 음식을 먹고 있습니다.")

class 개(동물): # 자식 클래스 🐕
    def 짖다(self):
        print(f"{self.이름}이 멍멍!라고 짖습니다.")

class 고양이(동물): # 자식 클래스 🐈
    def 야옹하다(self):
        print(f"{self.이름}이 야옹!라고 합니다.")

뭉치 = 개("뭉치")  # 개 객체 생성
뭉치.먹다() # 부모 클래스에서 상속받은 메소드 호출 가능
뭉치.짖다() # 자식 클래스의 새로운 메소드 호출 가능

야옹이 = 고양이("야옹이") 
야옹이.먹다()  # 동물의 먹는 행동을 상속
야옹이.야옹하다() # 고양이의 특별한 행동


```

**💡 초보자 폭풍 질문!** 🤔

* 위 예시에서 `뭉치` 객체는 어떤 타입일까요? (`개` 클래스의 인스턴스!)
* `짖다()` 메소드는 어디에서 정의되었나요? (`개` 클래스에서!)

### 🚨 실무주의보: 파이썬 상속의 활용!


* **OOP(객체 지향 프로그래밍)** : 상속은 OOP의 핵심 개념 중 하나입니다. 재사용성을 높이고 코드의 가독성과 유지보수를 용이하게 합니다. 💪
* **도메인 모델링** : 현실 세계의 다양한 대상들을 파이썬 클래스로 표현할 때 상속은 매우 효율적입니다. 예를 들어, "차량" 클래스에서 "자동차", "오토바이", "버스" 등을 자손 클래스로 만들 수 있습니다. 🚗🏍️🚌
