### 💡  Singleton 패턴:  코드도 유니크하게 하나만 생성하고 싶다면? 🤔

Singleton 패턴은 **한 번만 인스턴스를 생성하고, 그 인스턴스에만 접근할 수 있도록** 하는 디자인 패턴입니다. 마치 **"전 세계에서 한 개뿐인 최고의 파이썬 강사"** 처럼! 🧙‍♂️  유니크하면서도 모든 사람들이 접근할 수 있는 방법이죠.

> Singleton 패턴은 어떤 상황에 유용할까요?
> - 데이터베이스 연결 정보를 공유하고 싶을 때 (단 하나의 DB 커넥션을 활용)
- 전체 시스템에서 사용되는 설정 파일이나 기본 정보를 관리할 때 (원래 파이썬 프로그램 실행시 윈도우의 모든 창과 같은 UI와 동일한 기능을 제공하는 UI 프레임워크)

###  🤔  Singleton 패턴의 장점:  독점적이고 효율적인 코딩! 🚀

- **데이터 접근 통제**: 데이터베이스 커넥션, 파일 열기 등 중요한 리소스를 한 곳에서 관리하여 중복된 사용을 방지할 수 있습니다.
- **효율성 증가**: 하나의 인스턴스만 생성되므로 메모리 활용도를 높이고 시스템 성능을 향상시킬 수 있습니다.
- **코드 간결성**: 유일한 인스턴스 접근을 위한 간단한 코드로 관리하여 코드 작성 및 유지보수 비용을 줄일 수 있습니다.

### 💻  실제 코딩 예시: Singleton 패턴 만들기 😎

```python
class Singleton:
    _instance = None

    def __new__(cls, *args, **kwargs):
        if not cls._instance:
            cls._instance = super(Singleton, cls).__new__(cls, *args, **kwargs)
        return cls._instance

    def __init__(self, name):  # 생성자 함수 (인스턴스 초기화)
        self.name = name
        print(f"싱글톤 인스턴스 생성: {self.name}")

# 싱글톤 인스턴스 생성 및 사용
s1 = Singleton("Alice")   # 인스턴스를 최초로 생성
s2 = Singleton("Bob")    # 이제는 s1과 같은 인스턴스를 참조하게 됩니다!
print(s1 is s2)        # True

```


**코드 분석**:

* `_instance`:  Singleton 클래스 내부에 유일한 인스턴스를 저장할 변수입니다. 처음에는 None으로 초기화되어 있습니다.
* `__new__(cls, *args, **kwargs)`: 이 부분이 핵심! Singleton을 생성할 때마다 항상 동작합니다. 만약 이미 _instance가 있다면 새 인스턴스를 생성하지 않고 기존 인스턴스를 반환하며,  `super(Singleton, cls).__new__(cls, *args, **kwargs)` 을 통해 새로운 인스턴스를 생성하여 `_instance` 에 할당하고 있습니다.
* `__init__(self, name)`: 이 함수는 인자로 받은 이름을 사용하여 생성된 Singleton 인스턴스의 속성을 설정합니다. 

>  **🚨 실무주의보!**:   Singleton 패턴은 과도하게 사용할 경우 코드 복잡성을 높일 수 있습니다. 따라서 필요한 상황에서만 적용하는 것이 좋습니다. 💡
