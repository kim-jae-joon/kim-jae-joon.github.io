## 18강: 오류 처리 (try/except) - 코드의 요정, 마법사가 되어봐요! ✨

**안녕하세요 파이썬 친구들 👋!** 저희 열렬한 파이썬 여행에 오늘도 함께 해주셔서 진심으로 감사합니다! 🎉  오늘 강좌에서는 **"오류 처리 (try/except)"**라는 파워풀한 기술을 배우겠습니다. 💥 

지금까지 코딩하면서 코드가 깨끗하게 작동하는 모습에 만족하셨지만, 코드는 인간처럼 쓸모없는 오류를 남길 수 있습니다. 😵‍💫  혹시 'ZeroDivisionError'나 'TypeError' 같은 문구를 보았던 적 있나요? 마치 요정에게 깨진 유리병을 들려주고 마법으로 다시 만들어달라고 요청하는 것과 같아요! 🤔 그런데 지금부터 파이썬의 마법사가 되어서 코드 오류를 예측하고, 부드럽게 관리하는 방법을 배울거예요. 😎 

###  왜 오류 처리 필요할까요? 😩😩😩

코드는 우리가 생각하는 것처럼 항상 완벽하지 않아요. 때로는 예외적인 상황이 발생해서 코드가 실행되지 못하거나 원치 않는 결과를 만들기도 합니다. 이런 경우에 **"오류 처리"**가 필요해요! 🚀 


*  예시: 웹사이트에서 계산 시 '0으로 나누기' 오류가 발생할 수 있어요. 😨
   * 일반적인 코드로는 프로그램이 완전히 종료될 수 있습니다. 💥
   * 하지만, 오류 처리를 사용하면 "죄송합니다, 숫자를 다시 입력해주세요!" 라고 메시지를 보여주면서 사용자 경험을 개선할 수 있습니다. 😉

###  마법의 약속 - try/except! ✨✨

파이썬은 'try'와 'except' 블록을 제공하는 강력한 오류 처리 기능을 가지고 있습니다! 💫 


* **`try`:** 코드가 실행될 가능성 있는 부분을 정의합니다.
* **`except`:** `try`블록에서 예상되는 특정 오류 유형이 발생하면, 'except' 블록의 코드를 실행하게 합니다.

###  코드로 맛보기! 🧑‍💻

```python
# ZeroDivisionError 예외 처리
try:
    result = 10 / 0  # 0으로 나누려고 하면 오류 발생!
except ZeroDivisionError:
    print("앗! 0으로 나눌 수는 없어요. 다시 시도해보세요.")

print("프로그램은 계속 실행됩니다!") 
```


**설명:** 💡
* `try` 블록 안에서 10을 0으로 나누려고 하는 코드가 실행되면 `ZeroDivisionError` 오류가 발생합니다. 😩
* `except ZeroDivisionError:` 라는 구문은 지정된 오류 유형 (`ZeroDivisionError`)이 발생하면 해당 부분의 코드를 실행하게 합니다.
* '앗! 0으로 나눌 수는 없어요. 다시 시도해보세요.' 메시지가 출력되면서 프로그램은 원활하게 계속 진행됩니다. 💪



###  다양한 오류 처리 🧲

`except ZeroDivisionError:` 와 같이 특정 오류 유형만 처리하는 것을 **"구체적 오류 처리"** 라고 합니다. 🎯


* 예시: `TypeError`는 자료 형식에 맞지 않는 연산을 시도할 때 발생합니다.
```python
try:
    result = "10" + 5  # 문자열과 숫자를 더하려면 오류 발생!
except TypeError:
    print("다양한 자료형을 사용하면 안되요! 조금 다시 생각해볼까요?")
```



**💡 초보자 폭풍 질문!** 🔥

* 여러 개의 `except` 블록을 추가하여 다양한 오류를 처리할 수 있나요?
* 오류가 발생하지 않으면 어떤 코드가 실행될까요?


###  마법의 최종 결말 - finally 블록 🔮



'finally' 블록은 'try/except' 블록과 함께 사용되고, **어떤 상황이든 항상 실행되는 코드를 포함**할 수 있습니다. 🤩


```python
try:
    file = open("my_file.txt", "r")  # 파일을 열려고 함
    content = file.read()
except FileNotFoundError:
    print("파일을 찾을 수 없어요!")
finally:
    print("무조건 실행되는 코드입니다! 🤖") # 파일이 열렸는지 여부와 상관없이 실행됨
```



**🚨 실무주의보:** ⚠️


* `finally` 블록에서는 **필요한 자원 (파일, 연결 등)을 반드시 해제해야 합니다.** 그렇지 않으면 리소스 누수 문제 발생! 💧
