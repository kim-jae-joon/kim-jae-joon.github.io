## 🔥16강: 모듈 사용하기 - 파이썬 프로그래밍의 마법사!🔥

안녕하세요, 파이썬 매니아 여러분! 👋  저는 대한민국 최고의 파이썬 일타 강사, 흔히 '코딩 코끼리' 라고 불리는 저입니다. 15년 동안 수천 명의 학생들을 성공으로 이끈 저만은 바로 제가 쌓아온 경험과 지식을 가득 담아 여러분에게 파이썬 세계를 완전히 열어줄 거예요! 🚀

오늘부터 시작하는 **'모듈 사용하기'** 강좌는 파이썬 프로그래밍 마법사의 도구, 모듈에 대한 깊이 있는 이해를 얻을 수 있도록 설계된 특별한 시간입니다. 😎 그동안 파이썬 기본 문법과 자료 구조까지 충분히 다뤄봤죠? 그럼 이제 우리는 **작성 효율성과 코드 재사용성**을 높여주는 멋진 모듈 세계로 여정을 시작할 거예요! 🧳

### ✨모듈이란 무엇일까요?✨

>  '모듈'은 파이썬에서 특정 기능이나 코드 블록을 정리해서 재사용 가능한 단위입니다. 생각보다 신기하고 강력하죠? 마치 Lego 조각처럼 여러 모듈들을 연결하여 복잡한 프로그램을 만들 수 있답니다! 🧱

예를 들어, 웹 사이트 개발에 필요한 '날짜' 계산 기능이나 'DB 접속' 코드 등은 각각 별도의 모듈로 구현할 수 있습니다. 이렇게 분리된 모듈들을 프로그램에서 불러오면 원하는 기능을 효율적으로 활용할 수 있죠. 👍

### 💫  모듈 사용 방법!💫

지금부터 '파이썬 마법사'가 되어서 모듈을 사용해보는 거예요! 😎 아주 간단한 예시를 통해 모듈의 기본적인 작동 원리를 알아볼까요?


```python
#  날짜 계산 모듈인 datetime을 불러옵니다.

import datetime

# 현재 시간 정보를 얻어봅니다.
now = datetime.datetime.now()

# 현재 년도 출력합니다.
print(now.year) 
```

> **설명:** 위 코드에서는 `datetime` 모듈을 불러오고, 현재 날짜와 시간 정보를 얻는 `datetime.datetime.now()` 함수를 사용하여 현재 년도를 출력하는 프로그램입니다. 마치 시계로 시간 확인하듯이 간편하게! 🕰️

### 🚀  모듈 생성하기: 내 작은 마법사의 길잡이 🚀

자, 지금까지 모듈을 활용하는 방법을 알아봤는데요? 🤔 그럼 이제는 우리 스스로 **새로운 모듈**을 만들어 보는 건 어떨까요? 🔥 🤯  멋진 마법사가 되기 위한 첫걸음이죠!

#### `모듈 생성하기` 주의 사항들 🚨


* 모듈 파일 이름은 `.py` 확장자를 가지고 있어야 합니다. 예를 들어, 'my_module.py'와 같은 이름으로 저장합니다.
    
* 모듈 내부에는 함수, 변수 등 원하는 코드들을 작성합니다.

* 모듈을 사용하려면 해당 파일을 파이썬 인터프리터에서 실행해야 합니다.


```python
#  my_module.py (모듈 파일)

def greet(name):
    print(f"Hello, {name}!") 

```



> **설명:** 위 코드는 `my_module.py`라는 이름의 모듈을 생성하는 예시입니다. 'greet' 함수를 정의하여 이름을 입력하면 "안녕하세요, [이름]!"와 같은 메시지를 출력하는 기능을 가진 모듈입니다.

>  🚨 **실무주의보:** 
>   모듈 파일은 여러 개의 파일에 분산되어 저장될 수 있습니다. 이때, 각 모듈 간의 상호작용과 의존 관계를 명확히 관리하는 것이 중요합니다.


#### **모듈 사용하기**

지금 만든 `my_module.py` 모듈을 다른 파이썬 코드에서 활용해 볼까요? 😉



```python
#  main.py (모듈 사용 예시)

import my_module 

my_module.greet("철수")  
```

> **설명:** `main.py` 파일에 `import my_module`를 통해 'my_module' 모듈을 불러와서, `my_module.greet("철수")`를 실행하여 "안녕하세요, 철수!"가 출력됩니다. 



### 🎉🎉  모듈 사용의 장점!🎉🎉

이제까지 파이썬 모듈의 기본 개념과 사용 방법을 배웠죠? 🤔 그럼 이 놀라운 도구가 프로그래밍에 어떤 긍정적인 영향을 미치는지 알아볼까요?


* **재사용성:**  한번 작성된 모듈은 여러 프로그램에서 재사용할 수 있어 시간과 노력을 절약합니다.
* **코드 정돈:** 코드를 모듈로 분리하여 관리하면 프로그램의 구조가 명확해지고 유지보수가 용이해집니다. 

* **협업 효율성:** 다양한 개발자가 각각 모듈을 담당하여 작업할 수 있기 때문에 프로젝트 협업 속도를 높일 수 있습니다.
