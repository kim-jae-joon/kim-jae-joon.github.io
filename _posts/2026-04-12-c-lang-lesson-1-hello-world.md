---

## 🚀 왜 하필 C 언어일까요?

세상에는 수많은 프로그래밍 언어가 있어요. 파이썬, 자바, 자바스크립트 등등… 그런데 왜 우리는 하필 C 언어부터 시작할까요?

C 언어는 마치 **모든 프로그래밍 언어의 튼튼한 뿌리**와 같아요. C 언어를 배우면 컴퓨터가 어떻게 작동하는지, 메모리는 어떻게 관리되는지 등 컴퓨터 내부의 깊은 원리를 이해하는 데 큰 도움이 됩니다. 다른 언어들이 C 언어의 개념을 많이 가져다 썼기 때문에, C 언어를 탄탄하게 익혀두면 어떤 언어를 배우든 훨씬 쉽게 느껴질 거예요. 속도도 빠르고, 시스템 프로그래밍(운영체제, 임베디드 등)에도 강력하게 사용되는 정말 매력적인 언어랍니다!

겁먹지 마세요! 처음엔 어려워 보여도, 그 깊이를 알게 되면 엄청난 재미와 성취감을 느낄 수 있을 거예요.

---

## 🛠️ C 언어를 위한 나만의 개발 작업실 꾸미기 (개발 환경 구축)

자, 이제 C 언어를 시작하기 위한 나만의 '작업실'을 꾸며볼 시간이에요. 프로그램을 만들려면 몇 가지 도구가 필요하답니다.

1.  **텍스트 에디터 (Text Editor):** 코드를 작성하는 노트나 종이와 같아요.
2.  **컴파일러 (Compiler):** 우리가 쓴 사람의 언어(C 코드)를 컴퓨터가 이해하는 언어(기계어)로 번역해 주는 번역기예요.
3.  **링커 (Linker):** 번역된 여러 조각의 코드를 하나의 실행 가능한 프로그램으로 묶어주는 역할을 해요. (이 부분은 컴파일러가 알아서 해주니 너무 걱정 마세요!)
4.  **디버거 (Debugger):** 만약 프로그램에 오류가 발생했을 때, 어디서 잘못되었는지 찾아내고 수정할 수 있도록 도와주는 도구예요.

이 모든 도구를 하나로 통합해서 편리하게 사용할 수 있도록 도와주는 것이 바로 **IDE (Integrated Development Environment, 통합 개발 환경)** 입니다. 초보자분들은 IDE를 사용하는 것이 훨씬 편할 거예요!

저와 함께 **Visual Studio Code (VS Code)** 와 **MinGW (윈도우용 C 컴파일러)** 를 설치해볼게요. VS Code는 가볍고 강력하며, 다양한 언어를 지원하는 요즘 개발자들에게 가장 인기 있는 에디터 중 하나입니다.

### 윈도우(Windows) 사용자분들을 위한 필수 설치 가이드 (VS Code + MinGW)

#### 1. MinGW (Minimalist GNU for Windows) 설치

C 언어 코드를 컴퓨터가 이해할 수 있는 언어로 번역(컴파일)해 줄 `gcc` 컴파일러를 설치할 거예요.

*   **설치 파일 다운로드:** [MinGW-w64 설치 관리자 다운로드](https://sourceforge.net/projects/mingw-w64/files/mingw-w64/mingw-w64-release/) 페이지로 이동하여 최신 버전을 다운로드하세요. 보통 `x86_64-posix-seh` (64비트 시스템용) 버전을 선택하시면 됩니다.
    *   다운로드한 `.exe` 파일을 실행하세요.
    *   설치 옵션은 기본값으로 두시는 것이 좋아요.
    *   **Destination Folder:** 설치 경로를 기억해두세요. (예: `C:\Program Files\mingw-w64\x86_64-8.1.0-posix-seh-rt_v6-rev0` 또는 `C:\mingw-w64`)

#### 2. 환경 변수(PATH) 설정

컴퓨터가 `gcc` 명령어를 어디서 찾아야 할지 알려주는 과정이에요. 이 작업을 해야 터미널(명령 프롬프트)에서 `gcc`를 사용할 수 있습니다.

*   **'시작' 버튼**을 클릭하고 `환경 변수`라고 검색하여 **'시스템 환경 변수 편집'** 을 선택합니다.
*   **'환경 변수(N)...'** 버튼을 클릭합니다.
*   **'시스템 변수'** 목록에서 `Path`를 찾아 선택한 후 **'편집(I)...'** 버튼을 클릭합니다.
*   **'새로 만들기(N)'** 를 클릭하고, 아까 MinGW를 설치한 경로의 `bin` 폴더 경로를 입력합니다.
    *   예시: `C:\Program Files\mingw-w64\x86_64-8.1.0-posix-seh-rt_v6-rev0\mingw64\bin` (여러분의 실제 설치 경로에 따라 달라질 수 있습니다!)
*   모든 창을 **'확인'** 버튼을 눌러 닫습니다.

*   **확인:** 명령 프롬프트(CMD)를 열고 `gcc --version`을 입력했을 때, 버전 정보가 나타나면 성공입니다!
    ```bash
    gcc --version
    # gcc (MinGW.org GCC Build-20200227-1) 9.2.0 (예시)
    # Copyright (C) 2019 Free Software Foundation, Inc.
    # This is free software; see the source for copying conditions.  There is NO
    # warranty; not even for MERCHANTABILITY or FITNESS FOR A PARTICULAR PURPOSE.
    ```

#### 3. VS Code (Visual Studio Code) 설치

코드를 작성하고 실행하는 우리의 메인 작업 공간입니다.

*   **다운로드:** [VS Code 공식 웹사이트](https://code.visualstudio.com/) 에 접속하여 윈도우용 설치 파일을 다운로드하고 실행합니다.
*   설치 과정에서 특별한 설정 없이 기본값으로 진행하시면 됩니다. 마지막에 'Visual Studio Code 실행'에 체크되어 있는지 확인하고 '마침'을 누르세요.

#### 4. VS Code C/C++ 확장 설치

VS Code에서 C 언어를 편리하게 사용할 수 있도록 도와주는 확장 프로그램입니다.

*   VS Code를 실행합니다.
*   왼쪽 사이드바에서 **'확장 (Extensions)' 아이콘** (네모 상자 네 개 모양)을 클릭합니다.
*   검색창에 `C/C++`을 입력하고, `Microsoft`에서 제공하는 **'C/C++ Extension Pack'** 을 찾아 **'Install'** 버튼을 클릭하여 설치합니다.

#### 5. MinGW와 VS Code 연결 확인 (선택 사항이지만 추천)

VS Code 터미널에서 `gcc`가 잘 작동하는지 확인하는 과정입니다.

*   VS Code를 열고 `Ctrl + Shift + ` (백틱) 키를 눌러 터미널을 엽니다.
*   터미널에 `gcc --version`을 입력하고 엔터를 눌러보세요. MinGW 설치 시 확인했던 것과 동일하게 버전 정보가 출력되면 성공입니다!

**맥(macOS) / 리눅스(Linux) 사용자분들은 `gcc`가 기본적으로 설치되어 있거나 Homebrew, apt 등을 통해 쉽게 설치할 수 있습니다.** VS Code 설치 후 C/C++ 확장만 설치하시면 됩니다.

## ✨ 우리의 첫 번째 C 프로그램: "Hello, World!"

개발 환경 구축을 마쳤으니, 이제 역사적인 첫 C 프로그램을 작성해볼 시간이에요! "Hello, World!"는 모든 프로그래밍 언어의 첫걸음이자, 화면에 가장 기본적인 메시지를 출력하는 프로그램입니다.

VS Code를 열고 새 파일을 만드세요 (`Ctrl + N` 또는 `File > New File`). 그리고 아래 코드를 입력한 후, `hello.c`라는 이름으로 저장해주세요. (`File > Save As` 또는 `Ctrl + S`, **확장자는 반드시 `.c` 여야 합니다!**)

```c
// hello.c 파일
#include <stdio.h> // 표준 입출력 함수를 사용하기 위한 헤더 파일 포함

int main() { // 프로그램의 시작점, main 함수
    printf("Hello, World!\n"); // "Hello, World!"를 화면에 출력하고 줄 바꿈
    return 0; // 프로그램이 성공적으로 종료되었음을 운영체제에 알림
}
```

### 코드 한 줄 한 줄 뜯어보기

어려워 보이지만, 하나씩 뜯어보면 전혀 어렵지 않아요!

*   `// hello.c 파일`
    *   `//`로 시작하는 줄은 **주석(Comment)** 이라고 해요. 사람이 코드를 이해하기 쉽게 설명해주는 부분으로, 컴파일러는 이 부분을 무시하고 번역하지 않습니다. 마치 노트에 메모를 적는 것과 같아요.

*   `#include <stdio.h>`
    *   `#include`는 **'포함하다'** 라는 뜻이에요. `<stdio.h>`는 **'Standard Input/Output Header'** 의 약자로, C 언어에서 기본적인 입출력(화면에 글자를 보여주거나, 키보드 입력을 받는 등) 기능을 사용할 때 필요한 '도구 상자'라고 생각하시면 돼요. 이 도구 상자를 가져와야 `printf` 같은 함수를 쓸 수 있답니다. 마치 요리사가 칼이나 냄비를 사용하기 위해 주방 도구를 준비하는 것과 같아요.

*   `int main() { ... }`
    *   `main`은 C 프로그램의 **'시작점'** 입니다. 컴퓨터가 프로그램을 실행할 때 가장 먼저 찾아 실행하는 함수예요. 모든 C 프로그램에는 반드시 하나의 `main` 함수가 있어야 합니다.
    *   `int`는 `main` 함수가 정수(Integer) 값을 반환한다는 의미예요.
    *   `()`는 함수라는 것을 나타내고, `{}` (중괄호) 안에는 `main` 함수가 실행할 명령어들이 들어갑니다. 마치 어떤 작업의 시작과 끝을 알려주는 문 같아요.

*   `printf("Hello, World!\n");`
    *   `printf`는 `<stdio.h>`에 들어있는 함수 중 하나로, **화면에 문자열을 출력**할 때 사용해요. 'print formatted'의 줄임말입니다.
    *   `"Hello, World!\n"`는 `printf` 함수에게 "이 내용을 출력해줘!"라고 전달하는 **문자열**이에요. 큰따옴표(`"`)로 묶인 것이 문자열입니다.
    *   `\n`은 **'줄 바꿈(Newline)'** 을 의미하는 특수 문자예요. `Hello, World!`를 출력한 후에 다음 줄로 커서를 옮겨줘서 출력이 깔끔하게 보이게 합니다.
    *   맨 뒤의 `;` (세미콜론)은 **'문장의 마침표'** 와 같아요. C 언어에서는 하나의 명령문이 끝났음을 알릴 때 반드시 세미콜론을 붙여야 합니다. 이걸 빼먹으면 컴파일러가 "에러!"라고 소리칠 거예요.

*   `return 0;`
    *   `main` 함수가 잘 실행되고 **성공적으로 끝났다**는 것을 운영체제에 알려주는 역할이에요. `0`은 보통 '문제없이 잘 끝났다'는 의미로 사용됩니다.

## ▶️ 작성한 코드, 실행해볼까요? (컴파일과 실행)

코드를 작성했으니, 이제 컴퓨터가 이해할 수 있도록 번역하고 실행해야겠죠? 이 과정을 **컴파일(Compile)** 이라고 합니다.

1.  우리가 작성한 C 언어 코드가 들어있는 `hello.c` 파일이 있습니다. (사람이 이해하는 언어)
2.  **컴파일러 (`gcc`)** 가 이 `hello.c` 파일을 읽어서 컴퓨터가 이해하는 **실행 파일 (`hello.exe` 또는 `a.out`)** 로 번역합니다.
3.  이제 이 실행 파일을 실행하면, 우리가 원하는 결과(`Hello, World!`)가 화면에 나타나는 거예요!

### VS Code에서 실행하기

VS Code에서는 이 과정을 아주 쉽게 할 수 있어요.

1.  `hello.c` 파일이 VS Code에서 열려있는지 확인합니다.
2.  상단 메뉴에서 **'Run (실행)' > 'Run Without Debugging (디버깅 없이 실행)'** 을 클릭하거나, 단축키 **`Ctrl + F5`** (Mac: `Cmd + F5`)를 누릅니다.
3.  VS Code 하단에 터미널 창이 열리면서 아래와 같이 "Hello, World!"가 출력되는 것을 볼 수 있을 거예요!

    ```
    Hello, World!
    ```

    만약 `tasks.json` 파일을 생성하라는 메시지가 뜨면, `gcc build active file`을 선택하시면 됩니다. VS Code가 자동으로 컴파일 명령을 설정해 줄 거예요.

### (참고) 직접 터미널에서 실행하기

만약 좀 더 깊이 있게 과정을 이해하고 싶다면, VS Code의 터미널에서 직접 명령어를 입력하여 컴파일하고 실행할 수도 있습니다.

1.  VS Code에서 `Ctrl + Shift + ` (백틱) 키를 눌러 터미널을 엽니다.
2.  `hello.c` 파일이 있는 경로로 이동합니다. (VS Code 터미널은 보통 현재 열려있는 폴더 경로로 시작합니다.)
3.  아래 명령어를 입력하여 컴파일합니다.
    ```bash
    gcc hello.c -o hello
    ```
    *   `gcc`: C 컴파일러를 호출하는 명령어
    *   `hello.c`: 컴파일할 소스 코드 파일 이름
    *   `-o hello`: 생성될 실행 파일의 이름을 `hello`로 지정하는 옵션 (윈도우에서는 `hello.exe`로 자동 생성)

4.  컴파일이 성공적으로 완료되면, `hello.exe` (윈도우) 또는 `hello` (맥/리눅스)라는 실행 파일이 생성됩니다. 이제 이 파일을 실행합니다.
    ```bash
    ./hello
    ```
    *   `./`: 현재 디렉토리에 있는 실행 파일을 의미합니다. (윈도우에서는 그냥 `hello` 라고만 입력해도 됩니다.)

5.  터미널에 `Hello, World!`가 출력되는 것을 확인하세요!

## 🚨 잠깐! 에러가 나도 괜찮아요!

어때요, `Hello, World!`를 성공적으로 출력하셨나요? 만약 에러가 났더라도 절대 좌절하지 마세요! 개발자의 삶은 에러와의 싸움이자, 에러를 해결하며 성장하는 과정입니다.

가장 흔한 에러는 다음과 같아요.

*   **컴파일러를 찾을 수 없습니다 (`gcc` not found):** MinGW 설치 후 환경 변수(PATH) 설정이 제대로 안 되었을 가능성이 커요. 위 2번 과정을 다시 확인해주세요.
*   **세미콜론(;) 누락:** `printf` 문 뒤에 세미콜론을 빼먹었을 경우. C 언어 문법은 엄격해서 이런 작은 실수에도 에러가 납니다.
*   **오타:** `printf`를 `prntf`로 썼다거나, `main`을 `mian`으로 쓰는 등의 오타는 컴파일러가 알 수 없으니 에러를 냅니다.

에러 메시지를 잘 읽어보면 어디가 문제인지 힌트를 얻을 수 있어요. 구글링(검색)도 훌륭한 학습 방법이랍니다!
