## 🚀  31강: GDB 사용하기 - C 코드 디버깅 도구, 이제 더 이상 두려워하지 마세요!

🔥 안녕하세요! 최고의 C언어 강사 '코딩 스승'이라고 불리는 저(나)입니다. 오늘은 C 언어 프로그래밍의 핵심 기술 중 하나인 **GDB (GNU Debugger)**를 알아보도록 하겠습니다. 

🚨 **실무주의보:** GDB는 코드 디버깅에 필수적인 도구이므로, 이 강좌를 통해 GDB 마스터가 되시면 C 언어 개발자가 아니라 *프로그래밍의 마법사*라고 불릴 수 있습니다! 😎

### 🤔  왜 GDB가 필요할까요?

지금까지 C언어 프로그래밍을 배우고 계신 중, 코드를 실행하면 예상치 못한 오류 메시지가 나타나 좌절했던 경험이 있으실 거예요. 😩 이때, 우리의 든든한 동반자 GDB는 등장합니다! 🦸‍♂️

GDB는 C 코드에 스며들어 코드의 작동 과정을 한 줄 한 줄 살피고 문제를 찾아내는 마법 같은 도구입니다. 코드가 어떤 값을 가지는지, 어떤 함수가 실행되는지, 프로그램이 어떻게 진행되는지 **모든 것을 볼 수 있습니다!** 👀✨

### 🤩 GDB의 핵심 기능: 디버깅의 다양한 기술을 자세히 알아보자!

1. **단계별 실행 (Step-by-step execution):** 코드를 한 줄씩 실행하며, 변수 값의 변화를 확인하고 프로그램이 어떻게 작동하는지 정확하게 이해할 수 있습니다. 💪
2. **어디서 멈추는지 설정하기 (Breakpoint setting):** 특정 위치에서 코드 실행을 중단하여 그 부분에 집중적으로 분석할 수 있습니다. 🎯
3. **변수 값 확인 (Variable inspection):** 프로그램 실행 중 변수의 값을 실시간으로 확인하고, 갑작스러운 변화를 파악할 수 있습니다. 📊

###  🤯 GDB 활용법: 간단한 예제부터 시작해보자!

다음은 `Hello World` 프로그램과 GDB를 사용하여 디버깅하는 예시입니다.

```c
#include <stdio.h>

int main() {
    printf("Hello, World!\n"); 
    return 0;
}
```

1. **빌드:** C 코드를 컴파일하고 실행 가능한 프로그램을 만듭니다.
   ```bash
   gcc hello_world.c -o hello_world  
   ```
2. **GDB 실행:** GDB를 실행하여 `hello_world` 프로그램을 디버깅합니다. 
   ```bash
   gdb hello_world
   ```

3. **브레이크포인트 설정:** `printf("Hello, World!\n");` 라인에 브레이크포인트를 설정합니다. 이제 코드가 이 줄에 도달하면 실행이 중단됩니다.
   ```gdb
   break main 
   ```

4. **실행 시작:** `run` 명령으로 프로그램을 실행합니다.


5. **변수 확인:** `print` 명령어로 변수 값을 확인합니다. 예를 들어 `print i` 명령어는 현재 주변에서 값이 있는 변수 
'i'의 값을 보여줍니다. 

   ```gdb
   print i
   ```


6. **단계별 실행:**  `next` 명령어로 코드를 한 줄씩 실행합니다. 프로그램은 브레이크포인트에 도달하면 중단되고 `next`를 사용하여 한 줄씩 진행됩니다. 

   ```gdb
   next
   ```


7. **종료:** `quit` 명령어로 GDB를 종료합니다.

   ```gdb
   quit
   ```

### 🤔  추가적인 탐험:

* **다양한 브레이크포인트 설정:** 숫자, 문자열, 변수 값 기준으로 브레이크포인트를 설정할 수 있습니다.
* **watchpoint 활용:** 특정 변수의 값이 변경되면 실행을 중단하는 기능을 활용해 코드 흐름 파악에 도움을 받을 수 있습니다.
* **callstack 분석:** 함수 호출 순서를 확인하고 불필요한 함수 호출을 줄이는 데 유용합니다.
