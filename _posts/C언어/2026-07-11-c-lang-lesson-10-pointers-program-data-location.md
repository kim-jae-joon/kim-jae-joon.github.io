**10강: 포인터 - 프로그램의 데이터 위치 파악**

😎 오늘까지 포인터라는 핵심 개념을 이해하셨나요? 그럼, 이제부터는 정말 재밌는 주제에 대해 알아볼게요! 💡 **포인터** 는 변수의 주소를 저장하는 데 사용되는 특수한 형태의 변수입니다. 프로그램에서 데이터를 다루는 방법에 대해 더深히 공부할 수 있게 도와줄 수 있는 강력한 도구입니다.

### 포인터란?

💬 첫 번째 질문은 "포인터"라는 단어를 들었을 때 가장 먼저 생각하는 사람들에게는 "뭘지?"일 것입니다. 그럼, 포인터가 뭔가요? 😊

```c
int x = 10;
```

위의 코드에서 `x`라는 변수에 10이라는 값을 할당했습니다. 이변수 `x`에는 실제 데이터가 저장된 주소가 있습니다. 이 주소를 얻기 위해 우리는 포인터를 사용할 수 있습니다.

```c
int* px = &x;
```

여기서 `&`는 "주소"의 의미입니다. 따라서, `px`라는 포인터 변수는 `x`의 주소를 저장하고 있음을 의미합니다. 이 포인터는 `x`에 저장된 데이터에 접근할 수 있습니다.

### 포인터의 용도

🚀 포인터를 사용할 때마다, 가장 먼저 궁금해하는 사람은 "이거 어떻게 사용해야 하나?"입니다. 하지만, 포인터를 사용하려면 꼭 알아야 하는 점이 있습니다. 그건 바로 **포인터는 실제 데이터에 접근하기 위해 주소를 저장하는 변수**라는 점입니다.

```c
#include <stdio.h>

int main() {
    int x = 10;
    int* px = &x;

    printf("주소: %p\n", (void*)px);
    printf("데이터: %d\n", *px);

    return 0;
}
```

위의 코드에서 `printf` 함수를 사용하여 포인터가 저장한 주소와 실제 데이터를 출력했습니다. 이 중에서 `*px`는 포인터에 저장된 주소를 따라 actual 데이터에 접근합니다.

### 포인터의 종류

📚 이제부터는 포인터의 종류에 대해 알아볼게요! 포인터는 변수에 저장되는 값에 따라 **배열 포인터, 함수 포인터, 일반 포인터**로 분류할 수 있습니다. 이중에서 가장 흔히 사용하는 것은 일반 포인터입니다.

```c
#include <stdio.h>

int main() {
    int x = 10;
    int* px = &x;

    printf("주소: %p\n", (void*)px);
    printf("데이터: %d\n", *px);

    return 0;
}
```

위의 코드에서 일반 포인터를 사용하였습니다. 하지만, 만약에 배열이나 구조체 같은 데이터 타입을 다루고 싶다면 이들이 제공하는 다양한 함수와 함께 사용해야 합니다.

```c
#include <stdio.h>

int main() {
    int x[5] = {1, 2, 3, 4, 5};
    int* px = x;

    printf("주소: %p\n", (void*)px);
    printf("데이터: %d\n", *px);

    return 0;
}
```

위의 코드에서 배열 포인터를 사용하였습니다. 이처럼 다양한 데이터 타입에 대해 포인터를 사용하는 방법은 많으며, 이것이 정말 유용한 이유입니다!

### 포인터의 한계

💔 하지만, 포인터는 무제한으로 사용할 수 없습니다. 그 이유는 **포인터가 실제 데이터의 주소를 저장하고 있기 때문**입니다.

```c
#include <stdio.h>

int main() {
    int* px = NULL;

    printf("주소: %p\n", (void*)px);
    printf("데이터: %d\n", *px);

    return 0;
}
```

위의 코드에서 `NULL` 포인터를 사용하였습니다. 만약에 이 포인터가 실제 데이터의 주소를 저장하지 않으면, 아래와 같은 오류가 발생할 것입니다.

```c
Segmentation fault (core dumped)
```

이러한 오류는 포인터가 실제 데이터의 주소를 저장하지 않았기 때문에 생기는 오류이며, 따라서 포인터를 사용할 때 항상 주의해야 합니다.

### 포인터의 장점

🚀 하지만, 포인터는 정말 유용합니다. 그 이유는 **포인터가 프로그램에서 데이터를 다루는 가장 빠르고 효율적인 방법**이기 때문입니다.

```c
#include <stdio.h>

int main() {
    int x = 10;
    int* px = &x;

    printf("주소: %p\n", (void*)px);
    printf("데이터: %d\n", *px);

    return 0;
}
```

위의 코드에서 포인터를 사용하여 실제 데이터에 접근하였습니다. 만약에 이 방법으로 데이터를 다루지 않는다면, 아래와 같은 오류가 발생할 것입니다.

```c
Segmentation fault (core dumped)
```

이러한 오류는 프로그램이 실제 데이터의 주소를 저장하지 않았기 때문에 생기는 오류이며, 따라서 포인터를 사용하는 것이真的 유용합니다!
