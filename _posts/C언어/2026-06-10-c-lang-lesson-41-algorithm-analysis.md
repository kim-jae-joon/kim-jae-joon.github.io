###  💥 **시간 복잡도: 알고리즘의 속도**

시간복잡도는 알고리즘이 실행되는데 걸리는 시간을 나타내는 수학적 표현입니다. 마치 장소까지 가는 데 걸리는 시간처럼, 알고리즘의 크기가 커질수록 계산량이 늘어나서 시간이 오래 걸립니다!

**예시**: "1부터 10까지 합을 구하는 코드"를 생각해보세요. 이때 입력값이 10일 때 실행되는 시간, 100일 때 실행되는 시간은 어떻게 될까요? 🤔 정답: 입력값이 커질수록 계산 횟수가 증가하여 실행 시간이 오래 걸린다는 거죠!

```c
int sum(int n) {  
    int total = 0; // 초기화, 숫자들을 더할 공간을 만듭니다.
    for (int i = 1; i <= n; ++i) { // 1부터 n까지 반복문 돌려 합계를 계산합니다!
        total += i; 
    }
    return total; 
}

// 결과: sum(10) 실행하면 55가 출력되고, sum(100) 실행하면 시간이 오래 걸리게 될 거예요!
```

**💡 초보자 폭풍 질문!**:  무슨 코드가 더 빠를까요? 🤔  시간 복잡도는 이렇게 비교할 수 있죠. 알고리즘의 효율성을 이해하면 더 빠르고 안정적인 C언어 프로그램을 만들 수 있다는 점이 중요합니다! 🎯

###  🌌 **공간 복잡도: 알고리즘이 사용하는 메모리**

공간복잡도는 알고리즘이 실행 중에 사용하는 메모리의 양을 나타내는 것입니다. 마치 방 개수를 이용하여 집의 크기를 비교하듯, 코드의 복잡도가 높아질수록 더 많은 메모리를 사용하게 됩니다!

**예시**: "텍스트 파일을 저장하는 프로그램"은 파일 크기에 따라 필요한 메모리 양이 달라지죠. 큰 파일일수록 더 많은 메모리가 필요합니다.


```c
// 함수 정의: 문자열을 저장하기 위한 공간을 먼저 만들어야 합니다!
void save_text(char* filename, char* text) {
    FILE *fp = fopen(filename, "w"); // 파일 열기
    fprintf(fp, "%s", text); // 파일로 내용 쓰기
    fclose(fp); // 파일 닫기
}
```

**🚨 실무주의보**: 너무 많은 메모리를 사용하면 프로그램이 느려지거나 오류가 발생할 수 있으니 주의하세요! 😅 공간 복잡도를 최적화하는 것은 C언어 개발자에게 매우 중요한 기술입니다.


### 🧙‍♂️ **시간 & 공간 복잡도 분석: 알고리즘 마스터의 비밀**

알고리즘 분석은 단순히 시간과 공간을 고려하는 것이 아니라, 프로그램의 효율성을 평가하고 최적화하는 데 필수적인 기술입니다. 😉


*   **빅 O 표기법**:  복잡도를 나타내는 수학적 표현 

    `O(1)`: 상수 시간 복잡도 (무조건 동일한 시간 소요)
    `O(log n)`: 로그 시간 복잡도 (n이 커짐에 따라 log n만큼 증가)
    `O(n)`: 선형 시간 복잡도 (n이 커질수록 n배 증가)
    `O(n log n)`: n log n 시간 복잡도 (선형 로그 함수, 중간 정도!)
    `O(n^2)`: 제곱 시간 복잡도 (n이 두 배로 증가할 때 시간이 네 배 증가)

*   **알고리즘 분석**:  코드를 작성하고 실행하여 시간과 공간 복잡도를 계산하는 과정입니다.
