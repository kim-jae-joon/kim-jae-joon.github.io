# 2강: Rust 기본 문법 - 타입 시스템: 타입 마스터가 되는 당신의 여정

안녕하세요, 코딩의 세계에 오신 것을 환영합니다! 오늘은 Rust 언어의 핵심 중 하나인 **타입 시스템**에 대해 깊이 파고들어볼 거예요. 만약 코딩이 처음이라면 걱정 마세요, 여러분도 이 강좌를 통해 타입 마스터가 될 수 있습니다! 💪

## 타입 시스템이란 무엇인가요? 🤔

타입 시스템이란 프로그래밍 언어에서 변수나 데이터가 어떤 종류의 값을 가지는지 명확히 지정하고 관리하는 시스템을 말합니다. 쉽게 말해, 컴퓨터에게 "이 데이터는 숫자야, 이건 문자열이야" 라고 알려주는 역할을 하는 거죠. Rust에서는 타입 시스템이 특히 강력해서 안전성과 효율성을 극대화합니다.

### 왜 타입 시스템이 중요할까요? 🤔

- **안전성 향상**: 타입 오류를 컴파일 시점에 잡아줘서 런타임 오류를 줄입니다.
- **코드 가독성**: 명확한 타입 지정은 코드의 의도를 더 쉽게 이해할 수 있게 합니다.
- **성능 최적화**: 컴파일러가 타입 정보를 활용해 최적의 실행 코드를 생성할 수 있습니다.

## 기본 타입들 소개 🎉

Rust에서 주로 사용되는 기본 타입들을 살펴보겠습니다. 간단한 예제를 통해 하나씩 이해해 보겠습니다.

### 1. 정수 타입 (`i32`, `u32`, `f64`)

#### 예제 코드 1: 정수 타입 사용
```rust
fn main() {
    // 32비트 부호 있는 정수
    let signed_int: i32 = 42; // 42는 정수형 변수에 할당됩니다.
    println!("Signed Integer: {}", signed_int); // 출력: Signed Integer: 42

    // 32비트 부호 없는 정수 (보통 메모리 효율성이 좋을 때 사용)
    let unsigned_int: u32 = 42; // 양수만 할당 가능
    println!("Unsigned Integer: {}", unsigned_int); // 출력: Unsigned Integer: 42

    // 64비트 부동소수점 숫자
    let float_num: f64 = 3.14; // 부동소수점 숫자는 실수 연산에 적합
    println!("Float Number: {}", float_num); // 출력: Float Number: 3.14
}
```

**해설:**
- `i32`: 부호 있는 32비트 정수입니다. 음수와 양수 모두 표현 가능합니다.
- `u32`: 부호 없는 32비트 정수로, 오직 양수만 표현 가능합니다. 메모리 효율성이 좋습니다.
- `f64`: 64비트 부동소수점 숫자로, 정밀한 실수 연산에 사용됩니다.

### 2. 문자열 타입 (`String`, `&str`)

#### 예제 코드 2: 문자열 타입 사용
```rust
fn main() {
    // 고정 크기 문자열 슬라이스 (가변 길이 문자열을 참조)
    let path = "C:/Users/User/Documents"; // 문자열 리터럴
    println!("Path: {}", path); // 출력: Path: C:/Users/User/Documents

    // String 타입: 동적으로 크기를 조정할 수 있는 문자열
    let mut dynamic_string = String::from("Hello, Rust!"); // 동적으로 크기 조정 가능
    dynamic_string.push_str(", 타입 마스터!"); // 문자열에 추가
    println!("Dynamic String: {}", dynamic_string); // 출력: Dynamic String: Hello, Rust!, 타입 마스터!
}
```

**해설:**
- `&str`: 문자열 슬라이스로, 주로 함수 인자나 고정된 문자열 리터럴에서 사용됩니다.
- `String`: 동적으로 크기를 조정할 수 있는 문자열 타입입니다. `.push_str()` 메서드를 사용해 문자열을 확장할 수 있습니다.

### 3. 불리언 타입 (`bool`)

#### 예제 코드 3: 불리언 타입 사용
```rust
fn main() {
    // 불리언 값
    let is_raining: bool = true; // 비가 오는지 여부
    let is_sunny: bool = false; // 해가 비치는지 여부

    // 조건문 예시
    if is_raining {
        println!("우산을 챙기세요!"); // 출력: 우산을 챙기세요! (만약 true라면)
    } else {
        println!("햇살이 밝네요!"); // 출력: 햇살이 밝네요! (만약 false라면)
    }
}
```

**해설:**
- `bool`: `true` 또는 `false` 두 가지 값만 가질 수 있는 타입입니다.
- 조건문에서 불리언 값을 활용해 다양한 동작을 제어할 수 있습니다.

### 타입 추론 (Type Inference) 🌟

Rust는 타입 추론 기능을 통해 코드를 간결하게 만들 수 있습니다. 컴파일러가 변수 타입을 자동으로 결정할 수 있습니다.

#### 예제 코드 4: 타입 추론 활용
```rust
fn main() {
    let number = 42; // 컴파일러가 i32 타입으로 자동 추론
    println!("Number: {}", number); // 출력: Number: 42

    let text = "안녕하세요, Rust!"; // String 타입으로 자동 추론
    println!("Text: {}", text); // 출력: Text: 안녕하세요, Rust!
}
```

**해설:**
- 위 코드에서 `number`와 `text`는 각각 `i32`와 `String` 타입으로 자동으로 추론됩니다. 이로 인해 코드가 더 간결해집니다.

## 초보자 폭풍 질문! 💡

**Q1:** 타입이 지정되지 않은 변수는 어떻게 작동하나요?
**A1:** Rust에서는 명시적으로 타입을 지정해야 합니다. 타입을 명시하지 않으면 컴파일 오류가 발생합니다. 예를 들어, `let x = 5;`는 정상 작동하지만, `let x = "five";`처럼 타입이 다른 경우 컴파일러는 오류를 발생시킵니다.

**Q2:** 타입 변환(Type Casting)은 언제 필요할까요?
**A2:** 타입 변환은 주로 다른 타입 간의 연산이 필요할 때 사용됩니다. 예를 들어, 정수와 부동소수점 숫자 간의 연산에서 정밀도를 맞추거나, 특정 타입이 요구되는 함수 호출 시 필요합니다. 하지만 Rust에서는 이러한 변환이 컴파일 시점에 엄격하게 관리되므로 주의가 필요합니다.

## 실무 주의보 🚨

타입 시스템은 개발 초기 단계에서는 조금 더 시간을 필요로 할 수 있습니다. 하지만 장기적으로 보면 타입 오류를 미리 잡아서 안정적인 소프트웨어를 개발하는 데 큰 도움이 됩니다. 특히 대규모 프로젝트에서는 타입 안정성이 핵심입니다.
