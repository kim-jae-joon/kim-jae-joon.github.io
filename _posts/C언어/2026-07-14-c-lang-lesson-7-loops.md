## 🚀 7강: 반복의 마법, 반복문 마스터하기! 😎 (for, while, do-while)

**"똑같은 일을 계속해서 해야 한다고? 그럴 시간이 없어요!"** 라고 생각하시나요? 걱정 마세요! 반복문을 익히면 컴퓨터에게 지루한 반복 작업을 맡겨두고 훨씬 더 멋진 코드를 만들 수 있게 됩니다! 🤩

**🔥 반복문의 세 영웅: for, while, do-while**

반복문은 크게 세 가지 종류로 나뉩니다. 각각의 영웅은 특유의 힘을 가지고 있어요. 이제 하나씩 만나볼까요?

### 1. 🧙‍♂️ **for 루프: 정확한 횟수만큼 반복!**

**for 루프**는 마치 **"한 번, 두 번, 세 번... 딱 정해진 횟수만큼!"** 이라는 지시어를 던지는 것과 같습니다.  

**개념 설명:**

* **초기화 (Initialization):**  "출발선에 서서 준비운동 완료!" 변수를 초기값으로 설정합니다. 예를 들어, 숫자를 세기 시작할 때 1부터 시작한다고 가정해 봅시다.
* **조건 (Condition):** "몇 번 반복할지 정해줄게!"  반복을 계속할지 멈출지 결정하는 조건문입니다. 예를 들어, 10번까지 반복할지 정할 수 있습니다.
* **증감 (Increment/Decrement):** "한 발짝씩 앞으로!"  매 반복마다 변수 값을 얼마나 올릴지, 내릴지 결정합니다. 숫자를 세는 경우에는 1씩 증가시키죠.

**코드 예시:**

```c
#include <stdio.h>

int main() {
    // 1부터 5까지의 숫자를 출력하는 for 루프
    for (int i = 1; i <= 5; i++) { // 초기화: i = 1, 조건: i <= 5, 증감: i++
        printf("반복 횟수: %d\n", i); // 현재 반복 횟수 출력
    }
    return 0;
}
```

**코드 해부:**

1. `for (int i = 1; i <= 5; i++)`:  
    - `int i = 1`: 변수 `i`를 1로 초기화합니다. (출발선!)
    - `i <= 5`: `i`가 5보다 작거나 같을 때까지 루프를 계속합니다. (몇 번 반복할지 정해주기)
    - `i++`: 매번 루프 실행 후 `i` 값을 1 증가시킵니다. (한 발짝씩 앞으로!)

2. `printf("반복 횟수: %d\n", i);`: 현재 반복 횟수 `i`를 출력합니다.

**실무 활용 팁:**

* **배열 처리:**  배열 요소에 접근하거나 조작할 때 유용합니다. 예를 들어, 학생들의 성적을 모두 출력하려면 `for` 루프를 사용하면 편리합니다.
* **일정 간격의 작업:** 특정 시간 간격으로 작업을 수행해야 할 때에도 적합합니다. 예를 들어, 게임에서 캐릭터의 움직임을 일정 시간 간격으로 업데이트할 때 사용됩니다.

**💡 초보자 폭풍 질문!**

* **Q: `for` 루프에서 조건이 만족하지 않으면 어떻게 될까요?**
* **A:**  조건이 거짓이 되면 루프가 종료되고 코드는 다음 라인으로 넘어갑니다. 마치 춤을 추다가 멈추는 순간처럼 생각하면 됩니다!

### 2. 🧞‍♀️ **while 루프: "조건 만족할 때까지 계속!"**

**while 루프**는 **"특정 조건이 참일 때까지 계속 반복!"** 라는 마법 주문입니다.  

**개념 설명:**

* **조건 (Condition):** 루프 실행 여부를 결정하는 핵심 조건문입니다. 조건이 참이면 루프가 계속되고, 거짓이면 종료됩니다.

**코드 예시:**

```c
#include <stdio.h>

int main() {
    int guess = 0; // 추측 값 초기화
    int number = 5; // 정답 숫자 (비밀로 간직!)

    // guess가 정답 숫자보다 작을 때까지 반복
    while (guess < number) { // 조건: guess가 number보다 작을 때까지
        printf("숫자를 맞춰보세요! (0~%d 사이)\n", number);
        scanf("%d", &guess); // 사용자 입력 받기
        printf("정답은 %d 입니다.\n", number); // 정답 알려주기 (힌트!)
    }
    printf("축하합니다! 정답을 맞추셨습니다.\n");
    return 0;
}
```

**코드 해부:**

1. `int guess = 0; int number = 5;`: 사용자 추측값 (`guess`)과 정답값 (`number`)을 초기화합니다.
2. `while (guess < number) { ... }`: 사용자가 정답을 맞출 때까지 루프를 반복합니다.
3. `printf`, `scanf`: 사용자 입력을 받고 결과를 출력합니다.

**실무 활용 팁:**

* **입력 검증:** 사용자 입력이 특정 조건을 만족할 때까지 계속해서 입력을 받을 때 유용합니다. 예를 들어, 올바른 형식의 데이터 입력을 요구할 때 사용됩니다.
* **이벤트 처리:** 특정 이벤트가 발생할 때까지 계속 기다리는 상황에 적합합니다. 예를 들어, 사용자가 버튼을 클릭할 때까지 기다리는 프로그램에서 활용됩니다.

**🚨 실무주의보:**

* **무한 루프 주의:** `while` 루프의 조건이 영원히 참이 되는 경우 (**무한 루프**) 프로그램이 멈추어버릴 수 있습니다. 항상 루프 종료 조건을 명확하게 설정하는 것이 중요합니다!

### 3. 🪄 **do-while 루프: "먼저 실행하고 조건 확인!"**

**do-while 루프**는 **"먼저 코드를 실행하고 나서 조건을 확인해 루프 여부를 결정!"** 하는 독특한 마법입니다. 마치 **"한번 해보고, 괜찮으면 다시 해봐!"** 라는 느낌이죠.

**개념 설명:**

* **초기 코드 블록:** 루프 시작 전에 **반드시** 실행됩니다.
* **조건 확인:** 루프 종료 여부를 결정하는 조건을 루프 끝에서 확인합니다.

**코드 예시:**

```c
#include <stdio.h>

int main() {
    int userChoice = 1; // 사용자 선택 초기값

    do { // 조건 확인 전에 실행
        printf("메뉴 선택:\n");
        printf("1. 게임 시작\n");
        printf("2. 설정 변경\n");
        printf("선택: ");
        scanf("%d", &userChoice); // 사용자 입력 받기

        if (userChoice == 1) {
            printf("게임 시작!\n");
        } else if (userChoice == 2) {
            printf("설정 변경 완료!\n");
        } else {
            printf("잘못된 선택입니다.\n");
        }
    } while (userChoice != 0); // 조건: userChoice가 0이 아닐 때까지 반복

    printf("프로그램 종료!\n");
    return 0;
}
```

**코드 해부:**

1. `do { ... } while (userChoice != 0);`: 루프 본체 코드가 먼저 실행되고, 조건 (`userChoice != 0`)을 루프 끝에서 확인합니다.
2. 사용자 메뉴 선택과 입력 처리: 메뉴를 출력하고 사용자 선택을 받습니다.
3. 선택에 따른 동작: 선택에 따라 다른 동작을 수행합니다.
4. 루프 종료: 사용자가 0을 선택할 때까지 반복합니다.

**실무 활용 팁:**

* **사용자 입력 검증:** 사용자 입력을 받아서 유효한 범위 내에 있는지 확인할 때 유용합니다. 예를 들어, 메뉴 선택이나 옵션 설정 시 유효한 값인지 검사할 때 사용됩니다.
* **최소한의 반복:**  조건 확인 전에 최소한 한 번 코드를 실행해야 하는 경우에 적합합니다.

**💡 초보자 폭풍 질문!**

* **Q: `do-while` 루프와 `while` 루프의 가장 큰 차이점은 무엇일까요?**
* **A:** `do-while` 루프는 코드 블록을 **한 번 먼저 실행**한 후 조건을 확인합니다. 반면 `while` 루프는 **조건부터 확인**하고 조건이 참이면 코드를 실행합니다. 마치 순서가 다른 두 가지 마법 주문처럼 생각하면 쉽겠죠?
