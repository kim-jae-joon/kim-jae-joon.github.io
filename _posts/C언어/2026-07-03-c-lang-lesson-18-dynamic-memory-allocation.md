## 🤔 기존 메모리 할당 vs. 동적 메모리 할당: 어떤 차이가 있을까요?

우리가 프로그램을 만드는 과정에서 '메모리'라는 공간에 변수를 저장하는 것을 생각해 보세요. 컴퓨터는 일종의 수납공간처럼, 우리가 사용하려고 하는 코드와 데이터를 담을 공간이 있습니다. 이때, C 언어에서는 두 가지 주요 방식으로 메모리를 할당할 수 있습니다.

* **정적 메모리 할당**: 프로그램 실행 전에 미리 정해진 크기의 메모리를 확보하는 방법입니다. 💡 예를 들어, 우리가 변수 'age'를 만들고 그 크기를 4바이트로 정했다면, 프로그램이 시작되자마자 4바이트 공간을 메모리에서 차지하게 됩니다.
* **동적 메모리 할당**: 프로그램 실행 중에 필요한 만큼의 메모리를 실시간으로 요청하는 방법입니다. 마치 아파트를 임대하듯, 프로그램이 실행되는 동안 필요해질 때마다 해당 크기의 공간을 메모리에서 가져오고, 사용하지 않으면 돌려주는 방식입니다. 🚀

### 💡 초보자 폭풍 질문!
> "그럼 정적 메모리 할당이 더 나은 건가요?" 🤔 🤔 🤔

잘 맞춰 보셨네요!  😊 정적 메모리 할당은 예상 가능한 크기의 변수들을 사용할 때 효율적으로 작동합니다. 하지만, 프로그램 내부에서 데이터의 크기가 알 수 없는 경우나, 실행 중에 데이터의 양이 유동적으로 변하는 경우, 동적 메모리 할당을 활용해야 합니다!

## ✨  malloc()와 free(): 동적 메모리 마법사 🔮

C언어에서 동적 메모리 할당을 담당하는 주요 함수 두 개를 소개합니다. 이 두 친구들은 C 언어 프로그래머에게 필수적인 도구입니다!

* **malloc()**: "**me**mory **alloc**ate"의 약자로, 필요한 크기만큼 메모리를 요청하는 함수입니다. 🪄
* **free()**: "**free** memory"의 약자로, 사용하지 않는 메모리 공간을 돌려주는 함수입니다. 🙏

### 🔥  malloc()를 활용해보세요!

`malloc()` 함수는 주어진 크기를 바이트 단위로 입력받아 동적 메모리 할당을 처리합니다. 
마치, "Hey C 언어, 나에게 10바이트 크기의 공간이 필요하단다!"라고 말하는 것과 같습니다. 😉

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
  // 변수 'ptr'는 malloc 함수를 통해 할당된 메모리 주소를 저장할 포인터입니다.
  int *ptr = (int *)malloc(10 * sizeof(int)); 

  if (ptr == NULL) { // 메모리 할당 실패 시 처리
    printf("메모리 할당에 실패했습니다!\n");
    return 1; 
  }

  // ptr를 통해 할당된 메모리 공간을 활용할 수 있습니다.
  for (int i = 0; i < 10; i++) {
    ptr[i] = i * 2; // 각 위치에 값을 저장
    printf("ptr[%d]: %d\n", i, ptr[i]);
  }

  // 사용하지 않는 메모리를 돌려주는 free 함수를 호출합니다. 
  free(ptr); 
  return 0;
}
```


**코드 설명:**

1. `#include <stdlib.h>`: 'malloc()' 및 'free()' 함수가 정의되어 있는 헤더 파일을 포함합니다.
2. `int *ptr = (int *)malloc(10 * sizeof(int));`: 
    -  `malloc()` 함수를 사용하여 10개의 정수 (`int`) 크기로 메모리를 할당합니다. 
    - `sizeof(int)`는 하나의 정수가 차지하는 바이트 크기를 반환하며, 우리 경우 4바이트입니다. 따라서 총 할당되는 메모리 크기는 10 * 4 = 40바이트 입니다!
3. `if (ptr == NULL) { ... }`:  `malloc()` 함수가 성공적으로 메모리를 할당하지 못했을 때, `ptr`는 NULL 값이 들어갑니다. 이를 확인하여 프로그램 오류를 방지합니다.
4. `free(ptr);`: 사용된 메모리 공간을 다시 해제하여 다른 프로그램들이 사용할 수 있도록 합니다.

### 🚨 실무주의보: free() 함수가 없으면 메모리 누수 발생! 😭


`free()` 함수는 동적 메모리를 돌려주고, 프로그램이 안정적으로 실행되기 위해 필수입니다!  
메모리를 할당했지만 `free()`를 호출하지 않고 프로그램을 종료하면 "메모리 누수" 문제가 발생하며, 이는 프로그램의 성능 저하 및 심각한 오류로 이어질 수 있습니다. 


## 🚀 동적 메모리 할당: 실제 개발에서 어떻게 활용할까요?

동적 메모리 할당은 다양한 상황에서 유용하게 사용됩니다.  

* **데이터 크기가 알수 없는 경우**: 프로그램 실행 중에 입력되는 데이터 크기가 미리 알 수 없을 때, 동적 메모리를 활용하여 필요한 만큼의 공간을 확보할 수 있습니다.
* **변동적인 데이터 처리**: 실시간 분석, 게임 개발 등 데이터의 크기가 유동적으로 변하는 상황에서 매우 효과적입니다.
* **자원 관리**: 파일을 읽고 쓰거나 네트워크 통신을 할 때, 필요한 메모리 공간을 동적으로 할당하여 자원을 최대한 활용할 수 있습니다.
