## 10강: C언어 심화 - 구조체와 공용체: 코드 속의 슈퍼히어로 팀!

**안녕하세요, 코딩의 모험가 여러분!** 오늘은 C언어의 진짜 숨겨진 보물, **구조체와 공용체**에 대해 탐험해볼 시간입니다. 이 두 친구는 마치 코드 속의 슈퍼히어로 팀처럼, 복잡한 데이터를 한 몸에 묶어 효율적으로 관리해주는 능력을 지녔어요!

**"구조체? 공용체? 그게 뭐죠? 제 코드는 왜 이렇게 복잡해진 거죠?"** 🤯 걱정 마세요! 걱정은 잠시 내려놓고, 함께 즐겁게 배우며 코드의 새로운 세계를 열어보도록 하죠!

### 🏆 구조체: 데이터 뭉치의 마스터

구조체는 마치 여러 개의 상자를 하나로 묶어 보관하는 **다용도 보관함**과 같습니다. 각각의 상자에는 다양한 데이터 타입 (숫자, 문자, 포인터 등)을 담아낼 수 있어요. 하나의 구조체 변수로 여러 종류의 정보를 깔끔하게 관리할 수 있다니, 정말 멋지죠?

**예시: 학생 정보 관리**

```c
#include <stdio.h>

// 구조체 선언: 학생 정보를 담을 '학생' 구조체 정의
struct Student {
    char name[50]; // 이름 (문자열)
    int age;       // 나이 (정수)
    float gpa;     // 학점 평균 (실수)
};

int main() {
    // 구조체 변수 생성: '홍길동' 학생 정보를 담을 변수
    struct Student kim;

    // 각 멤버 변수에 값 할당
    strcpy(kim.name, "홍길동"); // 이름 입력 (strcpy 함수 사용: 문자열 복사)
    kim.age = 20;              // 나이 입력
    kim.gpa = 3.8;             // 학점 평균 입력

    // 정보 출력
    printf("이름: %s\n", kim.name); // kim.name 에서 이름 출력
    printf("나이: %d\n", kim.age);   // kim.age 에서 나이 출력
    printf("학점 평균: %.2f\n", kim.gpa); // kim.gpa 에서 학점 평균 출력

    return 0;
}
```

**코드 해설:**

1. **`struct Student`**: 학생 정보를 담을 구조체 'Student'를 정의합니다. 각 멤버 변수는 데이터 타입과 함께 이름을 가지며, 데이터를 저장할 공간을 만듭니다.
2. **`struct Student kim;`**: 'kim'이라는 이름의 구조체 변수를 선언합니다. 이제 'kim'은 이름, 나이, 학점 평균을 모두 담을 수 있는 컨테이너가 됩니다!
3. **`strcpy(kim.name, "홍길동");`**: 문자열 복사 함수 `strcpy`를 사용하여 'kim.name'에 이름을 저장합니다. (주의: 문자열 길이를 고려해야 합니다!)
4. **출력문**: 각 멤버 변수에 접근하여 학생 정보를 출력합니다. `kim.name`, `kim.age`, `kim.gpa` 처럼 구조체 변수 이름 뒤에 멤버 변수 이름을 붙입니다.

**핵심 포인트:**

- 구조체는 데이터의 논리적인 그룹화를 가능하게 합니다.
- **여러 정보를 하나의 변수로 관리**하여 코드 가독성을 높입니다.
- **동일한 이름 공간 내에서 여러 데이터 타입을 효율적으로 사용**할 수 있습니다.

### ⚡ 공용체: 공간 절약 마법사!

공용체는 구조체와 비슷하지만, **한 공간에 여러 멤버 변수를 공유**하는 특별한 능력을 지녔어요. 마치 여러 개의 그릇이 하나의 큰 그릇에 겹쳐져 있는 것처럼, 메모리 공간을 **매우 효율적으로 사용**할 수 있습니다. 

**예시: 다양한 이미지 포맷 저장**

```c
#include <stdio.h>
#include <stdint.h> // uint8_t 사용을 위한 헤더

// 공용체 선언: 여러 이미지 포맷을 저장할 'ImageData' 공용체 정의
union ImageData {
    uint8_t rgb[3];  // RGB 색상 (각 성분: 0~255)
    uint8_t grayscale; // 그레이스케일 값 (0~255)
    uint16_t palette[8]; // 색상 팔레트 (최대 8개 색상)
};

int main() {
    union ImageData image; // 공용체 변수 'image' 생성

    // RGB 이미지 데이터 저장
    image.rgb[0] = 255; // 빨간색 (빨강)
    image.rgb[1] = 0;   // 초록색 (초록)
    image.rgb[2] = 0;   // 파란색 (파랑)

    // 그레이스케일 값으로 덮어쓰기
    image.grayscale = 128; // 중간 회색

    // RGB 데이터 확인
    printf("Red: %u, Green: %u, Blue: %u\n", image.rgb[0], image.rgb[1], image.rgb[2]);
    printf("Grayscale: %u\n", image.grayscale); // 값이 덮어쓰임을 확인!

    return 0;
}
```

**코드 해설:**

1. **`union ImageData`**: 여러 데이터 타입을 하나의 메모리 공간에 저장할 공용체 'ImageData'를 정의합니다.
2. **`uint8_t rgb[3]`**: RGB 색상 정보를 저장할 배열입니다. 각 색상 성분은 8비트 (0~255) 범위를 가집니다.
3. **`uint8_t grayscale`**: 그레이스케일 값을 저장합니다. 여기서도 8비트 범위를 사용합니다.
4. **메모리 공유**: 공용체 변수 `image`는 위에서 정의한 모든 멤버를 공유하는 동일한 메모리 공간을 사용합니다. 따라서 하나의 멤버를 변경하면 다른 멤버의 값이 덮어쓰여질 수 있습니다.

**핵심 포인트:**

- 공용체는 **메모리 공간 절약**에 탁월합니다. 특히, 다양한 데이터 형식을 효율적으로 저장해야 할 때 유용합니다.
- **주의**: 하나의 멤버 변수를 수정하면 다른 멤버 변수의 값은 **덮어쓰기**됩니다. 데이터 관리에 신중해야 합니다!

### 💡 초보자 폭풍 질문! 💡

**Q1: 구조체와 공용체의 주요 차이점은 무엇인가요?**
**A1:** 구조체는 **여러 데이터 타입을 독립적으로 묶어 관리**하는 반면, 공용체는 **동일한 메모리 공간을 여러 멤버 변수가 공유**합니다. 즉, 구조체는 데이터를 **확장**하고 공용체는 **공간 효율성**을 강조합니다.

**Q2: 공용체를 사용하면 어떤 상황에서 유용할까요?**
**A2:** 메모리 제약이 있는 환경 (예: 임베디드 시스템)이나 다양한 데이터 포맷을 효율적으로 저장해야 할 때 유용합니다. 예를 들어, 이미지 처리에서 다양한 색상 표현 방식을 하나의 변수로 관리할 수 있습니다.

### 🚨 실무주의보 🚨
