### 17강: C언어 활용 마스터하기: 디버깅 기법과 도구 소개

안녕하세요, 코딩의 세계에서 우리와 함께 성장해온 **5년 차 주니어 개발자**입니다! 오늘은 여러분의 디버깅 기술을 한층 더 업그레이드할 수 있는 강좌를 준비했어요. 디버깅은 마치 **영화의 후반 작업 편집**과 같아요. 초반에 잘 찍어놓은 영상이라도 후반 편집 없이는 완성도 높은 작품이 되지 않듯이, 초기 코드에도 탄탄한 디버깅이 필수랍니다! 🤯

#### **디버깅이란 무엇인가요?**

디버깅은 프로그램 내에서 발생하는 오류를 찾아내고 고치는 과정을 말합니다. 이걸 잘하면 코드가 마치 **마법사의 주문**처럼 완벽하게 작동하게 만들 수 있어요! 🧙‍♂️✨

#### **디버깅의 기본 단계**

1. **문제 인식**: "이건 뭐야? 왜 안 돼?" 라고 생각하는 순간이죠. 초보자라면 여기서 **한 숨 크게 쉬어보세요**! 스트레스 해소 후 다시 시작하는 것도 좋은 방법입니다. 😌
   
2. **코드 검토**: 오류가 의심되는 부분을 찾아봐요. 이때 **주석**을 활용하면 정말 도움이 됩니다. 주석은 코드의 **숨은 친구** 같아요. 코드가 무엇을 하는지 쉽게 이해할 수 있게 도와줘요.

3. **디버깅 도구 사용**: 이 부분이 오늘의 **핵심 포인트**입니다! 다양한 도구와 기법을 알아보도록 하죠.

##### **1) GDB (GNU Debugger) 사용법**

GDB는 가장 널리 사용되는 디버깅 도구 중 하나예요. 마치 **코드의 의사**처럼 작동합니다.

**예제 코드: 간단한 반복문 디버깅**

```c
#include <stdio.h>

int main() {
    int i = 0;
    // 반복문에서 문제 발생 시 GDB를 활용
    while (i < 5) {  // 의도적으로 조건 수정 (i > 5로 변경)
        printf("현재 값: %d\n", i);  // 코드가 멈추는 지점 설정
        i++;  // 의도적으로 오류를 유발하는 코드 수정 부분
        if (i == 3) {
            printf("중단점 도달!\n");
            // GDB 명령어 실행: breakpoint 설정
            // 컴파일 후 실행: ./your_program
            // GDB 내에서: break 6  // 6번째 줄에서 중단
            // 실행: run
        }
    }
    return 0;
}
```

**코드 설명**:
- **breakpoint 설정**: `break 6`으로 6번째 줄에서 코드 실행이 멈춥니다. 여기서 변수 값을 확인하거나 코드 흐름을 분석할 수 있어요.
- **변수 확인**: `print $i` 명령어로 `i` 변수의 현재 값을 확인합니다.

**실무주의보**: 실제 환경에서는 프로젝트에 맞는 적절한 중단점을 설정해야 합니다. 너무 많은 중단점은 디버깅을 복잡하게 만들 수 있으니 주의하세요!

##### **2) 비주얼 스튜디오 코드 (VS Code) 활용**

VS Code는 비주얼 스튜디오의 가벼운 버전으로, 강력한 디버깅 기능을 제공합니다.

**예제 코드: 조건문 디버깅**

```c
#include <stdio.h>

int main() {
    int num = 10;
    // if 문에서 오류 찾기
    if (num > 15) {  // 의도적으로 조건 수정 (num < 5로 변경)
        printf("큰 수!\n");
    } else {
        printf("작은 수!\n");
    }
    return 0;
}
```

**코드 설명**:
- **디버깅 시작**: VS Code에서 코드를 실행할 때 `F5`를 누르세요. 디버거가 활성화되면 변수 값을 실시간으로 확인할 수 있어요.
- **브레이크포인트 설정**: 코드 라인에 마우스 오른쪽 버튼을 클릭하고 `조건부 브레이크포인트 설정`을 통해 특정 조건 하에만 중단되도록 할 수 있습니다.

**💡 초보자 폭풍 질문!**
- **질문**: 만약 코드가 예상대로 동작하지 않을 때 어떻게 해야 할까요?
  - **답변**: 첫째, **코드를 천천히 읽어보세요**. 둘째, **주석을 활용해 코드의 역할을 명확히 이해**하세요. 셋째, **디버깅 도구를 적극 활용**하여 변수 값과 실행 흐름을 확인해보세요.

##### **3) 간단한 로그 활용**

디버깅 도구가 없는 환경에서도 로그를 활용하면 큰 도움이 됩니다. 간단한 `printf` 함수로 문제를 찾아봐요!

**예제 코드: 반복문과 로그 활용**

```c
#include <stdio.h>

int main() {
    int count = 0;
    // 반복문 내에서 로그 출력
    for (count = 0; count < 5; count++) {
        printf("반복 횟수: %d\n", count);  // 각 반복마다 상태 확인
        if (count == 2) {
            printf("특정 조건 도달!\n");  // 특정 조건에서 로그 출력
        }
    }
    return 0;
}
```

**코드 설명**:
- **로그 출력**: 각 반복마다 현재 상태를 출력하여 흐름을 추적합니다. 특정 조건에서 로그를 출력하면 문제 발생 지점을 더 쉽게 파악할 수 있어요.

**🚨 실무주의보**: 과도한 로그는 코드 성능에 영향을 줄 수 있으니 필요한 부분에만 사용하세요!
