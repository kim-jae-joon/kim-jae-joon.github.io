### 7강: C언어 심화 - 동적 메모리 할당의 마법: `malloc`, `free`로 데이터를 자유롭게!

안녕하세요, 코딩의 마법사들! 오늘은 여러분의 C언어 실력을 한층 더 높여줄 특별한 마법을 알려드리려고 합니다. 이름은 바로 **동적 메모리 할당**이에요. 이 기술은 마치 마법의 주문처럼 메모리를 즉석에서 만들어내고 정리하는 능력을 부여해줍니다. 이 강의를 마치고 나면, 메모리 관리의 고수로 변신할 수 있을 거예요! 🪄

#### 🧙‍♂️ 동적 메모리 할당이란 무엇인가요?

**동적 메모리 할당**은 프로그램 실행 중에 필요한 메모리 공간을 자동으로 할당하고 해제하는 기술입니다. 이는 정적 메모리 할당과 달리 컴파일 시점이 아닌 런타임 시점에서 이루어집니다. 왜 이렇게 복잡한 것을 쓰냐고요? 간단해요! **크기가 미리 정해지지 않은 데이터 구조**를 다룰 때 매우 유용하거든요. 예를 들어, 사용자 입력에 따라 크기가 달라지는 리스트나 배열을 다룰 때 딱 맞는 도구죠!

**핵심 명령어**:
- **`malloc`**: 메모리 할당
- **`free`**: 메모리 해제

#### 💡 개념 이해를 위한 비유

상상해보세요, 당신이 파티를 주최하고 있습니다. 손님이 도착할 때마다 테이블과 의자를 즉석에서 만들어야 해요. 만약 모든 테이블과 의자를 미리 준비했다면 공간이 턱없이 부족할 거예요. 하지만 **`malloc`**은 마치 마법의 테이블과 의자를 즉석에서 생성하는 마법사 같아요! 손님이 떠나면 (**`free`**), 그 자리를 다시 정리해 새로운 손님을 맞이할 수 있게 돕습니다.

#### 📖 코드로 알아보기

##### 1. 기본 메모리 할당과 해제 예제

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    // 1. 메모리 할당 (예: 정수 하나를 위한 메모리)
    int *ptr = (int *)malloc(sizeof(int));

    // 할당이 실패하면 NULL을 반환합니다! 확인 필수!
    if (ptr == NULL) {
        printf("메모리 할당 실패!\n");
        return 1; // 에러 처리
    }

    // 2. 메모리에 값 할당
    *ptr = 42; // 포인터를 통해 값을 넣습니다
    printf("할당된 메모리 값: %d\n", *ptr); // 출력: 42

    // 3. 메모리 해제
    free(ptr); // 마법의 해제 주문!

    return 0;
}
```
**코드 설명**:
- **`malloc(sizeof(int))`**: 정수 하나를 위한 메모리를 할당합니다. `sizeof(int)`는 정수 타입의 크기를 바이트 단위로 반환합니다.
- **`ptr == NULL` 체크**: 메모리 할당이 실패했는지 확인합니다. 실패하면 프로그램이 중단됩니다.
- **`*ptr = 42`**: 포인터를 통해 메모리에 값을 할당합니다.
- **`free(ptr)`**: 할당된 메모리를 해제하여 메모리 누수를 방지합니다.

##### 2. 동적 배열 생성 예제

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int n;
    printf("배열 크기를 입력하세요: ");
    scanf("%d", &n); // 사용자 입력 받기

    // 1. 동적 메모리 할당 (배열 크기만큼)
    int *arr = (int *)malloc(n * sizeof(int));

    // 할당 실패 확인
    if (arr == NULL) {
        printf("메모리 할당 실패!\n");
        return 1;
    }

    // 2. 배열 초기화
    for (int i = 0; i < n; i++) {
        arr[i] = i * 10; // 예시 값 할당
    }

    // 3. 배열 출력
    printf("배열 내용:\n");
    for (int i = 0; i < n; i++) {
        printf("%d ", arr[i]);
    }
    printf("\n");

    // 4. 메모리 해제
    free(arr);

    return 0;
}
```
**코드 설명**:
- **`malloc(n * sizeof(int))`**: 사용자 입력 `n`에 따라 동적으로 배열을 생성합니다.
- **`for` 루프**: 배열을 초기화하고 값을 설정합니다.
- **출력**: 배열의 내용을 화면에 출력합니다.
- **`free(arr)`**: 메모리 해제로 자원을 정리합니다.

##### 3. 다양한 조건 하의 메모리 관리

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int choice;
    printf("1: 큰 배열 할당\n");
    printf("2: 작은 배열 할당\n");
    printf("선택하세요: ");
    scanf("%d", &choice);

    int *dynamicArray;

    if (choice == 1) {
        // 큰 배열 할당 예시
        dynamicArray = (int *)malloc(100 * sizeof(int)); // 100개의 정수
        if (dynamicArray == NULL) {
            printf("큰 배열 할당 실패!\n");
            return 1;
        }
        for (int i = 0; i < 100; i++) {
            dynamicArray[i] = i * 5; // 예시 값 할당
        }
        printf("큰 배열 내용:\n");
        for (int i = 0; i < 100; i++) {
            printf("%d ", dynamicArray[i]);
        }
        printf("\n");
        free(dynamicArray);
    } else if (choice == 2) {
        // 작은 배열 할당 예시
        dynamicArray = (int *)malloc(10 * sizeof(int)); // 10개의 정수
        if (dynamicArray == NULL) {
            printf("작은 배열 할당 실패!\n");
            return 1;
        }
        for (int i = 0; i < 10; i++) {
            dynamicArray[i] = i * 2; // 예시 값 할당
        }
        printf("작은 배열 내용:\n");
        for (int i = 0; i < 10; i++) {
            printf("%d ", dynamicArray[i]);
        }
        printf("\n");
        free(dynamicArray);
    } else {
        printf("잘못된 선택!\n");
    }

    return 0;
}
```
**코드 설명**:
- **조건문 사용**: 사용자의 선택에 따라 다른 크기의 배열을 동적으로 할당합니다.
- **다양한 크기의 할당**: `malloc`을 통해 다양한 크기의 메모리를 할당하고 해제합니다.
- **메모리 누수 방지**: 각 할당 후 `free`를 통해 메모리를 정리합니다.

#### 🚨 실무주의보 🚨

**메모리 누수 주의**: 가장 흔한 실수 중 하나는 할당된 메모리를 해제하지 않는 것입니다. 메모리 누수가 발생하면 프로그램이 점차적으로 느려지고, 심각한 경우에는 크래시를 일으킬 수 있어요. 항상 **`free`**를 통해 할당된 메모리를 정리해주세요!

**💡 초보자 폭풍 질문!**

- **Q**: 메모리 할당이 실패하면 어떻게 해야 하나요?
  - **A**: `malloc`이 실패하면 반환값이 `NULL`이 됩니다. 이 경우 즉시 오류 메시지를 출력하고 프로그램을 종료하거나 다른 대처 방안을 찾아야 합니다.
  
- **Q**: 동적으로 할당한 메모리를 언제 해제해야 하나요?
  - **A**: 더 이상 사용하지 않는 순간 즉시 해제해야 합니다. 특히 루프나 함수 종료 시 반드시 `free`를 호출해 메모리 누수를 방지하세요.
