# 2강: 기본 설정 및 환경 구축 - C언어 세계로의 설렘 가득 여정

안녕하세요, 초보 개발자 여러분! 💪 오늘은 C언어의 아름다운 세계로 들어가는 문을 함께 열어보도록 하죠. 마치 마법의 문을 열면 다른 차원으로 이동하는 것처럼, 지금부터 여러분은 코딩의 마법 세계로 들어서게 될 거예요! 🌌✨

## 1. 환경 구축: 마법의 발판을 마련하기

### 1.1 설치 전 준비물 체크리스트

**"이거 모르면 큰일 납니다!"**  
시작하기 전에 꼭 체크해야 할 몇 가지가 있어요. 마치 캠핑을 가기 전에 텐트와 캠핑 의자를 챙기는 것처럼요!

- **운영 체제**: Windows, macOS, Linux 중 어떤 것을 사용하고 계신가요? 각각의 환경에 맞는 컴파일러가 필요해요.
- **컴파일러**: GCC (GNU Compiler Collection) 또는 Clang을 추천해요. 이들은 우리 코드를 컴퓨터가 이해할 수 있는 기계어로 바꿔주는 마법사들이랍니다.
- **텍스트 편집기**: 코드를 작성할 공간이 필요하죠! VS Code, Sublime Text, 또는 심지어는 메모장도 괜찮아요. 하지만 추천은 **Code::Blocks**나 **Dev-C++** 같은 IDE(Integrated Development Environment)를 활용하는 거예요. IDE는 코드 작성부터 컴파일까지 한 곳에서 해결할 수 있는 마법의 도구랍니다.

### 1.2 설치 과정 가이드

#### macOS 및 Linux 사용자를 위한 GCC 설치 가이드

> 터미널을 열고 다음 명령어를 입력해보세요:

```bash
# macOS (Homebrew 사용 시)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
brew install gcc

# Linux (Debian/Ubuntu 기반)
sudo apt update
sudo apt install gcc g++
```

**왜 이렇게 했는지 궁금하시죠?**  
- **터미널**은 컴퓨터와 대화할 수 있는 창문이에요. 명령어를 통해 컴퓨터에게 "GCC 컴파일러 설치해줘"라고 말하는 거죠!
- **brew**와 **apt**는 소프트웨어 설치를 도와주는 마법의 주문서 같은 존재입니다.

#### Windows 사용자를 위한 설치 가이드 (Code::Blocks 예시)

1. **Code::Blocks 다운로드**: [공식 웹사이트](https://www.codeblocks.org/downloads/)에서 최신 버전을 다운로드하세요.
2. **설치**: 설치 프로세스 중에 기본 설정을 선택하고 완료!

**코드::Blocks**는 마치 코딩 마법사의 지팡이 같아요. 코드를 작성하고 컴파일하며 디버깅까지 한 번에 해결해주는 마법사의 도구랍니다.

## 2. 기본 설정: 마법의 기초 다지기

### 2.1 첫 번째 프로그램 작성하기

#### 예제 1: "Hello, World!" 출력하기 (for문 활용 예시)

```c
#include <stdio.h>  // 표준 입출력 라이브러리 포함

int main() {
    for (int i = 0; i < 5; i++) {  // for문으로 5번 반복
        printf("Hello, World!\n");  // 문자열 출력
    }  // 반복문 종료
    return 0;  // 프로그램 종료 코드
}
```

**설명**:
- `#include <stdio.h>`: 이 줄은 마법의 도서관에서 필요한 책을 가져오는 것과 같아요. `stdio.h`는 입력과 출력을 위한 함수들을 제공합니다.
- `for (int i = 0; i < 5; i++)`: 이건 마법의 반복 마법진이에요! `i`가 0에서 시작해 5 미만일 때까지 계속 돌면서 "Hello, World!"를 다섯 번 출력합니다.
- `printf("Hello, World!\n");`: 마법의 주문으로 화면에 메시지를 출력합니다. `\n`은 새 줄을 의미해요.
- `return 0;`: 마법의 세계에서 성공적으로 임무를 수행했다는 신호를 보내는 거예요.

#### 예제 2: 사용자 입력 받기 (while문 활용 예시)

```c
#include <stdio.h>

int main() {
    char name[50];  // 이름을 저장할 공간
    printf("이름을 입력하세요: ");  // 사용자에게 입력 요청
    scanf("%49s", name);  // 이름 입력 (공백 포함 49글자까지)
    printf("안녕하세요, %s님!\n", name);  // 이름 출력
    return 0;
}
```

**설명**:
- `char name[50];`: 사용자의 이름을 저장할 공간을 준비합니다.
- `scanf("%49s", name);`: 사용자로부터 입력을 받아 `name` 변수에 저장합니다. `%49s`는 최대 49글자까지 입력받도록 제한합니다 (공백 포함).
- `printf("안녕하세요, %s님!\n", name);`: 입력받은 이름을 이용해 환영 메시지를 출력합니다.

### 2.2 다양한 반복문 활용하기

#### while문 예시: 무한 루프에서 벗어나기

```c
#include <stdio.h>

int main() {
    int counter = 0;
    while (counter < 10) {  // 카운터가 10 미만일 때까지 반복
        printf("카운터: %d\n", counter);
        counter++;  // 카운터 증가
    }  // 루프 종료 조건 만족 시 종료
    return 0;
}
```

**설명**:
- `while (counter < 10)`: 조건이 참일 때만 루프가 돌아가요. 여기서는 카운터가 10 미만일 때까지 계속 반복합니다.
- `counter++`: 각 반복마다 카운터를 하나씩 증가시켜 무한 루프에서 벗어나게 합니다.

#### do-while문 예시: 무조건 한 번은 실행

```c
#include <stdio.h>

int main() {
    int choice;
    do {
        printf("선택해보세요 (1 또는 2): ");
        scanf("%d", &choice);  // 사용자 입력 받기
        if (choice == 1) {
            printf("선택 1 선택!\n");
        } else {
            printf("선택 2 선택!\n");
        }
    } while (choice != 3);  // 선택이 3이 될 때까지 반복
    return 0;
}
```

**설명**:
- `do-while`은 최소한 한 번은 실행되는 루프예요. 조건이 참일 때만 계속 반복합니다.
- 여기서는 사용자가 3을 선택할 때까지 계속 반복해서 다른 선택지에 대한 메시지를 출력합니다.

## 💡 초보자 폭풍 질문! 💡

**질문**: 컴파일러가 왜 필요한가요?
- **답변**: 컴파일러는 우리가 작성한 인간이 이해할 수 있는 코드를 컴퓨터가 이해할 수 있는 기계어로 변환해주는 마법사 같아요. 마치 요리 레시피를 따라 맛있는 음식을 만드는 것처럼, 코드를 입력하면 컴퓨터가 실행할 수 있는 실행 파일을 만들어주는 거죠!

**질문**: IDE와 텍스트 편집기의 차이점은 무엇인가요?
- **답변**: IDE는 마법의 스튜디오 같아요. 코드 작성부터 컴파일, 디버깅까지 모든 과정을 한 공간에서 해결할 수 있어요. 반면 텍스트 편집기는 주로 코드 작성에 집중하며, 별도의 컴파일 과정이 필요해요. IDE는 초보자에게 더 편리하지만, 전문가들은 종종 편집기의 유연성을 선호하기도 해요.
