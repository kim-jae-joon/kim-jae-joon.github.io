## 유닛 테스트의 기본 개념

### 1. 무엇이 유닛 테스트인가요?
유닛 테스트는 개별 코드 유닛(함수, 메소드 등)이 예상대로 동작하는지 자동으로 확인하는 프로세스입니다. 이를 통해 개발자는 코드 변경 시 기존 기능에 영향이 없는지 쉽게 검증할 수 있어요.

**비유:** 유닛 테스트는 요리 레시피의 한 단계를 체크하는 것과 같아요. 만약 한 단계에서 문제가 생기면 전체 요리의 맛이 변질되기 전에 그 부분만 고칠 수 있죠!

### 2. 왜 유닛 테스트를 해야 할까요?
- **버그 조기 발견:** 작은 오류를 빠르게 잡아낼 수 있어요.
- **코드 유지보수 용이성:** 변경 사항을 적용할 때 안전성을 높여줍니다.
- **문서화 역할:** 코드의 의도와 기대 동작을 명확하게 보여줍니다.

## 실전 유닛 테스트 작성 가이드

### 예제 1: 간단한 덧셈 함수 테스트

```c
#include <stdio.h>
#include <assert.h> // assert 라이브러리는 테스트 프레임워크 중 하나

// 덧셈 함수 정의
int add(int a, int b) {
    return a + b;
}

// 유닛 테스트 함수
void test_add() {
    // 테스트 케이스 1: 기본 덧셈
    assert(add(2, 3) == 5); // 만약 실패하면 프로그램이 중단되고 오류 메시지를 출력합니다.
    printf("테스트 케이스 1 통과: 2 + 3 = 5\n");

    // 테스트 케이스 2: 음수 덧셈
    assert(add(-1, 1) == 0); // 음수와 양수의 덧셈도 확인
    printf("테스트 케이스 2 통과: -1 + 1 = 0\n");

    // 테스트 케이스 3: 큰 수 덧셈
    assert(add(1000, 2000) == 3000); // 큰 수에 대한 검증
    printf("테스트 케이스 3 통과: 1000 + 2000 = 3000\n");
}

int main() {
    test_add(); // 테스트 함수 호출
    return 0;
}
```

**코드 설명:**
- **assert() 함수:** 조건이 맞지 않으면 프로그램을 중단하고 오류 메시지를 출력합니다. 이는 테스트 실패 시 즉시 알림을 주는 역할을 합니다.
- **printf() 함수:** 성공 메시지를 출력하여 개발자가 어떤 테스트가 통과되었는지 쉽게 파악할 수 있게 합니다.

### 예제 2: 조건문 기반 테스트 (if-else 활용)

```c
#include <stdio.h>
#include <assert.h>

// 조건에 따라 값을 반환하는 함수
int conditionalValue(int x) {
    if (x > 0) {
        return x * 2;
    } else {
        return x * -1;
    }
}

// 유닛 테스트 함수
void test_conditionalValue() {
    // 테스트 케이스 1: 양수 입력
    assert(conditionalValue(5) == 10); // 양수일 때 2배로 반환 확인
    printf("테스트 케이스 1 통과: 양수 입력 시 2배\n");

    // 테스트 케이스 2: 음수 입력
    assert(conditionalValue(-3) == 3); // 음수일 때 음수 그대로 반환 확인
    printf("테스트 케이스 2 통과: 음수 입력 시 그대로 반환\n");
}

int main() {
    test_conditionalValue(); // 테스트 함수 호출
    return 0;
}
```

**코드 설명:**
- **if-else 문:** 다양한 조건에 따라 다른 동작을 수행하는 함수를 테스트합니다. 각 조건에 맞는 결과를 확인하여 함수의 정확성을 검증합니다.

### 예제 3: 반복문을 활용한 테스트 케이스 생성

```c
#include <stdio.h>
#include <assert.h>

// 리스트의 합을 계산하는 함수
int sumArray(int arr[], int size) {
    int total = 0;
    for (int i = 0; i < size; i++) {
        total += arr[i];
    }
    return total;
}

// 유닛 테스트 함수
void test_sumArray() {
    int testArray1[] = {1, 2, 3, 4}; // 테스트 배열 1
    int testArray2[] = {-1, 0, 1};   // 테스트 배열 2 (음수 포함)

    // 테스트 케이스 1: 기본 배열 합
    assert(sumArray(testArray1, 4) == 10); // 배열 합이 예상대로인지 확인
    printf("테스트 케이스 1 통과: 배열 합 검증 완료\n");

    // 테스트 케이스 2: 음수 포함 배열 합
    assert(sumArray(testArray2, 3) == 0); // 음수 포함 배열 합 검증
    printf("테스트 케이스 2 통과: 음수 포함 배열 합 검증 완료\n");
}

int main() {
    test_sumArray(); // 테스트 함수 호출
    return 0;
}
```

**코드 설명:**
- **for 문:** 배열의 요소들을 반복적으로 합산하여 결과를 검증합니다. 다양한 데이터셋을 활용해 함수의 견고성을 테스트합니다.

### 예제 4: 조건문과 반복문 혼합 테스트

```c
#include <stdio.h>
#include <assert.h>

// 특정 조건을 만족하는 숫자만 필터링하여 합을 반환하는 함수
int filterAndSum(int arr[], int size, int threshold) {
    int total = 0;
    for (int i = 0; i < size; i++) {
        if (arr[i] > threshold) {
            total += arr[i];
        }
    }
    return total;
}

// 유닛 테스트 함수
void test_filterAndSum() {
    int testArray[] = {5, 10, 3, 15, 7}; // 테스트 배열
    int threshold1 = 5; // 임계값 1
    int threshold2 = 10; // 임계값 2

    // 테스트 케이스 1: 임계값 5 이상의 합
    assert(filterAndSum(testArray, 5, threshold1) == 30); // 예상 합 검증
    printf("테스트 케이스 1 통과: 임계값 5 이상 합 검증 완료\n");

    // 테스트 케이스 2: 임계값 10 이상의 합
    assert(filterAndSum(testArray, 5, threshold2) == 15); // 예상 합 검증
    printf("테스트 케이스 2 통과: 임계값 10 이상 합 검증 완료\n");
}

int main() {
    test_filterAndSum(); // 테스트 함수 호출
    return 0;
}
```

**코드 설명:**
- **for 문과 if 문 결합:** 특정 조건을 만족하는 숫자만 필터링하여 합산하는 로직을 테스트합니다. 다양한 임계값에 대해 함수의 정확성을 검증합니다.
