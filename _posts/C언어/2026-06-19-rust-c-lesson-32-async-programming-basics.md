## 🧙‍♂️ 비동기 프로그래밍이란 무엇인가요?

비동기 프로그래밍은 프로그램이 하나의 작업이 완료될 때까지 마냥 기다리지 않고, 다른 작업을 동시에 수행할 수 있게 하는 방법입니다. 쉽게 말해, **요리할 때 친구에게 전화해서 이야기하면서 동시에 밥을 짓는 것**과 비슷해요. 밥 짓는 걸 기다리는 동안에도 통화를 즐길 수 있잖아요!

### 개념 이해하기: 시간 여행자가 되어보자

#### 1. **동기 코드 예시**
```rust
fn main() {
    // 동기 코드 예시: 한 작업이 끝나야 다음 작업을 시작
    println!("시작합니다!");
    let sum = calculate_sum(5, 10); // 여기서 기다림
    println!("합계: {}", sum);
}

fn calculate_sum(a: i32, b: i32) -> i32 {
    // 간단한 덧셈
    a + b
}
```
- **설명**: 위 코드는 동기적입니다. `calculate_sum` 함수가 완료될 때까지 `println!`이 대기합니다. 마치 한 가지 요리가 완성될 때까지 다른 일을 미루는 것과 같아요.

#### 2. **비동기 코드 예시 (async/await 사용)**
```rust
use std::time::Duration;
use async_std::task;

#[async_std::main]
async fn main() {
    println!("시작합니다!");

    // 비동기 함수 호출
    let sum = async {
        task::sleep(Duration::from_millis(500)).await; // 500ms 기다림
        calculate_sum(5, 10) // 기다리는 동안 다른 작업 가능
    }.await;

    println!("합계: {}", sum);
}

fn calculate_sum(a: i32, b: i32) -> i32 {
    a + b // 단순 덧셈 함수
}
```
- **설명**: `async` 키워드를 사용해 비동기 함수를 정의하고, `await`로 작업이 완료될 때까지 기다리지 않고 다음 코드를 실행할 수 있어요. 여기서 `task::sleep`은 단순히 대기 시간을 시뮬레이션하는데 사용됩니다. **비유**: 요리하면서 동시에 스마트폰으로 레시피를 검색하는 것 같죠!

### 다양한 비동기 구조 살펴보기

#### **반복문 예시**
반복문을 비동기적으로 사용하면 각 반복마다 다른 작업을 수행할 수 있어요.

##### `for` 루프 비동기 버전
```rust
use async_std::task;

#[async_std::main]
async fn loop_example() {
    println!("시작!");
    
    for i in 0..5 {
        println!("반복: {}", i);
        task::sleep(Duration::from_millis(500)).await; // 짧은 대기 시간
        // 각 반복마다 다른 작업 수행 가능
    }
    println!("끝!");
}
```
- **설명**: `for` 루프 내에서 각 반복마다 `task::sleep`을 호출해 짧은 대기 시간을 두고 다른 작업을 수행할 수 있습니다. **비유**: 카페에서 커피를 마시면서 동시에 이메일 확인하는 것 같아요!

##### `while` 루프 비동기 버전
```rust
use async_std::task;

#[async_std::main]
async fn while_loop_example() {
    let mut counter = 0;
    println!("시작!");
    
    while counter < 5 {
        println!("반복: {}", counter);
        task::sleep(Duration::from_millis(500)).await; // 대기 시간
        counter += 1;
    }
    println!("끝!");
}
```
- **설명**: `while` 루프를 사용해 조건에 따라 반복하면서 비동기적으로 작업을 수행합니다. **비유**: 일정 시간마다 알람을 설정해 두고 그 시간마다 다른 작업을 처리하는 것 같아요.
