**24강: 함수 포인터 및 구조체 포인터** 🔥

안녕하세요, 여러분! 오늘은 C언어의 기초 문법 중 하나인 함수 포인터와 구조체 포인터에 대해 배워보겠습니다. 이 두 가지概念을 이해하면 프로그램 개발에서 매우 유용한 도구가 될 것입니다.

### 함수 포인터란?

함수 포인터는 다른 함수를 저장할 수 있는 변수입니다. 함수 포인터를 사용하면, 동일한 동작을 하는 여러 함수에 대한 참조를 가질 수 있습니다. 이는 프로그램의 유연성과 효율성을 높여주는 강력한 도구입니다.

**예제 1: 단순한 함수 포인터**

```c
#include <stdio.h>

void sayHello() {
    printf("안녕하세요!\n");
}

int main() {
    void (*funcPtr)(); // 함수 포인터 선언

    funcPtr = sayHello; // 함수 포인터에 sayHello 함수 할당

    (*funcPtr)(); // 함수 포인터를 통해 sayHello 함수 호출
    return 0;
}
```

*   `void (*funcPtr)()` : 이 줄은 함수 포인터를 선언하는 것입니다. `(*`와 `)` 사이의 내용이 함수 포인터에 할당될 함수의 프로토타입입니다.
*   `funcPtr = sayHello;` : 이 줄에서 함수 포인터 `funcPtr` 에 `sayHello` 함수가 할당됩니다. `funcPtr`는 이제 `sayHello` 함수와 동등한 위치를 차지합니다.
*   `(*funcPtr)();` : 이 줄에서 `funcPtr`에 저장된 함수 (`sayHello`)을 호출합니다.

### 구조체 포인터란?

구조체 포인터는 다른 구조체의 주소를 저장할 수 있는 변수입니다. 구조체 포인터를 사용하면, 동일한 동작을 하는 여러 프로그램 데이터의 주소를 가질 수 있습니다. 이는 프로그램의 유연성과 효율성을 높여주는 강력한 도구입니다.

**예제 2: 단순한 구조체**

```c
#include <stdio.h>

typedef struct {
    int x;
    int y;
} Point;

int main() {
    Point *p; // 구조체 포인터 선언

    p = malloc(sizeof(Point)); // 구조체 포인터에 구조체 할당

    p->x = 10;
    p->y = 20;

    printf("%d, %d\n", p->x, p->y); // 구조체 포인터를 통해 구조체 내용 출력
    free(p);
    return 0;
}
```

*   `typedef struct { ... } Point;` : 이 줄은 구조체 `Point`를 선언하는 것입니다. 구조체의 멤버 변수는 `x`, `y`로 정의됩니다.
*   `Point *p;` : 이 줄에서 구조체 포인터 `p`를 선언합니다. `p`에는 다른 구조체의 주소가 저장될 수 있습니다.
*   `p = malloc(sizeof(Point));` : 이 줄에서 메모리에서 `sizeof(Point)` 크기의 공간을 할당받고, 그 주소를 `p`에 저장합니다.
*   `p->x`, `p->y` : 이 줄에서 `p`가 가리키는 구조체의 멤버 변수에 접근할 수 있습니다. `.`은 구조체 멤버에 접근하는 연산자입니다.

### 초보자 폭풍 질문!

여러분, 지금까지 함수 포인터와 구조체 포인터를 사용하여 프로그램을 작성할 수 있게 되었습니다. 이제는 함수 포인터와 구조체 포인터의 차이점에 대해 물어볼까요? 😊

함수 포인터는 다른 함수를 저장하는 데 쓰이며, 구조체 포인터는 다른 구조체의 주소를 저장합니다. 하지만, 프로그램에서 이것들은 어떻게 활용될까요?

함수 포인터는 프로그램을 더 유연하게 만들 수 있습니다. 예를 들어, 다양한 함수에 대한 참조를 하나의 변수에 할당할 수 있습니다.

구조체 포인터도 마찬가지로 다양한 프로그램 데이터의 주소를 저장할 수 있으므로, 프로그램의 효율성을 높일 수 있습니다.

### 실무주의보

함수 포인터와 구조체 포인터는 프로그램에서 매우 유용한 도구입니다. 하지만, 이것들을 사용할 때에는 조심해야 합니다. 🚨

함수 포인터를 잘못 사용하면, 프로그램이 충돌하거나 오류가 발생할 수 있습니다.

구조체 포인터도 마찬가지로 잘못 사용하면, 프로그램의 효율성이 떨어지거나 오류가 발생할 수 있습니다.

이러한 위험을 피하기 위해, 함수 포인터와 구조체 포인터를 잘 이해하고 사용하는 것이 중요합니다. 💡
