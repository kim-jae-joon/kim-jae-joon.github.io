### 🤔  배열이란 무엇일까요?

> 배열은 같은 종류의 데이터를 순서대로 저장하고 관리하는 곳입니다.

마치 책장에 책을 정리하듯, 배열은 하나 이상의 값들을 **특정 위치**에서 
저장할 수 있도록 합니다!  위치는 각각 **인덱스**로 불리는 번호로 나타내죠. 인덱스는 일반적으로 0부터 시작합니다.

예를 들어, 이름을 저장하는 배열이 있다고 해보세요.

```c
char names[5][10]; // "names"라는 배열은 5개의 문자열을 저장할 수 있습니다!
                 // 각 문자열은 최대 10글자까지 저장 가능합니다!
names[0] = "철수";  
names[1] = "영희";   
names[2] = "민수";    
names[3] = "선미";   
names[4] = "지훈";  
```

위 코드처럼 배열을 선언하고, 각 인덱스에 이름을 저장합니다. 이제 `names[0]` 에 접근하면 "철수"라는 문자열을 얻을 수 있습니다. 똑같이 `names[3]` 에 접근하면 "선미"를 얻을 수 있죠!

###  ⚡️ 배열의 사용법 - 실전 코딩 사례!

**1. 점수 관리 프로그램:**

```c
int scores[5] = {85, 92, 78, 95, 88}; // 5명 학생들의 점수를 저장!
printf("학생 %d의 점수는 %d점 입니다.\n", 0, scores[0]); // 첫 번째 학생의 점수 출력
```

**2. 주식 가격 변동 프로그램:**

```c
float stock_prices[7] = {150.25, 148.50, 152.10, 155.75, 153.90, 156.50, 154.80}; // 주식 가격을 일주일 동안 저장!
printf("이번 주 토요일의 주식 가격은 %.2f 입니다.\n", stock_prices[6]); // 토요일 주식 가격 출력
```

**💡 초보자 폭풍 질문! **

> 배열의 크기를 정할 때 어떤 것을 고려해야 할까요? 🤔

*배열의 크기는 저장할 데이터의 양을 예상해서 결정하는 것이 중요합니다.* 너무 작은 배열이면 데이터를 모두 담지 못하고, 너무 큰 배열은 메모리 사용량을 낭비하게 될 수 있습니다!


###  🚨 실무주의보: 바깥 세계는 항상 변화한다!

배열의 크기를 정하는 것이 중요하다는 것을 알고 계신가요? 그 이유는 다음과 같습니다. 배열은 **정해진 크기**로 생성되기 때문에, 저장할 데이터 양이 예상보다 크거나 작을 경우 문제가 발생할 수 있습니다! ⚠️

* **"메모리 오버플로우":** 너무 적은 배열 공간으로 많은 데이터를 저장하려고 하면 메모리를 초과하고 프로그램이 멈추는 현상이 일어나게 되죠.
* **"미사용 메모리":** 너무 큰 배열을 만들면 필요하지 않은 메모리가 사용되어, 시스템 성능이 저하될 수 있습니다!
