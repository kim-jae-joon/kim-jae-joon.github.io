# 13강: 예외 처리: 에러 코드와 검사 - 코드의 보호자, 당신의 개발자 동반자

안녕하세요, 코딩의 세계에서 조금씩 성장해가는 우리의 멋진 팀! 오늘은 예외 처리에 대해 이야기해볼게요. 예외 처리는 마치 코드의 보호자처럼 작동하며, 프로그램이 예상치 못한 상황에 처했을 때 안전하게 빠져나오는 방법을 알려주는 중요한 기술입니다. 이 강의를 마치면, 에러 코드와 검사에 대해 "이거 모르면 큰일 납니다!"라고 할 수 있을 거예요. 그럼 출발해볼까요?

## 💡 기초 개념: 예외 처리란 무엇인가?

**예외 처리**란 프로그램이 오류나 예상치 못한 상황에 직면했을 때 그 상황을 부드럽게 관리하고 애플리케이션이 완전 중지되지 않도록 하는 기법입니다. 쉽게 말해, 프로그램이 마치 갑작스러운 비에 직면한 여행자처럼 "우천 경보! 잠깐 쉬어가자!"라고 말하는 것과 같아요.

### 에러 코드의 역할

에러 코드는 특정 상황이 발생했을 때 그 상황을 숫자나 특수한 값으로 표현해줍니다. 이 코드들은 개발자가 문제의 본질을 파악하고 해결하는 데 도움을 줍니다. 예를 들어, 파일을 열 때 파일이 존재하지 않으면 에러 코드 `ENOENT`를 반환할 수 있어요.

### 예외 처리의 중요성

- **안정성 향상**: 프로그램이 예기치 않게 종료되는 것을 방지합니다.
- **사용자 경험 개선**: 에러 메시지를 통해 사용자에게 명확한 피드백을 제공합니다.
- **디버깅 용이성**: 문제 발생 시 정확한 위치와 원인을 쉽게 파악할 수 있습니다.

## 💡 실용적인 예제: 다양한 예외 처리 기법

### 1. **파일 입출력 예외 처리**

파일을 읽거나 쓸 때 흔히 발생하는 문제를 해결해볼게요. 여기서는 `fopen`, `fclose`, `fread` 등의 함수를 사용해보겠습니다.

```c
#include <stdio.h>
#include <errno.h>

int main() {
    FILE *file;
    char buffer[100];

    // 파일 열기 시도
    file = fopen("nonexistentfile.txt", "r"); // "nonexistentfile.txt"는 존재하지 않는 파일 이름
    if (file == NULL) {
        // 에러 처리: 파일이 열리지 않았을 때
        perror("파일 열기 실패"); // 에러 메시지 출력
        printf("에러 코드: %d\n", errno); // errno로 에러 원인 확인
    } else {
        // 파일 읽기 성공 시 코드 실행
        fread(buffer, sizeof(char), sizeof(buffer), file);
        printf("파일 내용: %s", buffer);
        
        // 파일 닫기
        fclose(file);
    }
    return 0;
}
```

**코드 분석:**
- `fopen`이 `NULL`을 반환하면 파일 열기 실패입니다.
- `perror` 함수는 시스템 에러 메시지를 출력하고, `errno`는 구체적인 에러 코드를 제공합니다.

### 2. **입력 검증: 사용자 입력 예외 처리**

사용자 입력은 특히 보안이나 데이터 무결성 측면에서 중요합니다. 예를 들어, 사용자로부터 정수를 입력받을 때 유효한 숫자인지 확인해보겠습니다.

```c
#include <stdio.h>
#include <ctype.h>

int main() {
    char input[50];
    int number;

    printf("정수를 입력하세요: ");
    fgets(input, sizeof(input), stdin); // 입력 받기

    // 입력이 숫자인지 확인
    if (input[0] == '-' || input[0] == '+') {
        number = 0;
        for (int i = 1; input[i] != '\0'; i++) {
            if (!isdigit(input[i])) {
                printf("오류: 유효한 정수가 아닙니다.\n");
                return 1; // 에러 처리로 프로그램 종료
            }
            number = number * 10 + (input[i] - '0');
        }
    } else {
        printf("오류: 숫자 시작이 잘못되었습니다.\n");
        return 1; // 에러 처리로 프로그램 종료
    }

    printf("입력된 숫자: %d\n", number);
    return 0;
}
```

**코드 분석:**
- `fgets`로 입력을 받고, 각 문자를 검사하여 숫자인지 확인합니다.
- `-` 또는 `+`로 시작하는 경우에도 숫자인지 철저히 검사합니다.

### 3. **조건 분기: 다양한 예외 처리 방식**

조건 분기를 활용해 다양한 예외 상황을 처리해봅시다. 여기서는 반복문을 통해 여러 번의 입력을 받고 각 입력에 따른 예외 처리를 보여드리겠습니다.

```c
#include <stdio.h>

int main() {
    int input;
    int attempts = 3; // 최대 시도 횟수

    printf("숫자 입력 시도 (3번까지): \n");

    for (int i = 0; i < attempts; i++) {
        printf("숫자를 입력하세요: ");
        scanf("%d", &input); // 정수 입력 받기

        if (input < 0) { // 음수 입력 시 예외 처리
            printf("음수는 허용되지 않습니다. 다시 시도하세요.\n");
            continue; // 다음 반복으로 넘어감
        } else if (input > 100) { // 범위 초과 시 예외 처리
            printf("입력이 너무 큽니다. 다시 시도하세요.\n");
            continue; // 다음 반복으로 넘어감
        } else {
            printf("정상 입력: %d\n", input);
            break; // 정상 범위 내의 입력이 들어오면 반복문 종료
        }
    }

    if (i == attempts) {
        printf("시도 횟수 초과. 프로그램 종료.\n");
    }

    return 0;
}
```

**코드 분석:**
- `for` 반복문을 통해 사용자에게 최대 3번의 입력 기회를 제공합니다.
- `if-else` 문을 통해 입력 값이 특정 조건을 만족하지 않을 때 예외 처리를 합니다.
- `continue`로 다음 반복으로 넘어가거나, 정상 범위 내의 입력이 들어오면 반복문을 종료합니다.

**Q: 예외 처리를 왜 그렇게 중요하게 생각해야 하나요?**
**A:** 예외 처리는 프로그램의 안정성을 크게 향상시키고, 사용자 경험을 개선합니다. 에러가 발생했을 때 프로그램이 깔끔하게 대응하고 사용자에게 명확한 피드백을 제공하면, 프로그램의 신뢰성이 높아집니다. 또한, 디버깅 과정에서 에러의 원인을 쉽게 파악할 수 있어 개발 효율성이 향상됩니다.

**Q: `errno`를 사용하는 이유는 무엇인가요?**
**A:** `errno`는 시스템 호출이 실패했을 때 해당 오류의 종류를 나타내는 전역 변수입니다. 이 변수를 통해 구체적인 에러 코드를 확인할 수 있어, 개발자가 문제를 더 정확히 진단하고 해결하는 데 도움이 됩니다.
