**27강: 동적 메모리 할당 - 실습과 예제**

### 왜 동적 메모리가 필요한가?

정적 메모리는 프로그램이 시작되기 전에 컴파일 시에 이미 할당되는 메모리입니다. 하지만, 이것은 프로그램의 실행 중에 메모리를 추가로 할당하는 것을 지원하지 않습니다. 예를 들어, 사용자가 이름을 입력하거나 데이터베이스에서 값을 읽는 경우, 정적 메모리는 이러한 요구 사항을 처리할 수 없습니다.

동적 메모리 할당이 필요한 이유는, 프로그램의 실행 중에 메모리를 추가로 할당하고 사용할 수 있기 때문입니다. 예를 들어, 사용자가 이름을 입력하는 경우, 동적 메모리는 사용자에게 입력을 받은 후에만 메모리를 할당합니다.

### malloc() 함수

C언어에서 동적 메모리 할당을 위해 **malloc()** 함수를 사용합니다. 이 함수는 프로그램이 할당받을 메모리의 크기를 byte 단위로 입력하면, 지정된 크기의 메모리가 할당됩니다.

예제:
```c
#include <stdlib.h>

int main() {
    int* ptr;
    ptr = (int*)malloc(10 * sizeof(int));
    if(ptr == NULL) {
        printf("메모리 할당 실패\n");
        return -1;
    }
    free(ptr);
    return 0;
}
```
위의 예제에서, `malloc(10 * sizeof(int))` 함수는 int 타입으로 10개의 메모리를 할당받습니다. 만약 메모리가 부족한 경우, `NULL` 값을 반환합니다.

### realloc() 함수

동적 메모리 할당을 할 때, 필요한 메모리의 크기가 변경될 수 있습니다. 이때, **realloc()** 함수를 사용하여 메모리의 크기를 변경할 수 있습니다.

예제:
```c
#include <stdlib.h>

int main() {
    int* ptr;
    ptr = (int*)malloc(10 * sizeof(int));
    if(ptr == NULL) {
        printf("메모리 할당 실패\n");
        return -1;
    }
    // 5개를 더할당해라
    ptr = realloc(ptr, 15 * sizeof(int));
    if(ptr == NULL) {
        printf("메모리 할당 실패\n");
        free(ptr);
        return -1;
    }
    free(ptr);
    return 0;
}
```
위의 예제에서, `realloc()` 함수는 현재 할당된 메모리의 크기를 5개 더해서 총 15개의 int 타입으로 변경합니다.

### 동적 메모리 할당의 실무 활용

동적 메모리 할당을 사용하는 경우, 주의할 점이 있습니다. 바로 **메모리 관리**입니다. 만약, 메모리를 계속해서 할당하고 해제하지 않고, 프로그램이 종료되는 경우, 메모리가 남아있는 현상이 발생할 수 있습니다.

예를 들어, 아래의 예제에서, `free()` 함수를 사용하여 메모리를 해제하지 않는 경우, 메모리가 부족한 오류가 발생합니다.
```c
#include <stdlib.h>

int main() {
    int* ptr;
    while(1) {
        // 10개씩 할당해라
        ptr = (int*)malloc(10 * sizeof(int));
        if(ptr == NULL) {
            printf("메모리 할당 실패\n");
            return -1;
        }
    }
    return 0;
}
```
### 최종적으로...

동적 메모리 할당은 프로그램의 실행 중에 메모리를 추가로 할당하고 사용할 수 있게 해주는 함수입니다. `malloc()`과 `realloc()` 함수를 사용하여 동적 메모리를 할당하고 관리할 수 있습니다. 만약, 메모리를 계속해서 할당하지 않고, 프로그램이 종료되는 경우, 메모리가 남아있는 현상이 발생할 수 있으니 주의하시길 바랍니다.
