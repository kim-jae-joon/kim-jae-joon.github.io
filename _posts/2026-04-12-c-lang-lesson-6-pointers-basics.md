## 🎯 6강: C 언어의 심장! 포인터, 이제 두려워 말고 친해져 봐요 (기본 개념 편)

### 🚀 1. 포인터는 대체 뭘까요? (택배 주소록 비유)

"포인터"라는 단어, 벌써부터 뭔가 어렵게 느껴지시죠? 하지만 우리 일상생활에서 쉽게 비유할 수 있습니다. 여러분, 택배 배달을 생각해보세요!

**컴퓨터 메모리 = 거대한 아파트 단지**
*   우리 컴퓨터의 메모리는 수많은 방(메모리 공간)으로 이루어진 거대한 아파트 단지라고 생각할 수 있어요.
*   각 방에는 고유한 '호실 번호'(메모리 주소)가 붙어 있습니다.
*   우리가 만든 변수(예: `int age = 25;`)는 이 아파트의 특정 호실에 '25'라는 값이 저장되는 것과 같아요.

**변수 = 아파트의 특정 호실과 그 안에 있는 물건**
*   `age`라는 이름은 그 호실의 '이름'이고, `25`라는 값은 그 호실 안에 있는 '물건'이죠.

**포인터 = '택배 송장' 또는 '주소록'**
*   포인터는 변수의 '값' 자체를 저장하는 대신, 그 값이 저장된 '메모리 주소'(호실 번호)를 저장하는 변수입니다.
*   택배 기사님이 물건(값)을 배달하려면 '주소'(메모리 주소)를 알아야 하죠? 포인터가 바로 그 주소를 가지고 있는 역할을 합니다.
*   그래서 포인터는 어떤 물건(데이터)이 *어디에* 있는지 알려주는 **"위치 정보 저장 변수"**라고 할 수 있습니다.

"나는 101호라는 주소를 알고 있어!" 라고 말하는 변수라는 거죠!

### 🤔 2. 왜 포인터가 필요할까요? (그냥 변수 쓰면 안 돼?)

"에이, 그냥 변수에 값 저장하면 되지, 왜 주소까지 관리해야 해요?" 라고 생각하실 수 있어요. 하지만 포인터는 C 언어의 강력함을 극대화하는 핵심 도구입니다.

1.  **메모리에 직접 접근:** 포인터를 사용하면 메모리의 특정 위치에 직접 접근하여 값을 읽거나 쓸 수 있습니다. 이는 하드웨어 제어나 운영체제 개발처럼 저수준(low-level) 프로그래밍에 필수적입니다.
2.  **효율적인 데이터 처리:** 함수에 큰 데이터를 전달할 때, 데이터를 통째로 복사하는 대신 그 데이터의 '주소'만 전달하면 훨씬 빠르고 효율적입니다. (예: 배열)
3.  **동적 메모리 할당:** 프로그램 실행 중에 필요한 만큼 메모리를 할당하고 해제할 수 있게 해줍니다. (나중에 `malloc`, `free` 함수 배울 때 자세히 다룰 거예요!)
4.  **복잡한 자료구조 구현:** 연결 리스트, 트리, 그래프 등 복잡한 자료구조를 만들 때 포인터는 필수적인 요소입니다.

이러한 이유들 때문에 포인터는 C 언어에서 없어서는 안 될 '심장' 같은 존재인 것이죠!

### 🔍 3. 메모리 주소, 너 어디 있니? - 주소 연산자 `&`

자, 이제 포인터가 주소를 저장하는 변수라는 것을 알았으니, 가장 먼저 해야 할 일은 변수의 '주소'를 알아내는 것입니다. 이때 사용하는 것이 바로 **주소 연산자 `&` (앰퍼샌드)** 입니다.

`&` 연산자를 변수 앞에 붙이면, 해당 변수가 메모리의 어디에 저장되어 있는지 '주소'를 알려줍니다.

**[예제 코드 1: 변수의 주소 확인하기]**

```c
#include <stdio.h> // 표준 입출력 함수를 사용하기 위한 헤더 파일

int main() {
    int age = 25;       // 정수형 변수 age 선언 및 25로 초기화
    double gpa = 4.2;   // 실수형 변수 gpa 선언 및 4.2로 초기화
    char initial = 'J'; // 문자형 변수 initial 선언 및 'J'로 초기화

    // 각 변수의 값과 메모리 주소를 출력합니다.
    // %p는 포인터(메모리 주소)를 출력할 때 사용하는 형식 지정자입니다.
    printf("age의 값: %d, age의 메모리 주소: %p\n", age, &age);
    printf("gpa의 값: %.1f, gpa의 메모리 주소: %p\n", gpa, &gpa);
    printf("initial의 값: %c, initial의 메모리 주소: %p\n", initial, &initial);

    return 0; // 프로그램 성공적으로 종료
}
```

**[컴파일 및 실행 결과 예시]**
(실행할 때마다 주소 값은 다를 수 있습니다.)

```
age의 값: 25, age의 메모리 주소: 0x7ffee1f038c8
gpa의 값: 4.2, gpa의 메모리 주소: 0x7ffee1f038d0
initial의 값: J, initial의 메모리 주소: 0x7ffee1f038c7
```

**설명:**
*   `&age`는 `age` 변수가 저장된 메모리 공간의 시작 주소를 나타냅니다.
*   `%p`는 이 주소 값을 16진수 형태로 출력해 줍니다.
*   주소 값은 컴퓨터 환경마다, 그리고 프로그램을 실행할 때마다 달라질 수 있으니 걱정 마세요! 중요한 건 `&`가 주소를 알려준다는 사실입니다.

### 📝 4. 주소를 담는 변수: 포인터 선언하기 `*`

이제 변수의 주소를 알아내는 방법을 알았으니, 그 주소를 저장할 포인터 변수를 만들어야겠죠? 포인터 변수를 선언할 때는 **`*` (애스터리스크)** 기호를 사용합니다.

**[포인터 선언 문법]**

```c
자료형 * 포인터변수이름;
```

여기서 중요한 점은, `자료형`은 포인터가 **'어떤 종류의 데이터를 가리킬지'** 명시한다는 것입니다.
*   `int *ptr_int;` 는 `int`형 변수의 주소를 저장할 포인터 `ptr_int`를 선언합니다.
*   `char *ptr_char;` 는 `char`형 변수의 주소를 저장할 포인터 `ptr_char`를 선언합니다.

**왜 `자료형`이 필요할까요?**
포인터는 주소만 저장하지만, 그 주소에 어떤 타입의 데이터가 저장되어 있는지 알아야 나중에 그 데이터를 제대로 읽거나 쓸 수 있기 때문입니다. (예를 들어, `int`는 4바이트, `char`는 1바이트를 차지하므로, 포인터는 주소에서부터 몇 바이트를 읽어야 할지 알아야 합니다.)

**[예제 코드 2: 포인터 변수 선언 및 주소 저장]**

```c
#include <stdio.h>

int main() {
    int score = 100;       // 정수형 변수 score 선언 및 초기화
    int *ptr_score;        // 정수형 변수의 주소를 저장할 포인터 변수 ptr_score 선언

    // &score (score의 주소)를 ptr_score 포인터에 할당
    ptr_score = &score;

    printf("score의 값: %d\n", score);
    printf("score의 주소: %p\n", &score);
    printf("ptr_score가 가진 주소 값: %p\n", ptr_score); // 포인터 변수 자체가 저장하고 있는 값 (주소)

    // 참고: ptr_score 변수 자체의 주소도 알 수 있습니다.
    printf("ptr_score 변수 자체의 주소: %p\n", &ptr_score);

    return 0;
}
```

**[컴파일 및 실행 결과 예시]**

```
score의 값: 100
score의 주소: 0x7ffee23158e4
ptr_score가 가진 주소 값: 0x7ffee23158e4
ptr_score 변수 자체의 주소: 0x7ffee23158e8
```

**설명:**
*   `int *ptr_score;` 에서 `*`는 `ptr_score`가 포인터 변수임을 선언하는 역할을 합니다.
*   `ptr_score = &score;` 를 통해 `ptr_score`는 `score` 변수의 메모리 주소를 값으로 저장하게 됩니다.
*   보세요! `score의 주소`와 `ptr_score가 가진 주소 값`이 똑같죠? `ptr_score`는 이제 `score`의 위치를 정확히 알고 있는 겁니다!

### 🌟 5. 주소 속 값 꺼내보기: 역참조 연산자 `*`

포인터가 주소를 가지고 있다는 건 이제 알겠어요. 그런데 그 주소에 있는 '값'을 어떻게 꺼내올까요? 또는 그 주소에 새로운 '값'을 어떻게 저장할까요? 이때 다시 `*` (애스터리스크) 연산자를 사용합니다.

이번에는 선언할 때와는 다른 의미로 사용됩니다. 포인터 변수 앞에 `*`를 붙이면, 포인터가 가리키는 **메모리 주소로 찾아가서 그곳에 저장된 '값'을 가져오거나 변경**합니다. 이것을 **'역참조 (Dereferencing)'**라고 부릅니다.

비유하자면, 택배 송장(포인터)에 적힌 주소로 찾아가서(역참조) 그 집 안에 있는 물건(값)을 꺼내거나, 새 물건을 넣어주는 행동과 같습니다.

**[예제 코드 3: 포인터를 통한 값 접근 및 변경 (역참조)]**

```c
#include <stdio.h>

int main() {
    int num = 77;          // 정수형 변수 num 선언 및 77로 초기화
    int *ptr_num;          // 정수형 변수의 주소를 저장할 포인터 ptr_num 선언

    ptr_num = &num;        // num 변수의 주소를 ptr_num에 저장

    printf("1. num의 값: %d\n", num);
    printf("2. ptr_num이 가리키는 주소: %p\n", ptr_num);
    printf("3. ptr_num을 통해 접근한 num의 값: %d\n", *ptr_num); // 역참조! ptr_num이 가리키는 곳의 값을 가져옴

    // 이제 포인터를 통해 값을 변경해봅시다!
    *ptr_num = 123;        // ptr_num이 가리키는 메모리 위치의 값을 123으로 변경

    printf("4. *ptr_num을 통해 값을 변경 후, num의 값: %d\n", num);
    printf("5. *ptr_num을 통해 값을 변경 후, ptr_num이 가리키는 값: %d\n", *ptr_num);

    return 0;
}
```

**[컴파일 및 실행 결과]**

```
1. num의 값: 77
2. ptr_num이 가리키는 주소: 0x7ffee76e28e4
3. ptr_num을 통해 접근한 num의 값: 77
4. *ptr_num을 통해 값을 변경 후, num의 값: 123
5. *ptr_num을 통해 값을 변경 후, ptr_num이 가리키는 값: 123
```

**설명:**
*   `*ptr_num`은 `ptr_num`이 가리키는 메모리 주소로 찾아가 그곳에 있는 `int`형 값 77을 가져옵니다.
*   `*ptr_num = 123;` 이 구문이 정말 중요합니다! `ptr_num`이 가리키는 메모리 공간에 직접 접근하여 그 값을 123으로 변경하는 것입니다.
*   그 결과, `num` 변수의 값도 77에서 123으로 변경된 것을 확인할 수 있습니다. 포인터는 이렇게 원본 변수의 값에 직접적인 영향을 줄 수 있습니다!

### 💡 6. 포인터 타입은 왜 중요할까요?

앞서 포인터 선언 시 `int *`, `char *`처럼 자료형을 명시해야 한다고 했죠? 단순히 타입 검사를 위함만은 아닙니다. 포인터가 가리키는 데이터의 '크기'를 컴파일러에게 알려주는 중요한 역할을 합니다.

*   `int *`: "나는 `int`형(대부분 4바이트) 데이터가 있는 주소를 가리킬 거야!"
*   `char *`: "나는 `char`형(1바이트) 데이터가 있는 주소를 가리킬 거야!"
*   `double *`: "나는 `double`형(대부분 8바이트) 데이터가 있는 주소를 가리킬 거야!"

이 정보는 특히 포인터를 역참조하여 값을 읽거나 쓸 때, 또는 포인터 연산(다음 강에서 자세히 다룰 내용)을 할 때 컴파일러가 정확히 몇 바이트를 다루어야 할지 결정하는 데 사용됩니다.

**[예제 코드 4: 포인터 타입과 크기]**

```c
#include <stdio.h>

int main() {
    int     a = 10;
    char    b = 'X';
    double  c = 3.14;

    int *ptr_a = &a;       // int 변수 a의 주소를 가리키는 int 포인터
    char *ptr_b = &b;      // char 변수 b의 주소를 가리키는 char 포인터
    double *ptr_c = &c;    // double 변수 c의 주소를 가리키는 double 포인터

    printf("int 포인터가 가리키는 값: %d\n", *ptr_a);
    printf("char 포인터가 가리키는 값: %c\n", *ptr_b);
    printf("double 포인터가 가리키는 값: %.2f\n", *ptr_c);

    printf("\n--- 각 포인터가 가리키는 '대상'의 크기 ---\n");
    printf("sizeof(*ptr_a): %zu 바이트 (int 크기)\n", sizeof(*ptr_a)); // ptr_a가 가리키는 int의 크기
    printf("sizeof(*ptr_b): %zu 바이트 (char 크기)\n", sizeof(*ptr_b)); // ptr_b가 가리키는 char의 크기
    printf("sizeof(*ptr_c): %zu 바이트 (double 크기)\n", sizeof(*ptr_c)); // ptr_c가 가리키는 double의 크기

    printf("\n--- 각 포인터 '변수 자체'의 크기 (주소를 저장하는 공간의 크기) ---\n");
    printf("sizeof(ptr_a): %zu 바이트\n", sizeof(ptr_a)); // 포인터 변수 ptr_a 자체의 크기
    printf("sizeof(ptr_b): %zu 바이트\n", sizeof(ptr_b)); // 포인터 변수 ptr_b 자체의 크기
    printf("sizeof(ptr_c): %zu 바이트\n", sizeof(ptr_c)); // 포인터 변수 ptr_c 자체의 크기

    return 0;
}
```

**[컴파일 및 실행 결과 예시]** (시스템 환경에 따라 크기가 다를 수 있습니다. 64비트 시스템 기준.)

```
int 포인터가 가리키는 값: 10
char 포인터가 가리키는 값: X
double 포인터가 가리키는 값: 3.14

--- 각 포인터가 가리키는 '대상'의 크기 ---
sizeof(*ptr_a): 4 바이트 (int 크기)
sizeof(*ptr_b): 1 바이트 (char 크기)
sizeof(*ptr_c): 8 바이트 (double 크기)

--- 각 포인터 '변수 자체'의 크기 (주소를 저장하는 공간의 크기) ---
sizeof(ptr_a): 8 바이트
sizeof(ptr_b): 8 바이트
sizeof(ptr_c): 8 바이트
```

**설명:**
*   `sizeof(*ptr_a)`는 `ptr_a`가 가리키는 `int`형 데이터의 크기(4바이트)를 알려줍니다.
*   `sizeof(ptr_a)`는 `ptr_a`라는 **포인터 변수 자체**가 메모리 주소를 저장하는 데 필요한 공간의 크기(64비트 시스템에서는 8바이트)를 알려줍니다.
*   어떤 타입의 변수를 가리키든, 포인터 변수 자체의 크기는 (주소의 크기이므로) 동일하다는 것을 알 수 있습니다. **이 부분은 초보자들이 가장 헷갈려하는 부분이므로 꼭 기억해두세요!**

### ⚠️ 7. 안전하게 사용하기: NULL 포인터

포인터는 강력하지만, 잘못 사용하면 프로그램에 치명적인 오류를 발생시킬 수 있습니다. 특히, **'아무것도 가리키지 않는' 포인터**를 사용하는 경우입니다. 이런 위험을 줄이기 위해 `NULL` 포인터 개념을 사용합니다.

*   **`NULL`이란?** `NULL`은 C 언어에서 포인터가 **아무런 유효한 메모리 주소도 가리키고 있지 않음**을 나타내는 특별한 값입니다. (대부분의 시스템에서 0으로 정의됩니다.)
*   **사용 목적:**
    1.  **초기화:** 포인터를 선언할 때, 당장 가리킬 주소가 없다면 `NULL`로 초기화하여 '아무것도 가리키지 않음' 상태임을 명확히 합니다. 이는 예상치 못한 오류를 방지하는 좋은 습관입니다.
    2.  **유효성 검사:** 포인터를 사용하기 전에 해당 포인터가 `NULL`인지 아닌지 검사하여, 유효하지 않은 메모리 접근(역참조)으로 인한 프로그램 충돌을 막을 수 있습니다.

**[예제 코드 5: NULL 포인터 활용]**

```c
#include <stdio.h> // 표준 입출력
#include <stddef.h> // NULL 매크로를 포함하는 헤더 파일 (C99부터)

int main() {
    int *ptr_data = NULL; // 포인터를 NULL로 초기화하는 좋은 습관!

    // NULL 포인터를 역참조하면 프로그램이 비정상 종료될 수 있습니다! (런타임 에러)
    // 아래 코드는 주석 처리하여 에러를 방지합니다.
    // printf("NULL 포인터가 가리키는 값: %d\n", *ptr_data); // <-- 이 줄의 주석을 풀면 위험!

    // 포인터를 사용하기 전에 NULL 여부를 항상 확인하는 습관을 들입시다.
    if (ptr_data != NULL) {
        printf("ptr_data는 유효한 주소를 가리키고 있습니다.\n");
        printf("ptr_data가 가리키는 값: %d\n", *ptr_data);
    } else {
        printf("ptr_data는 현재 아무것도 가리키고 있지 않습니다 (NULL).\n");
    }

    int value = 42;
    ptr_data = &value; // 이제 ptr_data가 value의 주소를 가리킴 (유효한 주소)

    if (ptr_data != NULL) {
        printf("\nptr_data는 이제 유효한 주소를 가리킵니다.\n");
        printf("ptr_data가 가리키는 값: %d\n", *ptr_data);
    } else {
        printf("\nptr_data는 현재 아무것도 가리키고 있지 않습니다 (NULL).\n");
    }

    return 0;
}
```

**[컴파일 및 실행 결과]**

```
ptr_data는 현재 아무것도 가리키고 있지 않습니다 (NULL).

ptr_data는 이제 유효한 주소를 가리킵니다.
ptr_data가 가리키는 값: 42
```

**설명:**
*   처음에 `ptr_data`를 `NULL`로 초기화했기 때문에 첫 번째 `if`문에서 `else` 블록이 실행됩니다.
*   `ptr_data = &value;`를 통해 유효한 주소를 할당한 후에는 두 번째 `if`문에서 `if` 블록이 실행되고, `*ptr_data`를 통해 `value`의 값 42를 성공적으로 출력합니다.
*   `NULL` 포인터는 안전한 C 프로그래밍을 위한 필수 개념이니, 꼭 기억하고 활용해주세요!

## 맺음말: 포인터, 이제 친구가 될 시간!

*   `&` 연산자로 변수의 **주소**를 얻는 방법
