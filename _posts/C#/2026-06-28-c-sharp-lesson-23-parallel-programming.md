##  🔥 23강: 병렬 처리 기법, 단 하나의 CPU로 백만 개의 작업을 동시에?! 🚀

안녕하세요! C# 장인이 되는 여정, 함께 걸어가고 있는 **15년 차 시니어 개발자, 그리고 대한민국 최고의 C# 일타 강사**입니다. 😎 오늘은 C# 세계의 마법같은 기술, **병렬 처리 기법**을 배우게 될 거예요! 

### 병렬 처리 기법이란 무엇일까요? 🤔

가끔 우리 주변에서 "CPU 한 개로 백만 개의 일을 동시에 처리하는건 어떤 일지?" 라는 질문을 들어본 적 있나요?  말 그대로, 단 하나의 CPU를 이용해 여러 작업을 **동시**에 수행하는 기술이죠! 🤯

예를 들어, 웹페이지를 열었을 때 이미지, CSS, JavaScript 파일 등 다양한 정보를 동시에 불러와서 화면에 보여줍니다. 이것도 바로 병렬 처리 기법의 작품입니다.  

**💡 초보자 폭풍 질문!**
> "병렬 처리 = 여러 개의 CPU 사용?" 라는 생각을 하셨나요? 오히려 한 개의 CPU를 최대한 효율적으로 활용하는 기술이죠! 💪


### C#에서 병렬 처리, 어떻게 하는 걸까요? 🧐

C# 에서 병렬 처리를 위해 `Parallel` namespace와 같은 도구들을 사용할 수 있습니다. 주로 두 가지 방식으로 병렬 처리를 구현합니다.

**1.  `Parallel.For()`**: 반복문을 병렬로 실행하는 데 사용됩니다. 🔥

   ```csharp
   Parallel.For(0, 10000, (i) => {
       // 각 쓰레드가 하나씩 처리
       Console.WriteLine($"쓰레드 ID: {Thread.CurrentThread.ManagedThreadId}, i = {i}");
   }); 
   ```

   - `Parallel.For()`는 각 요소를 하나의 쓰레드에 할당하여 병렬로 실행합니다.


**2.  `Task Parallel Library (TPL)`**: 작업을 `Task` 객체로 표현하고, 여러 `Task`를 동시에 실행하는 방식입니다. 🚀

   ```csharp
   Task.Run(() => {
       // 작업 1
       Console.WriteLine("작업 1 완료!");
   });

   Task.Run(() => {
       // 작업 2
       Console.WriteLine("작업 2 완료!");
   });
   ```

   - `Task.Run()`은 새로운 쓰레드에서 실행하는 작업을 생성합니다. 두 작업이 각각 다른 쓰레드에서 병렬로 실행되며, 종료 시점에 결과를 확인할 수 있습니다.

###  🚨 실무주의보! 🚨


병렬 처리 기술은 프로그램의 성능을 크게 향상시킬 수 있지만, 적절하게 사용해야 합니다. **데이터 race condition**, **lock 영역 문제** 등 병렬 처리 과정에서 발생할 수 있는 오류를 주의해서 방지해야 합니다.
