## 17강: 예외 처리 - 오류 관리 및 복구: 코드의 불안정 요소를 제압하는 마법!

**🤯 초보자도 감탄할 만한 완벽 가이드로, C# 개발의 신비로운 세계에 한 발 더 다가가 보세요! 🧙‍♂️**

혹시 "예외 처리" 라는 단어를 들을 때 기분이 어울리지 않는다면… 당신은 아직 이 명석한 기술을 제대로 접하지 못했답니다. 마치 요리하는 중에 예상치 않게 불맛나는 냄새가 날 때, 혹시라도 화재로 번질 위험까지 노려보는 건 어떨까요? 🤔 아니죠! 바로 문제 상황에 맞춰 빠르고 효과적으로 대처하는 것이 중요하죠. 코드 개발 역시 마찬가지입니다. 오류나 예상치 못한 상황이 발생했을 때, 우리의 프로그램은 흔들리며 심각한 파국으로 이어질 수 있습니다. 하지만 걱정하지 마세요! 오늘부터 "예외 처리"라는 코드의 경찰관과 같은 존재를 통해 이러한 불안정 요소들을 효과적으로 관리하고 복구하는 법을 배우겠습니다. 🤘

**📚 예외 처리 개념 - 나뭇가지 위에 걸어보는 코딩!**


  예외 처리란 프로그램 실행 중 발생할 수 있는 오류나 예상치 못한 상황(이를 '예외'라고 부릅니다)을 감지하고, 해당 상황을 안전하게 억제하며 프로그램이 충돌하지 않도록 보호하는 시스템입니다. 마치 나뭇가지 위에 걸어다니는 것처럼, 예상치 못한 오류나 불안정 요소들이 나타났을 때 균형을 잃고 빠져내리지 않도록 '보호자' 역할을 하는 것입니다.

**💡 초보자 폭풍 질문!**

* "그럼 나뭇가지에서 떨어졌을 때 무슨 일이야?" 🤔
   > 잘못된 입력이나 파일 존재하지 않는 경고 메시지를 표시하는 것처럼 프로그램은 예외 상황을 인식하고 안전하게 종료되거나 특정 처리를 수행합니다.

* "나뭇가지 보호 장치는 어떻게 작동할까?" 🤔
   > 예외 처리 코드(예: `try-catch` 블록)를 사용하여 프로그램의 실행 과정을 모니터링하고 발생하는 예외 상황에 대응해 처리합니다.

**💪 C#에서 예외 처리 - 실전 코딩 튜토리얼!**



**1. try-catch 블록:** 가장 기본적인 예외 처리 도구입니다. 
  * `try` 블록 안에는 발생할 수 있는 예외를 포괄하는 코드를 작성합니다. 
  * `catch` 블록은 `try` 블록에서 발생한 특정 예외 유형을 감지하여 실행하고, 오류 상황을 안전하게 관리합니다.

```C#
try
{
    // 예상치 못한 오류가 발생할 수 있는 코드
    int result = 10 / 0; // 0으로 나누는 오류 발생!
    Console.WriteLine("결과: " + result); 
}
catch (DivideByZeroException ex)
{
    // 0으로 나누는 오류가 발생했을 때 실행되는 코드
    Console.WriteLine("오류 메시지: " + ex.Message); // 오류 메시지를 표시합니다.
}

```

**💡 초보자 폭풍 질문!**

* "그럼 `try-catch`를 사용하면 프로그램이 종료되지 않겠어요?" 🤔
   > 예외 처리가 성공적으로 실행되면 프로그램은 정상적으로 작동하며, 오류 상황을 안전하게 관리하여 충돌을 방지합니다.

 **2. finally 블록:** `try-catch` 블록에 추가 가능한 `finally` 블록은 반드시 실행되는 코드를 포함하는 부분입니다. 
  * 자원 해제(예: 파일, 네트워크 연결 등) 작업을 수행하거나 프로그램 종료 시 필요한 처리를 명령합니다.

```C#
try
{
    // 파일을 읽는 코드
}
catch (FileNotFoundException ex)
{
    Console.WriteLine("오류 메시지: " + ex.Message); 
}
finally
{
    // 파일 객체 해제 작업
}
```

**🚨 실무주의보!**

* `finally` 블록은 반드시 실행되기 때문에, 오류 처리 시 자원 해제 등 필수적인 작업을 이 블록에 담아야 합니다. 

 **3. 다양한 예외 유형:** C#는 다양한 예외 유형을 제공하여 프로그램에서 발생하는 특정 오류를 정확히 감지하고 처리할 수 있도록 합니다.
  * `DivideByZeroException`: 0으로 나누기 시 발생하는 오류입니다.

```C#
try
{
    int result = 10 / 0; // DivideByZeroException 발생!
}
catch (DivideByZeroException ex)
{
    Console.WriteLine("오류 메시지: " + ex.Message); // 예외 메시지를 출력합니다.
}

```


  * `FileNotFoundException`: 존재하지 않는 파일을 열려고 할 때 발생하는 오류입니다. 
   
```C#
try
{
    // 파일이 없을 경우 FileNotFoundException 발생!
    using (StreamReader reader = new StreamReader("존재하지않는파일.txt"))
    {
        string content = reader.ReadToEnd();
    }
}
catch (FileNotFoundException ex)
{
    Console.WriteLine("오류 메시지: " + ex.Message); // 예외 메시지를 출력합니다.
}

```


  * `ArgumentNullException`: null 값을 매개변수로 전달하는 경우 발생하는 오류입니다. 

```C#
try
{
    // string이 null일 경우 ArgumentNullException 발생!
    string text = null;
    Console.WriteLine(text.Length); // NullReferenceException 예외 발생 가능성!
}
catch (ArgumentNullException ex)
{
    Console.WriteLine("오류 메시지: " + ex.Message); // 예외 메시지를 출력합니다.
}

```



**💡 초보자 폭풍 질문!**

* "어떻게 이렇게 다양한 예외 유형을 활용할 수 있을까요?" 🤔


  > 각 예외 유형은 프로그램에서 발생하는 특정 오류 상황에 해당하며, `catch` 블록에서 맞는 예외 유형에 대한 코드를 작성하여 그에 맞는 처리 과정을 명령합니다. 예를 들어, 파일이 없을 때 `FileNotFoundException`을 사용하여 파일 존재 여부를 확인하고 적절한 메시지를 출력하거나 프로그램 실행 루프를 변경하는 등의 대처 방법을 선택할 수 있습니다.
