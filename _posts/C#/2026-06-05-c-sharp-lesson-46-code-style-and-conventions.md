## 🚀  46강: Code Style and Conventions - 코드 스타일 및 관습, 개발자의 정신 수칙이야!

안녕하세요! 👋 최고의 C# 일타 강사이자 15년차 시니어 개발자의 **최근호를 들여다보는** 시간을 가져봐요. 🔥 오늘은 "코드 스타일 및 관습" 에 대해 알아볼게요! 처음 코딩을 접하는 친구들은 이걸 '규칙'이라고 생각하지 마세요! 😜

> 이런 게 없으면 개발자들의 코드는 마치 장난감 모형처럼 복잡하고 혼란스러워지죠.  🤯


### 🤔  왜 코드 스타일과 관습이 중요할까요?

✨  **독창적인 코딩을 위한 토대!**

마법의 원칙은 이미 있다고 할게요. 마법사가 사용하는 주문처럼, 우리도 개발자로서 일관된 방식으로 코드를 작성해야 해요! 이렇게 하면 다른 개발자가 우리 코드를 읽으면서 "이 녀석 진짜 천재야!" 라고 생각하실 수 있도록 도와주겠죠. 😎

✨ **팀워크의 동반자!** 여러 개발자들이 함께 프로젝트에 참여하는 경우, 코드 스타일을 따르면 서로의 코드를 이해하고 수정하기 쉬워져요. 🤝 마치 같은 언어를 사용하는 것처럼 효율적이죠!

✨ **결함 없는 코드 만들기!**  규칙적인 코드는 오류를 줄이고 유지 보수가 용이해지는 결과를 가져옵니다. 😉

### ✨ 주요 C# 코드 스타일 및 관습 🕵️‍♂️

#### 1️⃣ 공간 (Whitespace) : 눈과 코드의 휴식 시간 🥱

- 너무 밀집된 코드는 읽기 어렵고, 마치 고소한 꽃밭에서 비명을 지르는 벌처럼 심각해요! 😬
- 문장 구분과 블록 분리에 공간을 사용하는 것이 중요합니다. 😎

```csharp
// 잘못된 코드 예시: 너무 밀집된 코드
int x = 10;
if(x>5) { 
    Console.WriteLine("큰 숫자!"); }
int y=20;
Console.WriteLine("마지막 출력");

```


```csharp
// 좋은 코드 예시: 공간 활용하여 가독성 향상!
int x = 10; 

if (x > 5) 
{
    Console.WriteLine("큰 숫자!");
}

int y = 20;
Console.WriteLine("마지막 출력");
```


#### 2️⃣ 코드 정렬 : 행복한 정리된 삶! 🎉

- 변수, 함수 호출 등의 코드를 일관되게 정렬하는 것이 중요합니다. 😉
- 개발자는 눈으로 읽을 때, 시각적으로 보기 좋은 모양새가 필요해요.  👀


```csharp
// 잘못된 코드 예시: 횡갈이 없는 행복은? :P

int x = 10;
String name = "김철수";
Console.WriteLine(x+name);

```



```csharp
// 좋은 코드 예시: 정렬해서 마음까지 편안해지는 코드!

int x = 10;
string name = "김철수";
Console.WriteLine(x + name); // 여기서는 간단하게 출력하기 때문에, 추가적인 공백은 생략했어요.


```




#### 3️⃣ 주석 :  코드의 해설서!  📚

- 코드를 작성할 때, 다른 개발자가 이해하기 쉽도록 주석을 활용해야 합니다. 😎
- 주석을 통해 코드의 목적, 사용 방법 등을 명확하게 설명하세요!


```csharp
// 이 부분에서 사용자 입력값을 확인하고 처리합니다.
string userInput = Console.ReadLine();

if (userInput == "hello") 
{
    Console.WriteLine("안녕하세요!"); // 주석으로 코드의 목적을 명시하여 이해도를 높입니다.
} 
else
{
    Console.WriteLine("잘못된 입력입니다.");
}


```



#### 4️⃣ 변수 이름 :  말로 하는 의미!  🗣️

- 사용하는 변수의 역할을 잘 나타내는 이름을 사용해야 합니다. 💪
`x`, `y`처럼 추상적인 단어를 사용하는 것은 피하세요! 🤔



```csharp
// 좋은 코드 예시: 명확한 변수 이름 사용하여 의미 전달!
int studentAge = 20; // 학생 나이는 20입니다. - 쉬운 이해

// 잘못된 코드 예시: 추상적인 변수 이름은 어려움을 야기합니다!
int a = 20; //  ? 🤔


```



#### 5️⃣ 함수 명명 :  내용을 직관적으로 전달해요! 🎯

- 함수의 역할을 분명하게 나타낼 수 있는 명칭을 사용해야 합니다. 😉



```csharp
// 좋은 코드 예시: 함수 명칭이 기능을 설명합니다!
int CalculateSum(int num1, int num2) { // 두 정수를 더하는 함수입니다.
    return num1 + num2;
}

```
