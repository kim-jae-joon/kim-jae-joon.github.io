## 🔥🚀  C# 매스터 🚀🔥  44강: Logging - 로그 기록, 그 안에는 무엇이?!

**안녕하세요, C# 일타 최고 강사 '코드왕'입니다! 😎✨ ** 15년 동안 C# 개발 생활을 하며 만난 다양한 코드 괴상들이 많지만, 특히 "로깅?"라고 물어보는 초보자들의 모습이 정말 귀엽죠.  🙈 그래서 오늘은 바로 로그 기록에 대해 몰입해서 배워볼 거예요!

로그 기록? 단순히 코드가 실행되는 과정을 기록하는 것일까요? 아닙니다, C# 개발자들은 **"코드의 DNA"**라고 부르기도 합니다. 🧬✨  

> 로그를 분석하면, 프로그램이 어떻게 작동하고 문제 발생 시 어떤 원인으로 인해 오류가 발생했는지 알 수 있어요! 🚀💡

###  🤔 로그 기록은 왜 필요할까요?


**🚨 실무주의보:** 프로그램 개발 과정에서 예상치 못한 에러가 발생하는 건 당연합니다.  😭 그런데 문제점을 해결하기 위해 "무엇이 잘못되었는지"를 파악해야죠! 이때 바로 로깅이 등장하며, 코드 숲 속에서 길을 찾아주는 마법 지팡이의 역할을 합니다. ✨

**⚡️ 프로그램의 행동 패턴 파악:**
* 프로그램 실행 순서와 각 함수 호출 시간을 기록하여 효율성 분석에 활용할 수 있습니다!  📈
* 예를 들어, 특정 로직이 오래 걸리는 경우, 그 원인을 찾아 개선점을 모색할 수 있죠. 💪

**🐛 문제 해결의 열쇠:**
* 에러 발생 시 로그 파일은 문제 현상을 파악하는 데 필수적인 증거입니다! 🕵️‍♀️
* 에러 메시지와 함께 호출된 함수, 변수 값 등을 기록하여 원인 분석이 용이해집니다.

**💡 초보자 폭풍 질문!: "로그를 저장할 곳은 어디인가요?"**

> 잘 맞춰 보셨네요! 프로그램 실행 중 생성되는 로그들은 파일 형태로 저장되며, 그 위치는 개발 환경에 따라 다릅니다!
   * **Console:**  실행하면 곧바로 화면에 출력되는 로그입니다. 간단한 디버깅에 유용하지만, 실제 로직 분석에는 적합하지 않아요.
   * **File Logging:** 프로그램 설정을 통해 지정된 파일 경로로 로그를 저장합니다. 가장 일반적인 방법이며, 흔히 "log.txt" 또는 "app.log"와 같은 이름으로 저장됩니다!

### 🌈  C# 로깅 기법 살펴보기

**1️⃣ Console Logging (간단한 화면 출력)**

```c#
Console.WriteLine("Hello from C#!"); // 콘솔 창에 "Hello from C!"를 출력합니다.
```

* 단순히 메시지를 화면에 출력하는 방식입니다. 디버깅 등 빠른 확인에 유용하지만, 파일 기록과 달리 로그 파일로 저장되지 않아 장기적인 분석에는 적합하지 않습니다!

**💡 초보자 폭풍 질문!: "Console logging은 어떻게 사용할 수 있을까요?"**

> C# 프로그램 실행 시, `Console.WriteLine()` 메서드를 사용하면 화면에 메시지를 출력할 수 있습니다! 예를 들어, "Hello World!" 라는 메시지를 콘솔 창에 보이게 하려면  `Console.WriteLine("Hello World!");` 와 같이 코드를 작성합니다. 📝

**2️⃣ File Logging (로그 파일 저장)**

```c#
using System;
using System.IO; // 파일 작업 관련 기능을 사용하기 위한 추가 라이브러리 필요!

public class Example
{
    public static void Main(string[] args)
    {
        // 현재 시간을 가져오기
        DateTime now = DateTime.Now;

        // 로그 파일 열기 (파일이 없다면 새로 만들어줍니다!)
        using (StreamWriter writer = new StreamWriter("log.txt", true)) 
        {  
            writer.WriteLine($"{now} - 프로그램 실행 시작."); // 현재 시간과 메시지를 로그 파일에 기록합니다!

            // 여기서는 간단한 프로그램 동작을 구현하는 코드를 추가할 수 있습니다.
            Console.WriteLine("프로그램이 실행 중입니다!");
        }

        Console.ReadLine(); // 콘솔 창을 닫지 않도록 명령문 추가

    }
}
```


* `StreamWriter` 클래스를 사용하여 파일로 로그 메시지를 기록합니다! `true` 매개변수는 기존에 존재하는 로그 파일에 내용을 추가하는 모드를 의미하며, "log.txt" 파일이 없으면 새로 만들어줍니다.
* 중요하게도, 코드 실행 중 발생하는 에러와 관련된 정보 또한 로그 파일로 저장하여 문제 해결 과정에서 큰 도움이 됩니다! 💪

**💡 초보자 폭풍 질문!: "로그 파일은 어떻게 관리할까요?"**

> 좋은 질문입니다! 로그 파일은 시간이 지남에 따라 커질 수 있으므로, 자동으로 백업하거나 정기적으로 삭제하는 작업을 설정하는 것이 좋습니다. ♻️  또한, 로그 분석 도구를 사용하여 효율적인 데이터 처리와 문제 해결을 위한 전략을 세우는 것도 좋은 방법입니다.

**3️⃣ Logging Libraries (더욱 강력하고 편리하게!)**

* **NLog:** 자유롭고 확장성이 뛰어난 라이브러리로, 다양한 로그 출력 형식과 플랫폼 지원, 오류 관리 기능을 제공합니다! 🚀
* **Serilog:** 현대적인 디자인과 성능 향상에 최적화되어 있으며, JSON 포맷으로 로그 데이터를 기록하여 분석이 용이합니다. ✨

**🚨 실무주의보:** 솔직히 말해서, 개발 초반에는 간단한 `Console` logging을 활용해도 충분합니다! 하지만 프로젝트가 커지면서 로깅의 중요성이 높아지고, 오류 관리 및 분석에 대한 필요성이 증가하면 `NLog` 또는 `Serilog`와 같은 라이브러리를 사용하는 것이 매우 효과적입니다. 

### ✨ 코드 왕의 마법 비밀: 좋은 로그 기록은 앱의 '영혼'을 드러냅니다!


*  **분명하고 명확한 메시지:** "이 부분이 오류가 발생했어요!" 보다는 "파일 경로에 문제가 있음! /home/user/data.txt 접근 실패" 와 같은 구체적인 정보를 제공하는 것이 중요합니다.
* **추적 가능성:** 로그 메시지에 함수 이름, 변수 값 등 필요한 정보를 포함하여 문제 해결 과정을 단순화하세요! 🧙‍♂️
* **로그 레벨 설정:** 에러 로그 외에도 정보, 경고 등 다양한 레벨의 로그를 구분하여 유용한 정보만 얻을 수 있도록 합니다.
