##  7강: 배열과 List: 데이터 저장 및 접근 - 우리 C# 개발자의 공장!

**안녕하세요, 코딩 초보자 여러분! 👋** 저는 5년차 주니어 개발자 '코드 마법사'입니다. 이번 강좌에서는 데이터를 저장하고 관리하는 가장 기본적인 요소인 **배열과 List** 에 대해 알아볼 거예요! 🥁

말 그대로, 우리 C# 개발자가 일을 하는 공장은 데이터를 효율적으로 관리해야만 최고의 코드를 만들 수 있어요. 🤔  그래서 배열과 List는 마치 공장에 필요한 스마트 저장소죠! ✨

###  1. 배열: 정렬된 저장 공간 📦


> 생각해보세요, 우리 아파트가 있다고 해요. 각 방은 고유 번호를 가지고 있잖아요? 같은 
> 이론으로 배열도 정해진 순서대로 데이터를 보관하는 저장공간이에요!

```C#
// 변수 선언:  인터넷 검색 결과 저장 용량이 5개입니다. (초보자들은 'int'는 숫자, 'string'은 글자라는 것을 기억하세요!)
string[] 검색결과 = new string[5]; // 배열 생성! 

// 데이터 입력 (저장): 각 방에 사람들을 살 수 있도록 하죠!
검색결과[0] = "코딩 강좌"; 
검색결과[1] = "C# 학습 가이드"; 
검색결과[2] = "개발자 커뮤니티"; 
검색결과[3] = "프로그래밍 프로젝트"; 
검색결과[4] = "코딩 게임";

// 데이터 출력 (조회): 각 방에 살고 있는 사람들을 볼 수 있습니다!
Console.WriteLine(검색결과[0]); // 출력: 코딩 강좌
```

💡 초보자 폭풍 질문!: `new string[5]` 부분이 어떤 의미일까요? 🤔


*   `string[]`:  변수의 타입이 문자열인 배열임을 나타냅니다. (어떤 데이터를 담을지 정합니다!)
*   `new`: 새로운 배열 객체를 생성하는 명령어입니다. 🪄
*   `[5]`: 배열에 저장할 수 있는 최대 크기(요소의 개수)를 정합니다.

>  **배열은 데이터가 순서대로 저장되는 것을 알고 계셨죠? 그렇기 때문에, 특정 위치의 데이터를 찾거나 변경하는 데에는 아주 효율적이죠! 😎**


### 2. List: 스마트한 공장 보관함 ✨

>  하지만 배열은 정해진 크기를 가지는데, 우리는 항상 고정된 크기로만 데이터를 저장할 수 있을까요? 🤔 아닌 거죠! 어쩌면 더 많은 검색 결과가 나오거나 다른 데이터가 추가될 수도 있습니다. 그럴 때 사용하는 것이 바로 List입니다. 

>  List는 배열처럼 데이터를 저장하지만, 크기가 고정되지 않고 필요에 따라 자동으로 확장되는 스마트한 공장 보관함이라고 할 수 있습니다! 😎


```C#
//  List 생성 (크기를 정해줄 필요 없어요!)
List<string> 웹페이지 목록 = new List<string>();

// 데이터 추가: 막대한 검색 결과들을 저장하자!
웹페이지 목록.Add("인터넷 속의 코딩 마법사"); // 추가할 때 'Add' 메서드를 사용합니다!
웹페이지 목록.Add("C# 프로그래머 길잡이"); 
웹페이지 목록.Add("개발자 블로그 - 최신 기술 소식");

// 데이터 출력: 모든 웹페이지 확인해보세요!
foreach (string 페이지 in 웹페이지 목록) {
  Console.WriteLine(페이지); // 루프를 통해 하나씩 출력합니다!
}
```


>  **List는 배열보다 유연하게 데이터를 관리할 수 있고, 쉽게 추가하거나 삭제할 수 있습니다.** 🎉
