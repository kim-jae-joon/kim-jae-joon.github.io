##  💖11강: 클래스 - 객체의 기본 구조! (C# 개발 초보자를 위한 완벽 가이드!) 💖

**안녕하세요, C# 개발 여정을 함께 시작하게 된 여러분!** 지금까지 변수와 데이터 타입에 대한 기초를 다졌으니, 이번 강에서는 **클래스**, 곧 **객체의 기본 구조**에 대해 알아보겠습니다. 🎉

기존에는 단순히 값만 저장하는 변수들을 사용했지만, 클래스는 **변수와 함수가 하나로 모여 있는 "블록"** 같은 존재입니다! 이 블록을 통해 우리는 복잡한 프로그램을 더욱 효율적이고 논리적으로 구현할 수 있습니다. 🧠

###  💖 클래스의 심장부: 필드와 메서드 💖

클래스는 크게 **필드**와 **메서드**로 구성됩니다. 🧐 


* **필드**: 클래스가 가지고 있는 특징이나 속성을 나타내는 변수입니다. 우리는 이를 **"속성"**이라고도 부릅니다. 예를 들어, "개" 클래스의 필드에는 "품종", "나이", "색깔"과 같은 정보들이 있겠죠?
* **메서드**: 클래스가 수행할 수 있는 행동이나 기능을 정의하는 함수입니다. 🤸‍♀️ 이를 통해 우리는 객체를 조작하고, 특정 작업을 시킬 수 있습니다. 예를 들어, "개" 클래스의 메서드에는 "짖다", "배우기", "뛰어놀다"와 같은 행동들이 있겠죠?

#### 👨‍💻 코드로 살펴보자! 👨‍💻

```csharp
// 개 클래스 정의
public class Dog {
  // 필드: 품종, 나이, 색깔
  public string Breed;
  public int Age;
  public string Color;

  // 메서드: 짖는 행동
  public void Bark() {
    Console.WriteLine("멍!");
  }
}
```


* 위 코드에서 'Dog'라는 클래스를 정의했습니다. 💪 이 클래스에는 "품종", "나이", "색깔"이라는 필드와 "짖다" 라는 메서드가 있습니다.

### ✨ 객체 생성: 클래스의 실제화 ✨

클래스를 정의했다고 해서 프로그램에 활용할 수 있는 '실체'가 생기지는 않습니다. 💡 이때 **객체**가 등장합니다!

객체는 클래스의 blueprint을 사용하여 만들어진 구체적인 인스턴스입니다. 마치 레시피(클래스)를 가지고 요리를 할 때, 그대로 먹지 않고 달콤한 케이크 (객체)를 만들어 먹는 것과 같습니다. 🍰


####  👨‍💻 코드로 살펴보자! 👨‍💻

```csharp
// Dog 클래스 인스턴스 생성
Dog myDog = new Dog();

// 필드 설정
myDog.Breed = "골든 리트리버";
myDog.Age = 3;
myDog.Color = "갈색";

// 메서드 호출
myDog.Bark(); // 결과: 멍! 출력
```


* 위 코드에서 'Dog' 클래스를 사용하여 "myDog"이라는 객체를 생성했습니다. ✅ 그리고, "품종", "나이", "색깔" 필드 값을 설정하고 "짖다" 메서드를 호출하여 실제로 멍! 소리를 내게 했다는 의미입니다.

### 🚀 클래스의 활용: 프로그램 구축의 기본! 🚀


클래스는 C#에서 코드를 효율적으로 작성하는 데 필수적인 요소입니다.  😊

* **데이터 가치 유지**: 변수처럼 단순히 값을 저장할 뿐 아니라, **함께 사용되는 데이터들을 모아서 관리**하는 기능이 있습니다.
* **재사용성 향상**: 같은 구조를 여러 번 만들 필요 없이, 하나의 클래스를 재사용하여 코드를 간결하고 효율적으로 작성할 수 있습니다.
