## 🔥18강: 예외 처리 기법 - 코드의 불량품 찾아내는 탐정!🔥

안녕하세요, 대한민국 최고 C# 일타 강사(자신감!)이자 15년 차 시니어 개발자인 **[당신의 이름]**입니다 😎  오늘은 우리 프로그래밍 세상에서 가장 중요한 개념 중 하나인 **예외 처리 (Exception Handling)**에 대해 알아보겠습니다!

### 🤔 코드, 진짜로 말이야? 예외란 무엇일까요?

프로그램을 실행할 때, 원하는 결과를 얻지 못하고 갑자기 멈춰버리는 경우가 있죠. 이럴 때 발생하는 문제들을 **예외 (Exception)** 라고 부릅니다.  
마치 당신이 정시에 버스에 타려고 할 때, 버스가 갑작스럽게 폭발하듯! 😳 🤯

**"버섯은 버그!"** 이렇게 생각하는 개발자도 있겠지만, 예외는 종종 예상하지 못한 상황에서 발생하기 때문에 **코드의 불량품을 찾아내는 탐정 역할**을 합니다.🕵️‍♂️


### 🚨 실무주의보: 예외 처리 없이는 위험해!

예외 처리를 안 한다면?  
* 프로그램이 갑자기 멈춰서 작업 결과가 저장되지 않고 **시간과 노력이 사라진다!!!**  😭 
* 사용자가 이상한 오류 메시지만 보고 **개발자를 질책한다!**😡 
* 심하면 시스템 전체가 **붕괴하는 위험도 있을 수 있다!**😱

### 🎉 예외 처리 기법: 코드의 안전벨트!🎉

예외 처리 기법은 프로그램이 예상치 못한 상황에서도 안정적으로 작동하도록 해주는 **안전벨트** 역할을 합니다. 💪 
C#에서는 `try`, `catch`, `finally` 블록을 사용하여 예외를 다루도록 합니다.


*  `try`:  예상되는 예외가 발생할 수 있는 코드를 작성합니다.

    ```csharp
    int num1 = 10;
    int num2 = 0; 

    try
    {
        int result = num1 / num2; // 얘는 오류 날 거임!
        Console.WriteLine("결과: " + result);  
    }
    catch (Exception ex) 
    {
        Console.WriteLine("오류 발생!: " + ex.Message); 
    }   
    finally
    {
       Console.WriteLine("무슨 일이든 실행됨!");
    }
    ```

*  `catch`: 예외가 발생하면 해당 블록을 실행합니다. `Exception`은 모든 예외 유형을 받아들이는 포괄적인 클래스이며, 더 구체적인 예외 유형을 처리하고 싶다면 (예: `DivideByZeroException`) `catch` 블록의 파라미터를 변경할 수 있습니다.

*  `finally`: `try` 블록 실행 후에 항상 실행되는 블록입니다. 
     자원 정리 작업 (예: 파일 닫기, 연결 해제)을 할 때 유용합니다.


### 💡 초보자 폭풍 질문!

"뭐가 왜 어디서 먼저 실행되냐고?" 🤔  걱정 마세요! 'try', 'catch', 'finally'의 순서는 쉬울 거예요:
* 코드가 `try` 블록에서 오류없이 완료되는 경우, `catch`와 `finally`는 모두 스킵하고 프로그램은 계속 진행됩니다. 😎

* 만약 예외가 발생하면 `catch` 블록으로 이동하여 문제를 해결하고, 그 다음에 `finally` 블록을 실행합니다. 💪 


### 🚀  예외 처리의 마법 - 실무 에피소드! 🚀

회사에서 프로젝트 중 하나에서 사용자 입력 값을 처리하는 부분이 있었습니다. 
만약 사용자가 숫자를 입력하지 않고 문자를 입력하면 오류가 발생할 수 있습니다.  🚨


```csharp
Console.Write("숫자를 입력하세요: ");
string input = Console.ReadLine();

try
{
    int number = int.Parse(input); // 문자를 숫자로 변환하려고!
    Console.WriteLine("입력하신 숫자는 " + number + " 입니다.");
}
catch (FormatException ex)
{
    Console.WriteLine("오류!: 숫자를 입력해주세요. " + ex.Message); // 사용자에게 친절하게 알려줘!
}
```

이렇게 예외 처리 코드를 추가하면, 잘못된 입력을 받았을 때 프로그램이 안정적으로 종료되고 사용자에게도 **친절한 오류 메시지**를 보여주는 멋진 경험을 제공할 수 있습니다. 🤩

### 🤔  요약: 예외 처리 기법


* `try`, `catch`, `finally` 블록을 사용하여 프로그램의 안정성을 높입니다.
* 예상치 못한 오류 발생 시, 프로그램이 원활하게 작동하도록 해줍니다.
* 사용자에게 친절하고 명확한 오류 메시지를 보여주어 개발 경험을 향상시킵니다.
