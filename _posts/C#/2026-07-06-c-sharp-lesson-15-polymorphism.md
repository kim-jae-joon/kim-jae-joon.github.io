##  🤯 15강: 다형성 - 객체의 마술! 여러 모습을 가진 거인 ✨

안녕하세요! C# 주니어 개발자 겸 유머도 리더십도 부족하지만 열정은 넘치는 **저**, **[개발자 이름]**입니다. 오늘부터 새로운 세계로 여행 출발! 🎉  **다형성**이란 신비로운 용어를 다뤄보겠습니다.  충격적인 미래 기술을 설명하는 대신, "마법"과 같은 매력적이고 유쾌한 비유를 통해 쉽게 이해할 수 있도록 노력할게요! 😉

### 🤔 마술사와 객체 - 서로 연결된 두 세계

물론 바쁘시겠지만 제 말 한번 들어주세요! 마술사가 모래알을 쓸어서 사자가 나타나지? 거기에는 그저 단순한 "모래"만 있잖아요. 하지만 마술사의 손길, 특별한 기술이 있어서 모래는 "사자"로 변신할 수 있다죠! 🤩

객체 프로그래밍에서도 마찬가지입니다. 우리가 만든 객체들은 마치 그 모래알처럼 시작합니다. 🐶 고양이, 🚗 자동차,  🏢 건물 등 다양한 형태의 객체가 존재하는데, 이들이 "다형성"을 통해 새로운 모습으로 변신할 수 있습니다!

### ✨ 다형성의 마법 주문 - '자손'을 통해 변신하기

단순히 코드를 늘려서 프로그램만 더 길게 하는 건 아니에요!  **추상 클래스**, **상속**, 그리고 **폴리모르피즘**이라는 세 가지 요소가 이 놀라운 변화를 가능하게 합니다.  마술사는 모래알을 다루고, 우리는 객체들을 다루며 

1. **추상 클래스**: 마술사의 지식 책과 같아요!  모든 객체의 기본적인 속성과 행동을 정의합니다.
    - 예를 들어 `Animal` 추상 클래스는 모든 동물들이 공유하는 "소리 내기"와 "움직이기" 같은 행동들을 정의할 수 있습니다.

2. **상속**: 마술사의 제자들이 지식을 배우고, 자신만의 특별한 기술을 가진 강력한 마술사가 되는 것과 같습니다!  
    - `Dog` 클래스와 `Cat` 클래스는 `Animal` 추상 클래스에서 상속받아 "짖다"와 "꼬리 흔들기" 와 같은 독특한 행동들을 추가할 수 있습니다.

3. **폴리모르피즘**: 마술사가 주문을 외치고, 다양한 효과를 만들어내는 것과 같습니다!  
    - 동일한 함수 호출(마법 주문)이라도 객체의 종류에 따라 다른 결과를 가져온다는 개념입니다. 

### 🚀 실생활 예시 - 다형성을 활용한 건축 현장

다형성은 어떤 코드 구조에서 유용할까요? 🤔  그렇죠! 바로 건축 현장이라면서도, 코딩으로!

- **현장 컨설턴트**: 마술사처럼 모든 도구에 대해 전문 지식을 가지고 있습니다.
- **장비**: 다양한 도구들 (소화기, 볼록거울 등)은 각자의 기능(마법 효과)을 가지고 있습니다.

**다형성은 컨설턴트가 어떤 장비를 사용하든, 동일한 방식으로 제어할 수 있도록 해줍니다.**

```C#
public interface ITool // 추상 클래스 - 인터페이스
{
    void Use(); // 모든 도구가 가지는 기본적인 기능 정의
}

public class Saw : ITool // 톱 - 자손 클래스
{
    public void Use()
    {
        Console.WriteLine("나뭇가지를 베서 나무조각을 만듭니다."); //  톱만 가진 특별한 기능
    }
}

public class Hammer : ITool
{
    public void Use()
    {
        Console.WriteLine("못을 박아 튼튼한 구조물을 만들어요!"); // 망치만 가진 특별한 기능
    }
}

// 주변 상황에 맞춰 필요한 도구를 선택하고 사용하는 코드
ITool tool = new Saw();  // 톱으로 작업할 경우
tool.Use();

tool = new Hammer();   // 망치로 작업할 경우
tool.Use();
```

### ✨ 이제 당신도 마법사! 다형성을 활용해봐요!
