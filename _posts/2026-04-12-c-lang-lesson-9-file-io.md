## 🚀 9강: 파일 입출력: 프로그램과 외부 세상의 소통

### 1. 왜 파일 입출력이 필요할까요? (feat. 우리들의 고민)

여러분, 우리가 지금까지 만들었던 프로그램들을 한번 떠올려볼까요?
변수에 값을 저장하고, 계산하고, 화면에 출력했죠. 아주 잘했어요! 그런데 한 가지 아쉬운 점이 있었습니다.

```c
#include <stdio.h>

int main() {
    int score = 100;
    printf("현재 점수: %d\n", score);
    // 프로그램이 종료되면 score 값은...??
    return 0;
}
```

이 프로그램이 종료되면 `score` 변수에 저장했던 `100`이라는 값은 어떻게 될까요?
네, 맞아요! **사라집니다.** 😭

마치 '포스트잇'에 중요한 메모를 해뒀는데, 컴퓨터를 껐다 켜면 포스트잇이 저절로 사라지는 것과 같아요. 우리가 만든 프로그램은 기본적으로 **휘발성 메모리(RAM)** 위에 올라가기 때문에, 프로그램이 종료되거나 컴퓨터의 전원이 꺼지면 모든 데이터가 사라집니다.

하지만 현실의 프로그램들은 어때요?

*   워드 프로그램으로 작성한 문서는 저장하면 나중에 다시 열어볼 수 있죠?
*   게임에서 저장한 진행 상황은 다음에 다시 시작할 때 그대로 불러올 수 있고요.
*   회원가입한 정보는 나중에 로그인할 때 다시 사용할 수 있죠?

이 모든 것이 바로 **'파일 입출력'** 덕분입니다!
파일 입출력은 프로그램의 데이터를 **하드 디스크(HDD), SSD, USB** 등과 같은 **비휘발성 저장 장치**에 영구적으로 저장하고, 필요할 때 다시 불러올 수 있게 해줍니다.

**결론:** 프로그램이 외부와 소통하고, 데이터를 영구적으로 저장하기 위해 파일 입출력이 필요합니다!

### 2. C 언어에서 파일을 다루는 기본 원리: `FILE` 포인터

C 언어에서 파일을 다룰 때는 `FILE`이라는 특별한 구조체 포인터를 사용합니다.
**`FILE` 포인터**는 여러분이 파일에 접근하기 위한 **'통로' 또는 '열쇠'** 역할을 한다고 생각하시면 됩니다.

마치 도서관에서 책을 빌릴 때 대출증이 필요한 것처럼, C 프로그램이 파일을 읽거나 쓰려면 먼저 `FILE` 포인터를 통해 파일에 대한 접근 권한을 얻어야 합니다.

파일 입출력의 기본적인 흐름은 다음과 같습니다.

1.  **파일 열기 (Open):** 어떤 파일을 어떤 목적으로(읽기/쓰기/추가 등) 사용할지 운영체제에 알리고, `FILE` 포인터를 얻습니다.
2.  **데이터 입출력 (Read/Write):** 얻은 `FILE` 포인터를 사용하여 파일에 데이터를 읽거나 씁니다.
3.  **파일 닫기 (Close):** 파일 사용을 마치면 운영체제에 파일을 반납하고 `FILE` 포인터를 해제합니다. 이 과정은 매우 중요합니다!

### 3. 파일 열기: `fopen()` 함수

파일 입출력의 첫 단계는 바로 파일을 여는 것입니다. `fopen()` 함수를 사용합니다.

```c
FILE *fopen(const char *filename, const char *mode);
```

*   `filename`: 열고자 하는 파일의 이름 (경로 포함 가능).
*   `mode`: 파일을 어떤 목적으로 열 것인지 지정하는 문자열.

#### 3.1. `fopen()` 함수의 모드 (Mode)

`mode`는 파일을 다루는 방식(읽기, 쓰기, 추가 등)을 결정하는 중요한 부분입니다. 다음 표를 통해 다양한 모드를 알아보세요.

| 모드 문자열 | 설명                                                                                                | 비고                                                                                                                                                                                                                                                        |
| :---------- | :-------------------------------------------------------------------------------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `"r"`       | **읽기 (read) 모드.** 파일이 존재해야 하며, 파일 포인터는 파일의 시작점에 위치합니다.               | 파일이 존재하지 않으면 `NULL` 반환                                                                                                                                                                                                                      |
| `"w"`       | **쓰기 (write) 모드.** 파일이 존재하면 내용을 **모두 지우고** 새롭게 씁니다. 파일이 없으면 새로 생성합니다. | **주의!** 기존 내용이 모두 사라지니 조심해야 합니다.                                                                                                                                                                                                    |
| `"a"`       | **추가 (append) 모드.** 파일이 존재하면 파일의 끝에 내용을 추가합니다. 파일이 없으면 새로 생성합니다. | 기존 내용을 유지한 채 뒤에 덧붙일 때 사용합니다.                                                                                                                                                                                                        |
| `"r+"`      | **읽기/쓰기 (read/write) 모드.** 파일이 존재해야 하며, 읽기와 쓰기가 모두 가능합니다.               | 파일 포인터는 파일의 시작점에 위치합니다.                                                                                                                                                                                                               |
| `"w+"`      | **읽기/쓰기 (write/read) 모드.** 파일이 존재하면 내용을 **모두 지우고** 새롭게 씁니다. 파일이 없으면 새로 생성합니다. 읽기와 쓰기가 모두 가능합니다. | `"w"` 모드와 마찬가지로 기존 내용이 모두 사라지니 조심해야 합니다.                                                                                                                                                                                      |
| `"a+"`      | **읽기/추가 (append/read) 모드.** 파일이 존재하면 파일의 끝에 내용을 추가합니다. 파일이 없으면 새로 생성합니다. 읽기와 쓰기가 모두 가능합니다. | 쓰기 작업 시 파일 포인터는 항상 파일의 끝으로 이동합니다. 읽기는 파일 내 어떤 위치에서도 가능합니다.                                                                                                                                                   |
| `...b`      | `b`가 붙으면 바이너리(binary) 모드를 의미합니다. (예: `"rb"`, `"wb"`, `"ab"`)                       | 텍스트 파일과 달리 데이터를 '있는 그대로' 저장하고 읽습니다. Windows 시스템에서는 `LF` (`\n`)를 `CRLF` (`\r\n`)로 자동 변환하지 않는 등 플랫폼 독립적인 처리를 위해 바이너리 모드를 명시하는 것이 좋습니다. UNIX/Linux에서는 텍스트/바이너리 구분이 크게 중요치 않습니다. |

#### 3.2. `fopen()` 예제: 파일 열기 및 에러 처리

`fopen()` 함수는 성공하면 `FILE` 포인터를 반환하고, 실패하면 `NULL`을 반환합니다. 따라서 **반드시 `NULL` 체크를 통해 에러를 처리**해야 합니다!

```c
#include <stdio.h> // 파일 입출력 함수를 사용하기 위해 포함

int main() {
    FILE *fp; // FILE 포인터 선언

    // 1. "새파일.txt" 파일을 "쓰기(w)" 모드로 열기
    // 파일이 없으면 생성하고, 있으면 내용을 지우고 새로 씀
    fp = fopen("새파일.txt", "w"); 

    // 파일 열기에 실패했는지 확인 (가장 중요!)
    if (fp == NULL) {
        printf("파일 열기 실패! ㅠㅠ\n");
        // 에러 발생 시 추가적인 에러 정보를 출력할 수 있습니다.
        // perror("새파일.txt"); 
        return 1; // 비정상 종료를 의미하는 값 반환
    }

    printf("새파일.txt 파일을 성공적으로 열었습니다! (쓰기 모드)\n");

    // 파일을 다 사용한 후에는 반드시 닫아주어야 합니다!
    // (여기서는 일단 열기만 해봤으니 바로 닫습니다.)
    fclose(fp);
    printf("새파일.txt 파일을 닫았습니다.\n");

    // -----------------------------------------------------

    // 2. 존재하지 않는 파일을 "읽기(r)" 모드로 열기 시도
    fp = fopen("없는파일.txt", "r");

    if (fp == NULL) {
        printf("\n'없는파일.txt' 파일 열기 실패! (예상된 결과: 파일이 없어서 실패)\n");
        // perror()는 에러 번호에 해당하는 메시지를 출력해줍니다.
        perror("없는파일.txt"); 
        return 1; 
    }

    printf("'없는파일.txt' 파일을 성공적으로 열었습니다! (읽기 모드) - 이 메시지는 보이지 않을 겁니다.\n");
    fclose(fp);

    return 0; // 정상 종료
}
```

**컴파일 및 실행 결과:**

```
새파일.txt 파일을 성공적으로 열었습니다! (쓰기 모드)
새파일.txt 파일을 닫았습니다.

'없는파일.txt' 파일 열기 실패! (예상된 결과: 파일이 없어서 실패)
없는파일.txt: No such file or directory
```

실행하면 `새파일.txt`라는 빈 파일이 생성되는 것을 확인할 수 있습니다.

### 4. 파일 닫기: `fclose()` 함수

파일을 다 사용했다면 반드시 `fclose()` 함수를 사용하여 파일을 닫아주어야 합니다.

```c
int fclose(FILE *stream);
```

*   `stream`: 닫을 `FILE` 포인터.
*   반환 값: 성공하면 `0`, 실패하면 `EOF` (End Of File)를 반환합니다.

#### 왜 `fclose()`가 중요할까요?

*   **데이터 손실 방지:** 운영체제는 성능 향상을 위해 파일에 바로 데이터를 쓰지 않고, 버퍼(임시 저장 공간)에 모아두었다가 한꺼번에 쓰곤 합니다. `fclose()`를 호출해야 이 버퍼의 내용이 파일에 완전히 기록됩니다.
*   **자원 해제:** `fopen()`으로 확보했던 시스템 자원(파일 핸들 등)을 반환하여 다른 프로그램이 해당 파일을 사용할 수 있게 합니다. 파일을 닫지 않으면 '메모리 누수'처럼 '파일 핸들 누수'가 발생할 수 있습니다.
*   **파일 무결성:** 파일이 제대로 닫히지 않으면 파일 내용이 손상될 가능성이 있습니다.

항상 `fopen()`과 `fclose()`는 한 쌍으로 생각하고, `fopen()`이 성공했다면 반드시 `fclose()`를 호출해야 합니다!

### 5. 텍스트 파일 입출력

이제 파일을 열고 닫는 방법을 알았으니, 실제 데이터를 파일에 쓰고 읽는 방법을 배워볼 차례입니다. 먼저 **텍스트 파일**에 데이터를 입출력하는 방법부터 살펴보겠습니다.

텍스트 파일은 사람이 읽을 수 있는 문자열 형태로 데이터가 저장되는 파일을 의미합니다. (예: `.txt`, `.c`, `.html` 파일 등)

#### 5.1. 문자(char) 단위 입출력: `fputc()`와 `fgetc()`

가장 기본적인 입출력 함수입니다. 한 번에 한 문자씩 파일에 쓰고 읽습니다.

*   `int fputc(int character, FILE *stream);`
    *   `character`: 파일에 쓸 문자 (int형으로 전달).
    *   `stream`: `FILE` 포인터.
    *   반환 값: 성공하면 쓴 문자, 실패하면 `EOF`.

*   `int fgetc(FILE *stream);`
    *   `stream`: `FILE` 포인터.
    *   반환 값: 성공하면 읽은 문자, 파일 끝(EOF)에 도달하거나 실패하면 `EOF`.

---

#### 5.1.1. `fputc()` 예제: 파일에 문자열 쓰기

```c
#include <stdio.h>

int main() {
    FILE *fp;
    char text[] = "안녕하세요, C 파일 입출력 세계에 오신 것을 환영합니다!\n";
    int i = 0;

    // "output.txt" 파일을 "쓰기(w)" 모드로 열기
    fp = fopen("output.txt", "w");

    if (fp == NULL) {
        perror("output.txt");
        return 1;
    }

    printf("파일에 문자열을 쓰고 있습니다...\n");

    // 문자열의 끝(\0)이 나올 때까지 한 문자씩 파일에 쓰기
    while (text[i] != '\0') {
        fputc(text[i], fp); // 한 문자씩 파일에 쓴다.
        i++;
    }

    printf("쓰기 완료! 'output.txt' 파일을 확인해보세요.\n");
    fclose(fp); // 파일 닫기

    return 0;
}
```

**컴파일 및 실행 결과:**

```
파일에 문자열을 쓰고 있습니다...
쓰기 완료! 'output.txt' 파일을 확인해보세요.
```

실행 후 `output.txt` 파일을 열어보면 "안녕하세요, C 파일 입출력 세계에 오신 것을 환영합니다!" 라는 문장이 저장되어 있는 것을 확인할 수 있습니다.

---

#### 5.1.2. `fgetc()` 예제: 파일에서 문자열 읽기

```c
#include <stdio.h>

int main() {
    FILE *fp;
    int ch; // 한 문자를 저장할 변수 (int형으로 선언하는 것이 EOF와 비교하기 용이)

    // "output.txt" 파일을 "읽기(r)" 모드로 열기
    // 이전 예제에서 생성한 파일이 있어야 합니다.
    fp = fopen("output.txt", "r");

    if (fp == NULL) {
        perror("output.txt");
        return 1;
    }

    printf("파일에서 문자열을 읽어오고 있습니다:\n");

    // 파일의 끝(EOF)에 도달할 때까지 한 문자씩 읽어서 화면에 출력
    while ((ch = fgetc(fp)) != EOF) {
        putchar(ch); // 읽은 문자를 화면에 출력
    }

    printf("\n읽기 완료!\n");
    fclose(fp); // 파일 닫기

    return 0;
}
```

**컴파일 및 실행 결과:**

```
파일에서 문자열을 읽어오고 있습니다:
안녕하세요, C 파일 입출력 세계에 오신 것을 환영합니다!
읽기 완료!
```

#### 5.2. 형식화된 입출력: `fprintf()`와 `fscanf()`

`printf()`와 `scanf()`처럼 형식 지정자를 사용하여 다양한 타입의 데이터를 파일에 쓰고 읽을 수 있는 함수입니다. 텍스트 파일을 다룰 때 가장 흔하게 사용됩니다.

*   `int fprintf(FILE *stream, const char *format, ...);`
    *   `stream`: `FILE` 포인터.
    *   `format`: 형식 문자열 (예: "%d %s %f").
    *   `...`: 출력할 변수들.
    *   반환 값: 성공하면 출력된 문자 수, 실패하면 음수.

*   `int fscanf(FILE *stream, const char *format, ...);`
    *   `stream`: `FILE` 포인터.
    *   `format`: 형식 문자열.
    *   `...`: 값을 저장할 변수들의 주소.
    *   반환 값: 성공하면 읽어온 항목 수, 파일 끝(EOF)에 도달하거나 실패하면 `EOF`.

---

#### 5.2.1. `fprintf()` 예제: 파일에 형식화된 데이터 쓰기

```c
#include <stdio.h>

int main() {
    FILE *fp;
    char name[20] = "김철수";
    int age = 25;
    double height = 178.5;

    // "data.txt" 파일을 "쓰기(w)" 모드로 열기
    fp = fopen("data.txt", "w");

    if (fp == NULL) {
        perror("data.txt");
        return 1;
    }

    printf("파일에 데이터 쓰고 있습니다...\n");

    // fprintf를 사용하여 형식에 맞춰 파일에 데이터 쓰기
    fprintf(fp, "이름: %s, 나이: %d, 키: %.1f\n", name, age, height);
    fprintf(fp, "이 프로그램은 파일 입출력 예제입니다.\n");

    printf("쓰기 완료! 'data.txt' 파일을 확인해보세요.\n");
    fclose(fp);

    return 0;
}
```

**컴파일 및 실행 결과:**

```
파일에 데이터 쓰고 있습니다...
쓰기 완료! 'data.txt' 파일을 확인해보세요.
```

실행 후 `data.txt` 파일을 열어보면 다음과 같이 저장되어 있습니다.

```
이름: 김철수, 나이: 25, 키: 178.5
이 프로그램은 파일 입출력 예제입니다.
```

#### 5.2.2. `fscanf()` 예제: 파일에서 형식화된 데이터 읽기

```c
#include <stdio.h>

int main() {
    FILE *fp;
    char name[20];
    int age;
    double height;
    char buffer[100]; // 기타 문자열을 읽어올 버퍼

    // "data.txt" 파일을 "읽기(r)" 모드로 열기
    fp = fopen("data.txt", "r");

    if (fp == NULL) {
        perror("data.txt");
        return 1;
    }

    printf("파일에서 데이터를 읽어오고 있습니다:\n");

    // 첫 번째 줄에서 "이름: %s, 나이: %d, 키: %lf" 형식으로 읽기
    // 주의: %lf는 double형을 읽을 때 사용합니다.
    // fscanf의 반환값이 읽은 항목의 수이므로, 이를 통해 성공 여부를 판단할 수 있습니다.
    if (fscanf(fp, "이름: %s, 나이: %d, 키: %lf\n", name, &age, &height) == 3) {
        printf("이름: %s, 나이: %d, 키: %.1f\n", name, age, height);
    } else {
        printf("첫 번째 줄 데이터 읽기 실패!\n");
    }

    // 나머지 한 줄을 읽어오기 (줄바꿈 문자를 포함하여 읽기 위해 fgets 사용)
    // fgets(버퍼, 버퍼크기, 파일포인터)
    if (fgets(buffer, sizeof(buffer), fp) != NULL) {
        printf("두 번째 줄: %s", buffer);
    } else {
        printf("두 번째 줄 데이터 읽기 실패!\n");
    }

    printf("\n읽기 완료!\n");
    fclose(fp);

    return 0;
}
```

**컴파일 및 실행 결과:**

```
파일에서 데이터를 읽어오고 있습니다:
이름: 김철수, 나이: 25, 키: 178.5
두 번째 줄: 이 프로그램은 파일 입출력 예제입니다.

읽기 완료!
```

**잠깐! `fscanf()` 사용 시 주의할 점!**

*   `fscanf()`는 공백(` `), 탭(`\t`), 줄바꿈(`\n`) 문자를 기준으로 데이터를 구분합니다. 따라서 `이름: 홍 길동`과 같이 이름에 공백이 있으면 `홍`만 읽어올 수 있습니다. 이런 경우에는 `fgets()` 함수로 한 줄 전체를 읽어온 후 `sscanf()`를 사용하여 문자열을 파싱하는 것이 더 안전하고 좋습니다.
*   `scanf()`와 마찬가지로 문자열이 아닌 변수에는 반드시 `&` 주소 연산자를 붙여야 합니다.

### 6. 바이너리 파일 입출력: `fwrite()`와 `fread()`

지금까지는 사람이 읽을 수 있는 **텍스트 파일**을 다루는 방법을 배웠습니다. 하지만 그림 파일, 음악 파일, 실행 파일 등 컴퓨터가 처리하기 위한 **바이너리(이진) 파일**도 있습니다.

바이너리 파일은 데이터를 텍스트 형태로 변환하지 않고, 메모리에 있는 데이터를 '있는 그대로' 비트 단위로 파일에 저장하고 읽습니다. 따라서 텍스트 파일보다 효율적이고, 다양한 구조체나 배열을 직접 저장하는 데 유리합니다.

*   `size_t fwrite(const void *ptr, size_t size, size_t count, FILE *stream);`
    *   `ptr`: 파일에 쓸 데이터가 있는 메모리 시작 주소.
    *   `size`: 하나의 데이터 항목의 크기 (바이트 단위, 보통 `sizeof()` 사용).
    *   `count`: 쓸 데이터 항목의 개수.
    *   `stream`: `FILE` 포인터.
    *   반환 값: 실제로 파일에 쓴 항목의 개수.

*   `size_t fread(void *ptr, size_t size, size_t count, FILE *stream);`
    *   `ptr`: 읽어온 데이터를 저장할 메모리 시작 주소.
    *   `size`: 하나의 데이터 항목의 크기.
    *   `count`: 읽을 데이터 항목의 개수.
    *   `stream`: `FILE` 포인터.
    *   반환 값: 실제로 파일에서 읽은 항목의 개수.

---

#### 6.1. `fwrite()`와 `fread()` 예제: 구조체 배열 저장 및 읽기

구조체 데이터를 바이너리 파일로 저장하고 읽는 것은 `fwrite()`와 `fread()`의 강력한 장점 중 하나입니다.

```c
#include <stdio.h>
#include <string.h> // strcpy를 사용하기 위해 포함

// 학생 정보를 저장할 구조체 정의
typedef struct {
    char name[20];
    int id;
    float score;
} Student;

int main() {
    FILE *fp;
    Student students[3]; // 학생 3명의 정보를 저장할 배열
    Student read_students[3]; // 파일에서 읽어온 정보를 저장할 배열

    // 1. 학생 데이터 초기화
    strcpy(students[0].name, "홍길동");
    students[0].id = 2023001;
    students[0].score = 90.5f;

    strcpy(students[1].name, "김영희");
    students[1].id = 2023002;
    students[1].score = 85.0f;

    strcpy(students[2].name, "이철수");
    students[2].id = 2023003;
    students[2].score = 92.3f;

    // 2. 바이너리 파일에 구조체 배열 쓰기
    // "students.dat" 파일을 "바이너리 쓰기(wb)" 모드로 열기
    fp = fopen("students.dat", "wb"); 
    if (fp == NULL) {
        perror("students.dat (쓰기)");
        return 1;
    }

    printf("학생 데이터를 바이너리 파일에 쓰고 있습니다...\n");
    // students 배열 전체 (Student 구조체 3개)를 파일에 쓴다.
    // sizeof(Student): 하나의 항목 크기
    // 3: 쓸 항목의 개수
    size_t written_count = fwrite(students, sizeof(Student), 3, fp);
    
    if (written_count == 3) {
        printf("쓰기 완료! %zu개의 학생 데이터가 저장되었습니다.\n", written_count);
    } else {
        printf("쓰기 실패 또는 일부만 저장되었습니다. (저장된 개수: %zu)\n", written_count);
    }
    fclose(fp);

    // 3. 바이너리 파일에서 구조체 배열 읽기
    // "students.dat" 파일을 "바이너리 읽기(rb)" 모드로 열기
    fp = fopen("students.dat", "rb");
    if (fp == NULL) {
        perror("students.dat (읽기)");
        return 1;
    }

    printf("\n파일에서 학생 데이터를 읽어오고 있습니다...\n");
    // read_students 배열에 Student 구조체 3개를 파일에서 읽어온다.
    size_t read_count = fread(read_students, sizeof(Student), 3, fp);

    if (read_count == 3) {
        printf("읽기 완료! %zu개의 학생 데이터가 불러와졌습니다.\n", read_count);
        printf("--- 불러온 학생 정보 ---\n");
        for (int i = 0; i < 3; i++) {
            printf("이름: %s, 학번: %d, 점수: %.1f\n", 
                   read_students[i].name, read_students[i].id, read_students[i].score);
        }
    } else {
        printf("읽기 실패 또는 일부만 불러왔습니다. (읽은 개수: %zu)\n", read_count);
        if (ferror(fp)) { // ferror() 함수로 읽기 도중 에러가 발생했는지 확인
            printf("파일 읽기 중 에러 발생!\n");
        } else if (feof(fp)) { // feof() 함수로 파일 끝에 도달했는지 확인
            printf("파일 끝에 도달했습니다.\n");
        }
    }
    fclose(fp);

    return 0;
}
```

**컴파일 및 실행 결과:**

```
학생 데이터를 바이너리 파일에 쓰고 있습니다...
쓰기 완료! 3개의 학생 데이터가 저장되었습니다.

파일에서 학생 데이터를 읽어오고 있습니다...
읽기 완료! 3개의 학생 데이터가 불러와졌습니다.
--- 불러온 학생 정보 ---
이름: 홍길동, 학번: 2023001, 점수: 90.5
이름: 김영희, 학번: 2023002, 점수: 85.0
이름: 이철수, 학번: 2023003, 점수: 92.3
```

**주목할 점:**
`students.dat` 파일을 메모장 같은 텍스트 편집기로 열어보면 알아볼 수 없는 깨진 문자들이 보일 것입니다. 이것이 바로 바이너리 파일의 특징입니다. 데이터가 텍스트 형태가 아닌 컴퓨터가 이해하는 이진 형태로 저장되었기 때문입니다.

### 7. 파일 입출력 시 주의사항 및 에러 처리

파일 입출력은 프로그램 외부의 자원을 다루는 일이기 때문에, 항상 예외 상황에 대비해야 합니다.

1.  **`fopen()` 후 `NULL` 체크는 필수!**
    파일이 존재하지 않거나, 접근 권한이 없거나, 디스크 공간이 부족하는 등 다양한 이유로 파일 열기에 실패할 수 있습니다. `fopen()`의 반환 값을 항상 확인하여 실패 시 적절히 처리해야 합니다.

2.  **`fclose()`는 반드시 호출!**
    파일을 연 후에는 어떤 일이 있어도 꼭 닫아주세요. `fclose()`를 하지 않으면 위에서 설명했듯이 데이터 손실, 자원 누수 등의 문제가 발생합니다.

3.  **`perror()` 함수 활용:**
    파일 입출력 함수가 실패했을 때, `perror("메시지")` 함수를 호출하면 왜 실패했는지에 대한 시스템 오류 메시지를 표준 에러 출력으로 보여줍니다. 디버깅에 큰 도움이 됩니다.

4.  **`feof()`와 `ferror()`:**
    `fgetc()`, `fread()`, `fscanf()` 등의 읽기 함수들이 `EOF`를 반환했을 때, 단순히 파일 끝에 도달해서인지 아니면 읽기 도중 에러가 발생해서인지 구분할 필요가 있습니다.
    *   `feof(FILE *stream)`: 파일의 끝에 도달했으면 0이 아닌 값 반환.
    *   `ferror(FILE *stream)`: 파일 작업 중 에러가 발생했으면 0이 아닌 값 반환.
