## 1. C 언어 개발 환경, 왜 필요할까요?

여러분, 우리가 한글이나 영어를 쓰려면 무엇이 필요하죠? 그렇죠! 연필과 종이, 혹은 컴퓨터의 워드 프로그램 같은 도구가 필요합니다. C 언어도 마찬가지예요. 우리가 작성한 C 언어 코드를 컴퓨터가 이해하고 실행할 수 있도록 변환해주는 '번역가'와, 우리가 코드를 편리하게 작성할 수 있게 도와주는 '작업 공간'이 필요합니다.

이러한 도구들을 통틀어 **'개발 환경(Development Environment)'**이라고 부릅니다.

C 언어 개발에 필요한 핵심 도구들은 다음과 같습니다.

*   **컴파일러 (Compiler):** 우리가 작성한 C 언어 코드를 컴퓨터가 알아들을 수 있는 언어(기계어)로 번역해주는 프로그램이에요. 마치 영어 원문을 한국어로 번역해주는 번역가와 같습니다. C 언어에서는 주로 **GCC**라는 컴파일러를 많이 사용합니다.
*   **코드 편집기 (Code Editor) / IDE (Integrated Development Environment):** 우리가 C 언어 코드를 작성할 수 있는 프로그램입니다. 메모장처럼 단순한 편집기도 가능하지만, 코드를 더 편리하게 작성하고 관리할 수 있도록 도와주는 다양한 기능을 제공하는 편집기(예: Visual Studio Code)나 통합 개발 환경(예: Dev-C++, Code::Blocks, Visual Studio)을 사용하는 것이 일반적입니다.

이 두 가지를 오늘 함께 설치해보고 준비하는 시간을 가질 겁니다!

## 2. 개발 환경 구축: GCC 컴파일러 설치 (MinGW-w64)

먼저 C 언어 코드를 번역해 줄 컴파일러를 설치해봅시다. Windows 운영체제에서는 **MinGW-w64**를 가장 많이 사용합니다. MinGW-w64는 Windows에서 GCC 컴파일러를 사용할 수 있게 해주는 도구라고 생각하시면 돼요.

### 2.1 MinGW-w64 다운로드 및 설치

1.  **MinGW-w64 다운로드 사이트 접속:**
    [https://sourceforge.net/projects/mingw-w64/](https://sourceforge.net/projects/mingw-w64/)

    이 링크로 접속하시면 'Download' 버튼이 보일 거예요. 클릭해서 설치 파일을 다운로드합니다.

2.  **설치 프로그램 실행:**
    다운로드한 `mingw-w64-install.exe` 파일을 실행합니다. 'Next' 버튼을 눌러 계속 진행합니다.

3.  **설정 선택:**
    중요한 단계입니다! 다음 그림과 같이 설정을 선택해주세요.
    (실제 블로그라면 여기에 스크린샷을 첨부할 수 있습니다)

    *   **Version:** 최신 버전 (예: 8.1.0)
    *   **Architecture:** `x86_64` (64비트 운영체제용)
    *   **Threads:** `posix`
    *   **Exception:** `seh` (64비트용)
    *   **Build revision:** 0
    *   'Next' 클릭!

4.  **설치 경로 지정:**
    설치 경로를 지정합니다. 기본 경로는 `C:\Program Files\mingw-w64` 일 텐데, 나중에 환경 변수 설정 시 편리하도록 **`C:\MinGW`** 와 같이 짧고 찾기 쉬운 경로로 변경하는 것을 추천합니다.
    **매우 중요! 경로에 한글이나 공백이 들어가지 않도록 주의하세요!**
    'Next' 클릭하면 설치가 시작됩니다. 설치 완료까지 시간이 조금 걸릴 수 있습니다.

5.  **설치 완료:**
    'Finish' 버튼을 눌러 설치를 마무리합니다.

### 2.2 환경 변수 설정 (PATH)

이 단계는 많은 초보자분들이 어려워하는 부분이지만, 아주 중요하니 저를 따라 차근차근 해봅시다!

우리가 명령 프롬프트(cmd)에서 `gcc`라는 명령어를 입력했을 때, 컴퓨터는 이 명령어가 어디에 있는지 찾아야 합니다. 컴퓨터는 기본적으로 정해진 몇몇 경로에서만 명령어를 찾는데, 우리가 설치한 `gcc`는 그 경로에 없기 때문에, 컴퓨터에게 `gcc`가 있는 위치를 알려주는 작업이 바로 '환경 변수 PATH 설정'입니다.

1.  **'시스템 환경 변수 편집' 열기:**
    *   Windows 검색창에 `시스템 환경 변수 편집` 이라고 입력하고 실행합니다.
    *   또는 `내 PC` (또는 `컴퓨터`) 아이콘에서 마우스 오른쪽 버튼 클릭 -> `속성` -> `고급 시스템 설정` -> `환경 변수` 버튼 클릭.

2.  **환경 변수 창에서 PATH 편집:**
    *   `환경 변수` 창이 뜨면, **`시스템 변수`** 목록에서 `Path`를 찾아 선택한 후 `편집` 버튼을 클릭합니다.
    *   (만약 사용자 변수에도 `Path`가 있다면, `시스템 변수`의 `Path`를 편집하는 것이 일반적입니다.)

3.  **MinGW-w64 bin 폴더 경로 추가:**
    *   `환경 변수 편집` 창에서 `새로 만들기` 버튼을 클릭합니다.
    *   그리고 우리가 MinGW-w64를 설치한 경로의 `bin` 폴더를 추가합니다.
        예시: **`C:\MinGW\bin`** (만약 `C:\MinGW`에 설치했다면)
    *   추가한 뒤, `확인` 버튼을 여러 번 눌러 모든 창을 닫습니다.

    (실제 블로그라면 여기에 환경 변수 편집 창 스크린샷과 추가 예시를 넣을 수 있습니다.)

    **[표] 환경 변수 PATH 설정 요약**

| 단계             | 설명                                           |
| :--------------- | :--------------------------------------------- |
| 1. `시스템` 검색 | `시스템 환경 변수 편집` 실행                   |
| 2. `환경 변수`   | `고급` 탭 -> `환경 변수` 버튼 클릭             |
| 3. `Path` 선택   | `시스템 변수` 목록에서 `Path` 선택 후 `편집`   |
| 4. `새로 만들기` | `C:\MinGW\bin` (설치 경로에 맞게) 추가 후 `확인` |

### 2.3 설치 확인

환경 변수 설정까지 마쳤다면, 이제 `gcc` 명령어가 잘 작동하는지 확인해볼 시간입니다!

1.  **새로운 명령 프롬프트(cmd) 실행:**
    **아주 중요!** 기존에 열려있던 명령 프롬프트가 있다면 **반드시 닫고 새로 실행**해야 합니다. 환경 변수는 새로 시작된 프로세스에만 적용되기 때문입니다.
    Windows 검색창에 `cmd`라고 입력하고 `명령 프롬프트`를 실행합니다.

2.  **`gcc -v` 명령어 입력:**
    명령 프롬프트 창에 다음 명령어를 입력하고 `Enter`를 누릅니다.

    ```bash
    gcc -v
    ```

3.  **결과 확인:**
    만약 다음과 비슷한 내용이 주루룩 출력된다면, 성공적으로 설치된 겁니다!

    ```
    Using built-in specs.
    COLLECT_GCC=gcc
    COLLECT_LTO_WRAPPER=c:/mingw/bin/../libexec/gcc/x86_64-w64-mingw32/8.1.0/lto-wrapper.exe
    Target: x86_64-w64-mingw32
    Configured with: ...
    Thread model: posix
    gcc version 8.1.0 (mingw-w64 x86_64-posix-seh-gcc-8.1.0)
    ```

    만약 `'gcc'은(는) 내부 또는 외부 명령, 실행할 수 있는 프로그램, 또는 배치 파일이 아닙니다.` 와 같은 오류 메시지가 나온다면, 위의 환경 변수 설정 단계를 다시 한 번 꼼꼼히 확인해주세요. 오타나 경로 오류가 있을 수 있습니다.

## 3. 코드 편집기 설치: Visual Studio Code

컴파일러는 준비되었으니, 이제 우리가 코드를 아름답게 작성할 '작업 공간'을 만들어봅시다. 저는 강력하고 편리하며, 많은 프로그래머들이 사랑하는 **Visual Studio Code (VS Code)**를 추천합니다.

### 3.1 Visual Studio Code 다운로드 및 설치

1.  **VS Code 다운로드 사이트 접속:**
    [https://code.visualstudio.com/](https://code.visualstudio.com/)

    접속 후 여러분의 운영체제에 맞는 버전을 다운로드합니다 (대부분 Windows겠죠?).

2.  **설치 프로그램 실행:**
    다운로드한 `VSCodeUserSetup-x64-버전.exe` (또는 비슷한 이름) 파일을 실행합니다.
    *   `동의함` 선택 후 `다음`
    *   설치 경로를 기본값으로 두고 `다음`
    *   시작 메뉴 폴더도 기본값으로 두고 `다음`
    *   **`바탕 화면 바로 가기 만들기`**, **`Code를 사용하여 파일/폴더 열기`** 옵션 등을 모두 체크하고 `다음`
    *   `설치` 버튼 클릭!

3.  **설치 완료:**
    `Visual Studio Code 시작`에 체크된 상태로 `마침`을 누르면 VS Code가 실행됩니다.

### 3.2 C/C++ 확장팩 설치

VS Code는 다양한 언어를 지원하지만, C 언어 개발을 위해선 관련 기능을 더 편리하게 사용할 수 있도록 **확장팩(Extension)**을 설치하는 것이 좋습니다.

1.  **VS Code 실행:**
    VS Code를 실행합니다.

2.  **확장 탭 클릭:**
    왼쪽 사이드바에서 네모 박스 4개가 겹쳐있는 아이콘 (`Extensions` 또는 `확장`)을 클릭합니다. (단축키: `Ctrl+Shift+X`)

3.  **C/C++ 확장 검색 및 설치:**
    *   검색창에 `C/C++`이라고 입력합니다.
    *   `Microsoft`에서 제공하는 **`C/C++` 확장팩**을 찾아서 `Install` 버튼을 클릭합니다.
        (실제 블로그라면 여기에 VS Code 확장팩 설치 스크린샷을 넣을 수 있습니다.)

설치 완료! 이제 여러분은 C 언어 코드를 작성하고 컴파일할 수 있는 완벽한 환경을 갖추게 되었습니다. 축하합니다! 🎉

## 4. Hello, C! 첫 프로그램 만들기

드디어 기다리고 기다리던 첫 번째 C 언어 프로그램을 만들어 볼 시간입니다! 전 세계 프로그래머들이 새로운 언어를 배울 때 가장 먼저 만드는 프로그램, 바로 "Hello, World!"입니다. 우리는 C 언어니까 "Hello, C!"라고 외쳐봅시다!

### 4.1 새 파일 생성 및 코드 작성

1.  **작업 폴더 만들기:**
    우리가 만든 C 언어 코드를 저장할 폴더를 하나 만들어둡시다. 예를 들어 `C:\my_c_project`와 같은 이름으로 만들어두면 좋습니다.

2.  **VS Code에서 폴더 열기:**
    *   VS Code에서 `File` (파일) -> `Open Folder...` (폴더 열기...)를 클릭합니다.
    *   방금 만든 `C:\my_c_project` 폴더를 선택하고 `폴더 선택`을 클릭합니다.
    *   (처음 열면 보안 경고가 뜰 수 있는데, `Yes, I trust the authors`를 눌러주세요.)

3.  **새 파일 생성:**
    *   VS Code 왼쪽 탐색기에서 폴더 이름 옆에 있는 `New File` (새 파일) 아이콘을 클릭합니다. (종이 모양 아이콘)
    *   파일 이름을 **`hello.c`** 라고 입력하고 `Enter`를 누릅니다.
        C 언어 파일은 항상 확장자가 `.c`로 끝납니다.

4.  **코드 작성:**
    `hello.c` 파일에 다음 코드를 똑같이 입력합니다. (오타 없이!)

    ```c
    #include <stdio.h> // 표준 입출력 라이브러리 포함

    int main() {
        printf("Hello, C!\n"); // "Hello, C!" 문자열을 화면에 출력
        return 0; // 프로그램이 성공적으로 종료되었음을 알림
    }
    ```

    코드를 작성하고 나면 `Ctrl+S`를 눌러 저장하는 것을 잊지 마세요!

### 4.2 코드 한 줄 한 줄 뜯어보기

이제 우리가 작성한 코드가 무슨 의미인지 한 줄 한 줄 자세히 살펴봅시다.

```c
#include <stdio.h> // 표준 입출력 라이브러리 포함
```

*   `#include`: '포함하다'라는 뜻입니다. 특정 파일을 우리 프로그램에 가져다 쓰겠다는 의미예요.
*   `<stdio.h>`: 'Standard Input Output header'의 약자입니다. C 언어에서 화면에 글자를 출력하거나, 키보드로부터 값을 입력받는 등 기본적인 입출력 기능을 사용하기 위해 필요한 기능들이 담겨 있는 파일입니다. 이 파일을 포함해야 `printf`와 같은 함수를 사용할 수 있어요.
*   `// ...`: 쌍 슬래시 (`//`) 뒤에 나오는 내용은 컴퓨터가 무시합니다. 이것을 **주석(Comment)**이라고 부르며, 코드를 설명하거나 나중에 내가 다시 봤을 때 쉽게 이해할 수 있도록 메모를 남기는 용도로 사용합니다.

```c
int main() {
    // ...
    return 0;
}
```

*   `int main()`: 모든 C 언어 프로그램은 반드시 **`main` 함수**를 가지고 있어야 합니다. 이 `main` 함수가 프로그램의 **시작점**이에요! 컴퓨터는 프로그램을 실행할 때 무조건 `main` 함수부터 찾아서 실행합니다.
    *   `int`: `main` 함수가 실행을 마치면 정수(Integer) 값을 반환하겠다는 의미입니다.
    *   `()`: 함수 이름 뒤에는 항상 괄호가 옵니다.
    *   `{ }`: 중괄호(`{ }`) 안에 있는 내용이 `main` 함수가 수행할 실제 작업들입니다.

```c
printf("Hello, C!\n"); // "Hello, C!" 문자열을 화면에 출력
```

*   `printf`: `stdio.h`에 포함된 함수 중 하나로, **화면에 문자열을 출력하는 역할**을 합니다. 마치 "프린트해줘!" 라고 명령하는 것과 같아요.
*   `("Hello, C!\n")`: `printf` 함수에게 "Hello, C!"라는 글자를 출력하라고 전달하는 내용입니다.
    *   `"`: 쌍따옴표(`"`) 안에 있는 내용은 **문자열(String)**이라고 부릅니다.
    *   `\n`: 이것은 특별한 문자입니다. '역슬래시(백슬래시) n'이라고 읽으며, **줄 바꿈(New Line)**을 의미해요. `printf`가 "Hello, C!"를 출력한 뒤에 다음 줄로 커서를 옮겨줍니다. 이게 없으면 다음에 출력되는 내용이 "Hello, C!" 바로 옆에 붙어서 나올 거예요.
*   `;`: 모든 C 언어 명령문의 끝에는 **세미콜론(Semicolon)**을 붙여야 합니다. "여기까지가 한 명령의 끝이야!"라고 알려주는 약속이죠. 이걸 빼먹으면 컴파일 에러가 발생합니다.

```c
return 0; // 프로그램이 성공적으로 종료되었음을 알림
```

*   `return 0;`: `main` 함수가 모든 작업을 마치고 운영체제에게 **`0`**이라는 값을 돌려주겠다는 의미입니다. C 언어에서는 보통 `0`을 반환하면 **'프로그램이 아무 문제 없이 성공적으로 종료되었다'**는 뜻으로 약속되어 있습니다. 만약 `0`이 아닌 다른 값이 반환되면 '뭔가 문제가 발생했구나'라고 해석할 수 있어요.

어때요? 이제 코드가 조금은 친숙하게 느껴지나요? 처음에는 낯설겠지만, 계속해서 보다 보면 익숙해질 거예요!

### 4.3 컴파일 및 실행

이제 우리가 작성한 C 언어 코드를 컴퓨터가 알아들을 수 있는 프로그램(실행 파일)으로 만들고, 직접 실행해봅시다!

1.  **터미널 열기:**
    *   VS Code에서 `Terminal` (터미널) -> `New Terminal` (새 터미널)을 클릭합니다.
    *   또는 `Ctrl+~` (물결표) 키를 누르면 터미널이 열립니다.
    *   터미널 창이 VS Code 아래쪽에 열리고, 현재 우리가 연 폴더(`C:\my_c_project`)의 경로로 자동 이동되어 있을 겁니다.

2.  **컴파일:**
    터미널에 다음 명령어를 입력하고 `Enter`를 누릅니다.

    ```bash
    gcc hello.c -o hello
    ```

    *   `gcc`: 우리가 설치한 C 컴파일러를 실행하는 명령어입니다.
    *   `hello.c`: 컴파일할 C 소스 코드 파일 이름입니다.
    *   `-o hello`: **O**utput (출력) 파일 이름을 `hello`로 지정하겠다는 의미입니다. 이 옵션을 사용하지 않으면 기본적으로 `a.exe` (또는 `a.out` on Linux/macOS)라는 이름으로 실행 파일이 만들어집니다.
    *   **잠시 기다리면 아무 메시지 없이 다음 줄로 넘어갈 겁니다.** 이것은 컴파일이 **성공적**으로 완료되었다는 뜻이에요! 만약 에러 메시지가 나온다면, `hello.c` 파일에 오타가 없는지 다시 한 번 꼼꼼히 확인해주세요.

    컴파일이 완료되면, `C:\my_c_project` 폴더에 `hello.exe`라는 새로운 파일이 생성되었을 겁니다. 이게 바로 우리가 만든 실행 프로그램이에요!

3.  **실행:**
    이제 컴파일된 프로그램을 실행해봅시다. 터미널에 다음 명령어를 입력하고 `Enter`를 누릅니다.

    ```bash
    ./hello
    ```
    (Windows에서는 `hello.exe` 또는 그냥 `hello`만 입력해도 됩니다.)

4.  **결과 확인:**
    짠! 터미널에 **`Hello, C!`** 라는 메시지가 출력되는 것을 볼 수 있을 겁니다!

    ```
    Hello, C!
    ```

성공적으로 "Hello, C!"를 외치셨다면 진심으로 축하드립니다! 여러분은 이제 어엿한 C 프로그래머의 첫걸음을 떼셨습니다. 🎉
