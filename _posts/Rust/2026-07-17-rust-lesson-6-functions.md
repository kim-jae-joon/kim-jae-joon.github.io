# 6강: Rust 기초 - 함수 정의와 호출: 코딩의 마법을 풀다!

안녕하세요, 여러분! Rust의 세계로 다시 한번 빠져들 준비가 되셨나요? 오늘은 **함수 정의와 호출**에 대해 배워볼 거예요. 이 주제는 마치 요리 레시피처럼, 코드를 재사용하고 효율적으로 관리하는 데 핵심적인 역할을 합니다. 진짜 신기하죠? 그럼, 지금부터 차근차근 시작해볼게요!

## 왜 함수가 중요한가요?

함수는 코드의 재사용성을 극대화하고, 복잡한 작업을 작은 단위로 나누어 관리할 수 있게 해줍니다. 마치 주방에서 각 재료를 준비하는 단계와 조리하는 단계를 나누는 것과 비슷해요. 이렇게 하면 코드가 훨씬 더 읽기 쉽고 유지보수하기도 편해집니다.

### 💡 초보자 폭풍 질문!
**Q: 함수를 안 쓰면 어떻게 되나요?**  
**A:** 함수 없이 모든 로직을 한 곳에 쓰면 코드가 길어지고 복잡해져서 오류 찾기가 어려워집니다. 마치 모든 요리 재료를 한 그릇에 섞어 넣고 조리하는 것과 같아요 – 결과는 예측 불가능할 수 있죠!

## 함수 정의하기: 첫걸음

Rust에서 함수를 정의하는 기본 구조는 다음과 같습니다:

```rust
fn 함수이름(매개변수1: 타입, 매개변수2: 타입) -> 반환타입 {
    // 함수 본문
    ...
    반환값
}
```

### 예제 1: 간단한 함수 정의

```rust
fn greet(name: &str) -> String {
    // 문자열 포맷팅을 사용해 인사말 생성
    format!("안녕하세요, {}님!", name)
}

fn main() {
    let welcome_message = greet("초보자");
    println!("{}", welcome_message); // 출력: 안녕하세요, 초보자님!
}
```

**설명:**
- `fn greet(name: &str)`: `greet`라는 이름의 함수를 정의하고, `name`이라는 매개변수를 `&str` (문자열 참조) 타입으로 받습니다.
- `-> String`: 이 함수는 `String` 타입의 값을 반환합니다.
- `format!` 매크로: 문자열 포맷팅을 통해 동적으로 메시지를 생성합니다.
- `main` 함수에서 `greet` 호출 후 결과를 출력합니다.

## 함수 호출: 마법을 실행하다

함수를 정의한 후에는 실제로 호출해서 사용해야죠! 함수 호출은 단순히 함수 이름 뒤에 괄호를 붙여 매개변수를 넘기는 것만으로 이루어집니다.

### 예제 2: 매개변수와 반환값 활용

```rust
fn add(a: i32, b: i32) -> i32 {
    // 두 수를 더하고 결과 반환
    a + b
}

fn main() {
    let sum = add(5, 3); // 함수 호출
    println!("두 수의 합: {}", sum); // 출력: 두 수의 합: 8
}
```

**설명:**
- `fn add(a: i32, b: i32) -> i32`: 두 정수를 받아 정수를 반환하는 함수입니다.
- `add(5, 3)`: 함수를 호출하며 `5`와 `3`을 매개변수로 넘깁니다.
- 결과는 `sum` 변수에 저장되고 출력됩니다.

## 다양한 함수 형태: 깊이 파고들기

함수는 다양한 형태로 사용될 수 있어요. 몇 가지 주요 형태를 살펴보겠습니다.

### 1. **매개변수 없는 함수**

```rust
fn say_hello() -> String {
    "Hello, Rust!".to_string()
}

fn main() {
    let msg = say_hello();
    println!("{}", msg); // 출력: Hello, Rust!
}
```

**설명:**
- 매개변수가 없으므로 괄호 안에 아무것도 넣지 않습니다.
- 단순히 메시지를 반환하는 함수입니다.

### 2. **반환 타입이 없는 함수 (단, `()`로 표현)**

```rust
fn display_info() {
    println!("Rust의 세계에 오신 것을 환영합니다!");
}

fn main() {
    display_info(); // 반환값이 없으므로 호출 후 결과 처리 필요 없음
}
```

**설명:**
- `-> ()`로 표현되지만, Rust에서 명시적으로 쓰지 않아도 됩니다.
- 출력만 수행하고 반환값이 없는 경우에 적합합니다.

### 💡 초보자 폭풍 질문!
**Q: 함수 호출 시 매개변수를 빠뜨리면 어떻게 될까요?**  
**A:** 컴파일 타임 에러가 발생합니다. Rust는 타입 안전성을 매우 중요하게 생각하므로, 정의된 매개변수를 모두 제공해야 합니다. 마치 레시피에 필요한 재료를 하나라도 빠뜨리면 요리가 제대로 되지 않는 것과 같아요!

## 실무에서의 함수 활용: 🚨 실무주의보

### 복잡한 로직 분리하기

실무에서는 복잡한 로직을 여러 함수로 나누는 것이 필수적입니다. 예를 들어, 사용자 인증 프로세스를 생각해보세요.

```rust
fn validate_credentials(username: &str, password: &str) -> bool {
    // 간단한 검증 로직 예시
    username == "admin" && password == "secure123"
}

fn authenticate_user(username: &str, password: &str) -> Result<String, String> {
    if validate_credentials(username, password) {
        Ok(format!("{}님, 환영합니다!", username))
    } else {
        Err("인증 실패: 잘못된 사용자 이름 또는 비밀번호".to_string())
    }
}

fn main() {
    match authenticate_user("admin", "secure123") {
        Ok(message) => println!("{}", message), // 성공 시 메시지 출력
        Err(error) => println!("{}", error),   // 오류 시 오류 메시지 출력
    }
}
```

**설명:**
- `validate_credentials`: 인증 로직의 일부를 분리하여 검증합니다.
- `authenticate_user`: 전체 인증 프로세스를 관리하며 결과를 `Result` 타입으로 반환합니다.
- `match` 구문을 사용해 결과에 따라 다르게 처리합니다.

### 실무주의보!
- **함수 분리**: 복잡한 로직을 작은 함수로 나누어 코드의 가독성을 높이세요.
- **타입 안전성**: 매개변수와 반환 타입을 정확히 정의하여 런타임 오류를 최소화하세요.
