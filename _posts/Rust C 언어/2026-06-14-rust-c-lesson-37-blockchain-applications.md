이제, **실제 코드로 블록체인의 신비를 풀어보죠!**

### 1. 블록체인 기본 구조 만들기: 블록 생성

블록체인의 핵심은 **블록**이라는 단위입니다. 각 블록에는 거래 데이터, 이전 블록의 해시 값 등이 담겨있어요. 다음 코드는 간단한 블록 구조를 정의하는 예시입니다.

```c
#include <stdio.h>
#include <string.h>

// 블록 구조체 정의
typedef struct {
    unsigned long long timestamp; // 생성 시간
    unsigned char data[100];   // 거래 데이터 (예시)
    unsigned char previousHash[32]; // 이전 블록 해시 (예시)
    unsigned char hash;         // 현재 블록 해시 (예시)
} Block;

int main() {
    // 새로운 블록 생성
    Block block;

    // 예시 데이터 설정
    block.timestamp = time(NULL); // 현재 시간으로 설정
    strcpy((char*)block.data, "거래 데이터 예시"); // 거래 내용
    memset(block.previousHash, 0, sizeof(block.previousHash)); // 이전 블록 해시 초기화 (생략)

    // 해시 함수 (간단 예시, 실제에는 SHA-256 등 사용)
    // ... (해시 함수 구현 생략)

    printf("생성된 블록:\n");
    printf("Timestamp: %llu\n", block.timestamp);
    printf("Data: %s\n", (char*)block.data);
    printf("Hash: ");
    for (int i = 0; i < sizeof(block.hash); i++) {
        printf("%02x", block.hash[i]); // 해시 값 출력 (예시)
    }
    printf("\n");

    return 0;
}
```

**해설**:

- **구조체** `Block`: 블록의 주요 정보를 담는 컨테이너 역할을 합니다.
- **timestamp**: 블록 생성 시각을 기록합니다. 시간 동기화가 중요해요!
- **data**: 실제 거래 정보를 담는 공간입니다. 여기에는 텍스트뿐 아니라 금액, 주소 등 다양한 데이터가 들어갈 수 있어요.
- **previousHash**: 이전 블록의 해시 값을 저장하여 **블록 연결**을 보장합니다. 마치 연쇄처럼요!
- **hash**: 현재 블록의 고유한 해시 값입니다. 이 값이 블록의 **지문** 역할을 합니다. 실제 구현에서는 강력한 해시 함수 (예: SHA-256)를 사용해야 합니다.

**💡 초보자 폭풍 질문!**: 해시 함수가 왜 중요한가요?

**답변**: 해시 함수는 데이터를 고정된 길이의 고유한 문자열로 변환하는 역할을 해요. 블록체인에서는 이 해시 값이 **데이터 무결성**을 보장하는 핵심 역할을 합니다. 만약 데이터가 조금이라도 변경되면 해시 값이 완전히 달라지기 때문에, 데이터 조작을 쉽게 감지할 수 있답니다!

### 2. 블록 연결: 체인 구축

블록들이 서로 연결되어 블록체인을 형성하는 과정을 보여줍니다.

```c
#include <stdio.h>
#include <string.h>

// ... (이전 블록 구조체 코드 참조)

typedef struct {
    Block blocks[10]; // 초기 블록들 (예시)
    int size;         // 현재 블록 개수
} Blockchain;

void addBlock(Blockchain* chain, Block newBlock) {
    // 새로운 블록 크기에 맞게 배열 확장 (간단 예시, 실제에서는 동적 메모리 할당 고려)
    if (chain->size >= sizeof(chain->blocks)/sizeof(Block)) {
        printf("블록 체인 용량 초과!\n");
        return;
    }

    chain->blocks[chain->size] = newBlock;
    chain->size++;

    // 이전 블록 해시 업데이트 (간단 예시)
    // ... (코드 생략)

    printf("블록 추가 완료!\n");
}

int main() {
    Blockchain myChain;
    myChain.size = 0;

    // 초기 블록 생성 (예시)
    Block genesisBlock = {0, "", NULL}; // 초기 블록 설정 (간단 예시)
    addBlock(&myChain, genesisBlock);

    // 새로운 블록 생성 및 추가
    Block newBlock = {time(NULL), "새로운 거래 데이터", NULL}; // ... (해시 계산 로직 생략)
    addBlock(&myChain, newBlock);

    printf("블록체인 상태:\n");
    for (int i = 0; i < myChain.size; i++) {
        printf("블록 %d:\n", i);
        printf("  Timestamp: %llu\n", myChain.blocks[i].timestamp);
        printf("  Data: %s\n", (char*)myChain.blocks[i].data);
        // ... (해시 출력 로직 생략)
    }

    return 0;
}
```

**해설**:

- **Blockchain 구조체**: 여러 블록을 저장하는 컨테이너 역할을 합니다.
- **addBlock 함수**: 새로운 블록을 블록체인에 추가하는 함수입니다. 여기서는 간단하게 배열에 블록을 추가하는 방식을 보여주지만, 실제 시스템에서는 메모리 관리 및 효율성을 고려한 동적 할당 방식이 필요합니다.
- **이전 블록 해시 업데이트**: 새로운 블록 생성 시 이전 블록의 해시 값을 활용하여 현재 블록의 해시를 계산하는 중요한 과정이 생략되었지만, 이는 블록체인의 핵심 **안전성**을 보장하는 핵심 부분입니다.
