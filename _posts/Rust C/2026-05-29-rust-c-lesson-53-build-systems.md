##  🦀 53강: Rust의 빌드 시스템과 구성 파일 이해하기: Cargo와 Make 사용법! 🚀

🔥 마치 레스토랑에서 요리사에게 주문을 하듯이, Rust 프로그램을 만들 때 필요한 모든 것을 준비해 줄 '주방장' 역할을 해줘요!  고급 레시피를 보면 재료 목록과 조리 과정이 나와있죠? 컴퓨터도 동일하게요!


### 💯 Cargo: Rust 개발의 최상급 도구

Cargo는 Rust 프로그램을 만들 때 필수적인 요소로, 프로젝트 관리자, 의존성 관리자, 및 빌드 시스템의 역할을 담당하는 **진짜 스파이더맨** 같은 존재랍니다! 😎  Rust 개발 생활에서 Cargo를 사용하지 않는 건 마치 우주여행에 지도 없이 날아다니는 것과 같아요. 😅

#### 💡 초보자 폭풍 질문!
* "Cargo가 무엇을 하는 거죠?"
* "왜 Rust 프로그램 만들 때 Cargo 필요해?"
* "단순히 코드만 쓰면 되는데, 이런 복잡한 도구까지?!"

진정하세요, 진짜 그렇게 어렵지 않습니다. 걱정 마세요!  저는 친절하게 설명할게요. 😊  Cargo의 역할을 간단히 정리하면 다음과 같습니다:

1️⃣ **프로젝트 관리:** Cargo를 사용하면 Rust 프로젝트를 효율적으로 관리할 수 있어요. 파일, 폴더 등 모든 것을 관리하고 깔끔하게 정리해주는 거죠!
2️⃣ **의존성 관리:** Rust 프로그램은 다른 라이브러리를 활용하여 더욱 강력한 기능을 가질 수 있습니다. Cargo는 필요한 라이브러리를 자동으로 다운로드하고 설치해주면서 개발 시간을 절약하는 꿀 도구예요!
3️⃣ **빌드 시스템:** 코드를 실행 가능한 프로그램으로 변환하는 것은 Rust의 핵심 과정이에요.  Cargo는 이러한 빌드 프로세스를 자동화하여 우리가 복잡한 설정을 하지 않고도 프로그램을 만들 수 있도록 도와주세요!

#### 🚀 Cargo 사용법: 간단하게 배우기!
 
`cargo new my_project` 명령어로 새로운 Rust 프로젝트를 생성할 수 있어요.  이렇게 하면 'my_project'라는 이름의 폴더가 만들리고, 내부에는 다음과 같은 파일들이 존재하게 됩니다:

- `Cargo.toml`: Cargo는 이 파일을 통해 프로젝트 정보와 의존성을 관리합니다.
- `src/main.rs`: Rust 코드를 작성하는 곳입니다!


```
// src/main.rs
fn main() {
    println!("Hello, world!"); 
}
```

이 `main.rs` 파일에서 `println!()` 함수는 화면에 "Hello, world!" 문자열을 출력합니다.  

* `cargo run`: 이 명령어를 실행하면 Cargo가 빌드 작업을 수행하고, 결과적으로 'Hello, world!'라는 메시지가 화면에 표시됩니다.   이때 실행되는 프로그램은 `target/debug/my_project` 에 존재하는 파일이라고 할 수 있습니다.

#### 🚨 실무주의보:
* Cargo는 Rust 개발의 필수 도구이며 숙달해야 합니다!


### ⚙️ Make: 자동화 마법사

Make는  **자동화의 신**입니다!  특정 작업을 명령어로 정의하고, 필요한 경우 자동으로 실행하는 기능을 가지고 있습니다. Rust에서도 Make를 사용하여 복잡한 빌드 프로세스를 간소화할 수 있습니다.


#### ✨ Makefile: 자동화 마법의 지침서

Make는 `Makefile` 이라는 파일을 사용하여 자동화 작업을 정의합니다.  이 파일에 특정 명령어와 해당하는 작업들을 정의하고, `make` 명령어를 통해 자동으로 실행시키면됩니다. 


#### 🚀 Make 사용법: Rust 프로그램에 적용하기

Rust에서는 Cargo가 주로 사용되지만, Make도 간단한 프로젝트에서 유용하게 활용할 수 있습니다.   예를 들어, 다음과 같은 `Makefile`을 작성하면 'hello_world'라는 파일을 컴파일하는 작업을 자동화 할 수 있습니다:


```makefile
TARGET = hello_world

all: $(TARGET)

$(TARGET): main.rs
	rustc main.rs -o $@

clean:
	rm -f $(TARGET)
```



* `TARGET`:  빌드할 프로그램의 이름을 지정합니다. 
* `all`: 모든 작업을 수행하는 명령어입니다.
* `$(TARGET)` : 타겟 프로그램명 (예: hello_world) 을 표시합니다.


* `main.rs`: Rust 코드를 작성하는 파일을 의미합니다.
*  `rustc main.rs -o $@`:  Rust 컴파일러(rustc)를 사용하여 `main.rs`를 빌드하고, 이름은 `$@`(타겟 프로그램명) 로 지정합니다.
* `clean`: 빌드된 파일 (`$(TARGET)`) 을 삭제하는 명령어입니다.



   이와 같이 Makefile을 작성하면 'make' 명령어를 사용하여 코드를 컴파일하고, 'make clean' 명령어를 사용하여 결과물을 삭제할 수 있습니다.

#### 💡 초보자 폭풍 질문!
* "왜 Cargo 대신 Make를 사용해야 할까요?"


Make는 Rust 개발에서 항상 필수적인 도구가 아니지만, 복잡한 프로젝트에서 자동화 기능을 활용하여 개발 효율성을 높일 수 있습니다.  

###  总结： 

Cargo와 Make는 Rust 프로그램을 만들 때 매우 유용한 도구입니다!  Cargo는 Rust의 모든 요소를 관리하는 강력한 도구이며, Make는 특정 작업들을 자동화하여 개발 프로세스를 효율적으로 만들어줍니다. 😊
