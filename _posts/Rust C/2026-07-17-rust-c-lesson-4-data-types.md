## 🔥 Rust의 자료형 이해: 정수, 실수 및 문자열 ⚡

안녕하세요! 저는 대한민국 최고의 Rust 강사이자 15년차 시니어 개발자 '코딩 비타민' 입니다! 😎 오늘은 Rust에서 가장 기본적인 구성 요소인 '자료형'에 대해 알아볼 거예요. 자료형은 코딩하는 건축장애인을 생각하면 이해가 더 쉬워요.  🧱 벽돌, 도로, 창문처럼 다양한 종류의 블록들이 필요하죠? Rust에서도 '정수', '실수', '문자열' 등 각각 다른 용도에 사용되는 자료형들이 존재해요!

### 💡 초보자 폭풍 질문!

> 저는 코딩 처음 시작하는데, 자료형이 정말 어려워요. 🤔  왜 여러 종류가 필요하죠? 

바로 그 질문 똑똑하게 내놓으셨네요! 🎉 다양한 자료형은 각자의 성질과 용도에 맞게 데이터를 처리하는 데 도움을 주기 때문이랍니다! 예를 들어, 사람의 나이는 정수로 나타내고,  몸무게는 실수로 표현하는 게 자연스럽죠? 📈

### 🎉 Rust의 자료형: 정수 타입 (integers) 🚀

정수는 0부터 시작하는 온전히 정수만 사용하는 데이터 형태예요. 🤖 아주 단순하지만 기본적인 연산에 활용되는데, 크게 두 가지 유형으로 나뉘어요!

* **i32:**  가장 일반적으로 사용되는 정수 타입! -2,147,483,648부터 2,147,483,647까지의 값을 표현할 수 있어요. 🤔 마치 지구에서 가장 큰 자동차 회사의 생산 수량처럼 상당히 큰 범위를 다룰 수 있습니다!
* **i64:** i32보다 더 큰 정수를 저장하기 위해 사용하는 타입! -9,223,372,036,854,775,808부터 9,223,372,036,854,775,807까지의 값을 표현할 수 있어요. 🤯 마치 우주 전체에 존재하는 별 개수처럼 거대한 수를 담을 수 있죠!

**코드 예시:**

```rust
let age: i32 = 25; // 나이는 정수형으로 저장합니다 (i32 타입)
let population: i64 = 10_000_000; // 인구는 더 큰 정수형으로 저장합니다 (i64 타입)
```

💡 **팁:** Rust에서 자료형을 명시할 때 `:` 기호를 사용하는 것을 알아두세요! 이렇게 하면 코드의 가독성이 향상되는데, 컴파일러도 더 정확하게 자료형을 파악할 수 있답니다. 😎


### 🎉 Rust의 자료형: 실수 타입 (floating-point numbers) 🚀

실수는 소수점을 포함하는 데이터 형태예요. 마치 부드럽게 조절되는 기계처럼, 좀 더 정밀한 값을 나타낼 수 있어요! 📈  주로 'f32' 와 'f64' 두 가지 타입으로 사용됩니다.

* **f32:** 실수를 표현하는데 가장 일반적인 형태이며 약 7자리까지 소수점을 유지할 수 있습니다. 🤯 마치 빵 한 조각의 무게를 정확하게 재는 것처럼!
* **f64:** f32보다 더 정밀한 값을 표현하기 위해 사용하는 형태입니다. 약 15~16자리까지 소수점을 유지할 수 있어요. 🚀 마치 우주의 방사선의 변화를 매우 정확하게 측정하는 것처럼!

**코드 예시:**

```rust
let price: f32 = 19.99; // 가격은 실수형으로 저장합니다 (f32 타입)
let temperature: f64 = 25.50_87; // 기온은 더 정밀한 실수형으로 저장합니다 (f64 타입)
```


### 🎉 Rust의 자료형: 문자열 타입 (string) 🚀

문자열은 여러 글자로 이루어진 데이터 형태예요. 📚  "Rust는 재밌다!"와 같이 우리가 사용하는 언어를 표현할 때 사용됩니다!

**코드 예시:**

```rust
let message: &str = "Hello, Rust!"; // 문자열은 &str 타입으로 저장합니다 (Immutable String)
let name = String::from("김철수"); //  String 타입을 이용하여 변경 가능한 문자열을 생성합니다.
```

💡 **팁:** Rust에서는 '&str'를 사용하여 변하지 않는 문자열과 'String'을 사용하여 수정이 가능한 문자열을 표현합니다! 


### 🚨 실무주의보: 자료형 선택의 중요성! 🚀

다양한 자료형을 이해하고 적절하게 선택하는 것이 Rust 코딩의 기본이에요!  잘못된 자료형을 사용하면 코드가 예상치 못하게 작동하거나 오류가 발생할 수 있어요. ⚠️

* 데이터의 크기와 정밀도를 고려하여 적절한 자료형을 선택해야 해요.
* 예를 들어, 사람들의 나이를 저장하는 경우 i32 타입이 적합하지만, 소수점까지 정확하게 표현해야 하는 가격은 f64 타입을 사용하는 것이 좋습니다!
