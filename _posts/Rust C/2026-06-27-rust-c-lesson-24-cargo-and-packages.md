## 🔥 24강: Rust의 라이브러리 관리 시스템 - Cargo와 패키지 사용법! 🚀

안녕하세요! 대한민국 최고의 Rust C 일타 강사, 저 역시 15년 차 실전 개발자 **[이름]**입니다! 😎 오늘은 Rust에서 가장 중요한 요소 중 하나인 **라이브러리 관리 시스템**과 **Cargo**에 대해 알아보겠습니다.  진짜 신기하죠? 🤔

Rust는 초보자가 쉬워서 배우고, 전문 개발자도 빠르게 작업할 수 있는 강력한 언어라는 건 여러분이 이미 알고 있을 거예요! 그 근본적인 이유가 바로 **Cargo** 라고 할 수 있습니다. 🔥

###  ✨ Cargo: Rust의 마법 스틱 ✨

Cargo는 딱히 코드를 작성하거나 실행하는 역할은 하지 않아요. 대신 Rust 프로젝트와 관련된 모든 것들을 관리하고 정리해주는 **"마법 스틱"**이라고 생각하면 편합니다!  💫

예를 들어, 라이브러리를 다운로드 하고 프로젝트에 연결하거나 빌드 과정을 자동화하는 등 여러 기능을 담당합니다. 이렇게 Cargo가 도와줘서 우리는 Rust 개발에 집중할 수 있다는 점이 정말 대단하지 않나요? 👏

###  📦 패키지: Rust의 Lego 블록! 🧱

Rust는 라이브러리들을 **"패키지"**라고 부르고, 각 패키지는 특정 기능을 담당합니다. 🧩 생각보다 많은 패키지들이 존재하며, 서로 연결하여 더 복잡하고 강력한 프로그램을 만들 수 있습니다. 🤩

예를 들어, 네트워크 통신이 필요하면 `tokio` 와 같은 패키지를 사용하고, 웹 프레임워크를 구축하려면 `actix-web` 등의 패키지를 활용할 수 있습니다! 💪 🤯 멋지죠?


###  💪 Cargo 사용하기: 3단계만 알고 있어도 충분해요!

1. **프로젝트 생성**: 처음 Rust 프로젝트를 시작하려면 `cargo new` 명령어를 사용합니다. 예를 들어, "my_project" 라는 이름으로 프로젝트를 만들려면 다음과 같이 입력하면 됩니다:
    ```bash
    cargo new my_project
    ```

     * Cargo가 자동으로  `Cargo.toml` 파일을 생성하며 이 파일은 프로젝트의 정보들을 담고 있습니다!
2. **패키지 추가**: `cargo add` 명령어를 사용하여 필요한 패키지를 추가할 수 있습니다. 예를 들어, `serde_json` 패키지를 추가하려면 다음과 같이 입력하면 됩니다:

    ```bash
    cargo add serde_json
    ```

   * Cargo가 해당 패키지를 다운로드하고 프로젝트에 자동으로 연결합니다! 🚀

3. **빌드 및 실행**: `cargo run` 명령어를 사용하여 프로젝트를 컴파일하고 실행할 수 있습니다. ✨

    * `cargo build` 명령어는 컴파일만 진행하는 데 유용합니다!


###  ⚠️ 실무주의보: 패키지 버전 관리 🧐
Cargo는 패키지의 버전을 자동으로 관리하여 프로젝트 안정성을 보장합니다. **Cargo.lock** 파일이 이 역할을 합니다.

> 🚨  `Cargo.lock` 파일에 적힌 버전 정보는 절대 수정하지 마세요!  🚨
