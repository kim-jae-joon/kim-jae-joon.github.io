##  25강: Git 사용법 배우기: Rust 프로젝트 관리 및 버전 제어 - 코드 개발의 마법 지팡이! ✨

안녕하세요, Rust 신입 개발자 여러분! 👋 이번 강좌에서는 우리가 Rust로 만드는 멋진 프로젝트를 효율적으로 관리하고 버전을 제어하는 **Git**에 대해 알아보겠습니다. Git은 코드 개발의 마법 지팡이라고 불릴 정도로, 개발자들에게 필수적인 도구입니다! 🚀

###  왜 Git가 필요할까요? 🤔

그래서 갑자기 Git는 어떤 매직 포켓몬 같은 존재일까요? 🧙‍♂️ 생각만 해도 머리가 막힐 수 있는 복잡한 코드 관리, 버전의 갈림길, 오류 수정의 고통... 이 모든 것을 해결해주는 바로 Git입니다!

자세히 살펴보면:

* **버전 제어:**  Git은 프로젝트의 모든 변화를 기록하고 이전 버전으로 돌아갈 수 있도록 합니다. 🔄 코드 수정 후 "오, 틀렸다!" 하더라도 겁먹지 마세요! Git는 우리를 구원해줄 거예요. 💪
* **협업:** 여러 개발자가 함께 프로젝트에 작업할 때, Git은 각자의 변경 사항을 조화롭게 결합하고 충돌을 최소화합니다.  🤝 이렇게 팀워크가 더욱 효율적으로! 🚀
* **프로젝트 역사 기록:** Git는 모든 변경 내역과 시점을 기록하며, 프로젝트의 발전 과정을 투명하게 보여줍니다.  📈  

###  Git 기본 명령어 배우기 🕹️

Git은 다양한 명령어를 사용하여 코드 관리를 수행합니다. 처음부터 모든 것을 아는 것은 불가능하죠! 😉 그래서 Git의 기본적인 명령어부터 차근차근 알아보도록 하겠습니다!

* **`git init`**:  새로운 프로젝트에 Git 저장소를 생성합니다. 🪄
   ```bash
   git init
   ```
   예를 들어, "my_rust_project"라는 이름의 새로운 Rust 프로젝트 작업을 시작한다면, 아래 명령어를 사용하여 Git 저장소를 만들 수 있습니다!

* **`git add <파일명>`**:  Git에 파일을 추가합니다. ➕ 지금까지 작성한 코드나 이미지 등을 Git 저장소에 올려야 다음 단계로 진행할 수 있습니다.
   ```bash
   git add main.rs
   ```
   여기서는 "main.rs"라는 Rust 코드 파일을 Git에 추가하는 예입니다! 

* **`git commit -m "메시지"`**:  Git 저장소에 변경 사항을 커밋합니다. 🗃️ 코드를 정리하고, 이전 버전과의 차이점을 기록해두는 중요한 단계입니다.
   ```bash
   git commit -m "첫 번째 코딩 작업 완료!"
   ```
   커밋 메시지는 변경 사항에 대한 간략한 설명으로, 이후 복구 시 무엇을 바꾼 것인지 알려줍니다! 👍 


* **`git status`**:  현재 Git 저장소의 상태를 확인합니다. 👀 어떤 파일이 추가되었는지, 커밋되지 않았는지 등을 볼 수 있습니다.
   ```bash
   git status
   ```

* **`git log`**:  Git 저장소의 변경 사항 이력을 확인합니다. 📑 시간 순서대로 모든 커밋 메시지를 보여줍니다!


* **`git branch`**:  새로운 브랜치를 생성하거나, 현재 브랜치를 확인합니다. 🌿 개발 과정에서 여러 가지 기능을 동시에 작업할 때 유용하게 사용됩니다!
   ```bash
   git branch new_feature
   ```
   여기서는 "new_feature"라는 이름의 새로운 브랜치를 만들어 특정 기능을 개발하는 데 사용할 예정입니다.

* **`git checkout <브랜치명>`**:  특정 브랜치로 변경합니다. 🔄 현재 작업 중인 브랜치에서 다른 브랜치로 이동하는 데 사용됩니다!
   ```bash
   git checkout new_feature
   ```



### Git 사용법을 위한 추가 팁! ✨

* **'git config'**: 개인 설정을 수정하는 명령어입니다. 예를 들어, 이름과 이메일 주소를 설정할 수 있습니다.

    ```bash
    git config --global user.name "이름 입력"
    git config --global user.email "이메일 주소 입력"
    ```


* **온라인 자료 활용**: Git은 강력하지만 복잡한 도구입니다! 공식 문서나 온라인 학습 자료를 활용하면 더욱 쉽게 이해할 수 있습니다. (https://git-scm.com/)
* **프랙티스, 프랙티스, 프랙티스!**:  Git을 마스터하는 가장 좋은 방법은 실제로 사용해 보는 것입니다. 작은 프로젝트부터 시작하여 점차 복잡한 작업으로 진행하면서 Git의 모든 기능을 익혀보세요! 💪
