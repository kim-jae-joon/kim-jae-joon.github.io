###  ## 🎯 디버거: 코딩 블랙박스를 열고 내부 세계 보기

Rust의 디버거는 **break point**와 **stepping** 기능을 이용하여 프로그램 실행 과정을 단계별로 살펴볼 수 있게 해줍니다. 마치 영화 감상 시 빠르게 앞뒤로 움직일 수 있는 버튼처럼, 코드 실행 속도를 조절하며 변수의 상태를 확인하고 불필요한 부분을 제거할 수 있습니다! 🚀

1. **Break Point 설정:** 어떤 코드 줄에서 실행 중지하고 분석을 시작할 지 정하는 기능입니다.  
     - 예시: `gdb` (Gnu Debugger)와 같은 Rust 디버거를 사용하여 `println!()` 함수가 실행되는 부분에 break point를 설정합니다.

```rust
fn main() {
    let x = 5; // Break Point 지정된 위치
    println!("x의 값은: {}", x); 
}
```

2. **Stepping:** 디버거에서 한 줄씩 코드를 실행하는 기능입니다.  단계별로 실행하면 변수가 어떻게 변경되는지 파악할 수 있습니다. 
     - 예시: `next` 명령어를 사용하여 한 줄씩 실행하고, 각 단계마다 변수 값을 확인합니다.

**💡 초보자 폭풍 질문!**:  "디버거는 어떻게 활용하면 좋을까요?" 🤔


### 📚 로그 분석법: 코드의 비밀 대화 해독하기

로그 출력은 Rust 프로그램에서 발생하는 모든 중요한 이벤트를 기록하는 역할을 합니다. 디버거와 달리, 실시간으로 실행되는 프로그램과 동시에 전역적으로 데이터를 기록하여 코드의 진행 상황 파악에 유용합니다. 🤓

1. **로그 레벨 설정:** DEBUG, INFO, WARN, ERROR 등 다양한 레벨로 로그 출력을 조절하는 기능입니다.  예를 들어, 프로그램 실행 초기 단계에서는 `INFO` 레벨 이상만 출력하고, 에러 발생 시에는 `ERROR` 레벨까지 출력할 수 있습니다.

```rust
// log crate 사용 예시
use log::info;

fn main() {
    info!("프로그램 시작!"); // INFO 레벨 로그 출력

    // ... 프로그램 실행 과정 ... 

    let result = some_function(); // 에러 발생 가능

    if let Err(e) = result {
        error!("에러 발생: {}", e); // ERROR 레벨 로그 출력
    } else {
        info!("작업 성공!"); // INFO 레벨 로그 출력
    }
}
```

2. **로그 형식 정의:**  날짜, 시간, 메시지 등 로그 정보를 어떻게 표시할 지 정하는 기능입니다.  템플릿 문자열을 사용하여 간결하고 명확한 로그를 생성할 수 있습니다.


**🚨 실무주의보!**: 로그 분석은 단순히 에러 메시지를 확인하는 것 이상으로, 프로그램의 실행 과정과 성능 측정에 유용합니다. 효율적인 로그 출력은 개발 및 디버깅 과정을 간소화하며, 문제 해결 속도를 높이는 데 도움이 됩니다.
