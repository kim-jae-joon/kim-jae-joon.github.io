## 🔥🚀 Rust 스트럭트의 마법: 생성자와 파괴자 배우기 - 14강 🎉

** 안녕하세요, Rust 탐험대 여러분! 😎  ** 

저는 당신들의 컴퓨팅 여정을 더욱 화려하게 만들어줄, 15년 차 Rust 시니어 개발자이자 최고의 일타 강사인 **'비주얼 코더'**입니다. 오늘은 스트럭트의 심장부에 위치한  **생성자와 파괴자!** 💪 이 두 가지 마법 같은 기능을 통해, 우리 코드는 더욱 깔끔하고 효율적이 될 거예요! 😉

### 🤔 스트럭트: 구조화된 데이터의 왕좌 👑

기억해 보세요. Rust에서 우리가 자료를 정리하고 관리하는 데 있어 가장 중요한 도구는 바로 **스트럭트**죠?  

>  `struct Point {
>      x: i32, 
>      y: i32,
>   } `

이렇게 스트럭트를 만들면 'Point'라는 타입을 가진 새로운 개체가 생성되고, 각각 'x' 와 'y' 라는 이름으로 부모님께서 준 선물처럼 데이터를 담을 수 있죠. ✨

### 💡 생성자: 스트럭트의 태어남 🎉

하지만 이렇게 만들어진 스트럭트 개체는 **아직 아무것도 안 가득 차 있는 새 집**이랑 같지요? 🤔 마치 새로운 생명체가 태어난 것과 같아서, 생성자를 통해 초기값을 설정해야죠!

> `struct Point {
>      x: i32, 
>      y: i32,
>   }
> 
> impl Point {
>     fn new(x: i32, y: i32) -> Self { // 생성자 함수 정의! 😎
>         Point { x, y }  // 스트럭트를 초기화하고 반환
>     }
> }`

 **'new'** 라는 이름을 가진 함수가 생성자가 됩니다. 이 함수 안에서 우리는 'x', 'y' 값을 받아서 새로운 `Point` 개체를 만들고, 그것을 돌려주죠! 🎉  생성자는 스트럭트의 설립과 초기화를 담당하는 중요한 역할을 하는 거예요!

### 🚨 실무주의보: 생성자 없이 살 수도 있겠지만... 🤔

> `let p1 = Point { x: 10, y: 20 };`  
>
> 위 코드처럼 직접 값을 지정하는 방법도 있어요. 하지만 **생성자가 아닌 경우 각 필드에 값을 명시해야** 하고, 코드가 복잡해질 수 있어요! 

### 🚀 파괴자: 스트럭트의 마지막 인사 👋

그리고 새로운 생명체가 태어나고 죽는 것은 자연스러운 순환이죠. 마찬가지로 Rust에서 스트럭트도 사용이 끝났을 때 **파괴자가** 작동하며, 기억해 둬야 할 데이터를 정리하고 해제합니다!

> `impl Drop for Point {
>     fn drop(&mut self) {
>         println!("Point({},{}) is being dropped!", self.x, self.y); // 파괴자 함수 실행 후 출력
>     }
> }`

 **'drop'** 이라는 마법적인 키워드로 파괴자를 정의합니다.  스텍에서 스트럭트가 삭제될 때 자동으로 실행되고, 우리는 `println!()` 와 같은 기능을 사용하여 'Point' 개체가 파괴되는 순간에 알림 메시지를 출력할 수 있습니다!

### 🤔 왜 생성자와 파괴자가 필요할까?

>  * **깔끔한 코드:** 모든 필드 초기화를 한 곳에서 처리하면 코드가 더욱 명확해요. 👍
> * **메모리 관리:** 스트럭트의 사용이 끝나면 메모리를 효율적으로 해제하여 프로그램 성능을 향상시킬 수 있어요! 🤯
