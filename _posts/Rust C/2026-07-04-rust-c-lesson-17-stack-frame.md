###  🧠 함수 호출: 프로그램의 "명령" 전달 👋

Rust에서 `fn` 키워드로 정의된 것은 바로 함수죠. 함수는 특정 작업을 실행하는 코드 블록이라고 생각하면 좋아요. 🕹️ 예를 들어, "더하기"라는 함수는 두 개의 숫자를 받아 합산하는 작업을 수행합니다.

```rust
fn add(x: i32, y: i32) -> i32 {
    x + y
}
```

이 코드를 살펴보면 `add`라는 함수가 정의되어 있습니다. 괄호 안의  `x` 와 `y`는 입력 값을 받는 인자이고, `-> i32`는 결과값이 정수형 (`i32`) 이라는 것을 의미합니다.

*   **인자:** 함수에 전달되는 데이터입니다.
*   **반환값:** 함수가 작업을 완료하고 돌려주는 값입니다. 💫

함수 호출은 명령처럼 생각하세요!  "add 함수, 5 와 3 을 더해줘!" 라고 말하는 것과 같습니다. 😉

```rust
let result = add(5, 3);
println!("더한 결과: {}", result); // 출력: 더한 결과: 8
```

여기서 `add(5, 3)` 부분이 함수 호출입니다.  `add` 함수에 숫자 5와 3를 입력값으로 전달하고, 그 결과 값을 변수 `result`에 저장합니다. 그리고 마지막에는 `println!` 함수를 사용하여 결과값을 출력합니다.

### 🚀 스택 프레임워크: 프로그램 실행의 무대 배경✨

함수 호출이 일어날 때마다 Rust는 **스택 프레임워크**라는 특별한 기구를 사용합니다. 이 스택은 함수 호출 순서와 관련된 정보들을 저장하는 "메모장"과 같다고 이해하면 쉬울 거예요!

*   각 함수 호출 시, 그 함수에 필요한 정보 (변수 값, 인자 값 등)가 스택에 추가됩니다. 📤
*   함수 실행이 완료되면 해당 정보들은 스택에서 제거됩니다.📥
*   `return` 키워드를 사용하여 함수에서 결과값을 돌려받으면 스택의 최상단에 저장되어 부르는 함수로 전달됩니다.

스택 프레임워크는 마치 프로그램 실행 과정을 단계별로 진행하는 "연극 무대"와 같아요! 🎭 각 함수가 스테이지에 등장하고, 연기를 하고(작업 수행) 다시 뒤로 물러나 (반환) 가는 순서를 통해 프로그램이 완벽하게 실행됩니다.

### 코드 예제를 통해 이해하기: 함수 호출과 스택 프레임워크

```rust
fn main() {
    let result = call_nested_function(10);
    println!("Main function received: {}", result);
}

fn call_nested_function(x: i32) -> i32 {
    call_another_function(x * 2)
}

fn call_another_function(y: i32) -> i32 {
    return y + 5; // 함수에서 결과값을 반환하는 부분입니다!
}
```

이 코드 예제를 살펴보세요. `main` 함수가 실행되면 `call_nested_function` 함수를 호출합니다.  `call_nested_function`은 다시 `call_another_function`을 호출하며, 스택 프레임워크는 각 함수 호출 정보를 저장하면서 실행 과정을 돕습니다!

*   **main 함수:** 가장 먼저 실행되고 `call_nested_function` 을 호출합니다.
*   **call_nested_function:**  함수 인자(`x`)를 기반으로 계산하고  `call_another_function` 에 값을 전달하여 스택에 추가됩니다.
    *   인자값 ( `x * 2` )이 함수 `call_another_function` 로 전달됩니다. 📤
*   **call_another_function:** 인자 (`y`) 를 기반으로 계산하고 결과(`y + 5`)를 반환합니다.  

스택 프레임워크는 각 함수가 호출되고 완료될 때마다, 이 정보들을 관리하여 프로그램이 원활하게 실행될 수 있도록 도와줍니다! 🤩
