## 30강: 리소스 관리와 메모리 누수 방지 - 메모리 마스터가 되어보자!

**진짜 신기하죠?** 코딩을 시작하면 가장 먼저 마주치는 난관 중 하나가 바로 리소스 관리와 메모리 누수입니다. 마치 주방에서 재료를 제대로 관리하지 않으면 요리가 망치기 쉽듯이, 프로그램에서 메모리를 제대로 관리하지 않으면 애플리케이션이 갑자기 멈추거나 느려지는 등의 문제가 생길 수 있어요! 오늘은 메모리 관리의 달인이 되어 보는 시간을 가져볼게요. 💪

### 1. 메모리 누수: 우리의 적!

**이거 모르면 큰일 납니다!** 메모리 누수는 마치 집에 들어오는 수도관에 구멍이 생겨 물이 새어나가는 것과 비슷해요. 코드에서 메모리 누수가 발생하면 프로그램이 사용하지 않는 메모리 공간을 계속 점유하게 되어 결국 시스템 리소스가 고갈되고 애플리케이션이 불안정해집니다. 예를 들어, 웹사이트를 새로고침할 때마다 점점 느려지는 현상이 바로 메모리 누수의 증상 중 하나죠!

#### 메모리 누수 예시 코드 (Rust)

```rust
fn main() {
    // 메모리 누수 발생 예시
    let mut vec = vec![1, 2, 3]; // 초기화된 벡터

    while true {
        // 여기서 벡터를 계속 참조하지만 해제하지 않음
        println!("현재 벡터: {:?}", vec);
        // ⚠️ 메모리 누수 발생! 벡터 `vec`이 해제되지 않음
    }
}
```

**코드 분석:**
- `vec![1, 2, 3]`로 초기화된 벡터가 생성되지만, `while true` 루프 내에서 벡터에 대한 참조가 계속 유지됩니다.
- 루프가 무한히 실행되면서 `vec`은 해제되지 않아 메모리 누수가 발생합니다.

**해결 방법:**
- 루프 밖으로 벡터를 이동시키거나, 필요 없는 경우에 `drop`을 사용하여 명시적으로 해제합니다.

```rust
fn main() {
    let mut vec = vec![1, 2, 3];

    while true {
        println!("현재 벡터: {:?}", vec);
        // 루프 밖으로 이동하여 메모리 관리 개선
        // 또는 필요에 따라 명시적으로 해제
        // drop(vec); // 필요 시 사용
    }
}
```

### 2. 리소스 관리의 핵심: 스마트 포인터와 소유권

Rust의 강점 중 하나는 **소유권(Ownership)** 시스템입니다. 이 시스템은 메모리 관리를 훨씬 더 안전하고 직관적으로 만들어줍니다. 간단하게 설명하면, 각 변수는 단일 소유자를 가지고 있으며, 그 소유자가 더 이상 필요하지 않으면 자동으로 메모리를 해제합니다.

#### 스마트 포인터: Rc와 Arc

**스마트 포인터**는 복잡한 메모리 관리를 단순화해줍니다. `Rc`와 `Arc`는 주로 참조 카운팅을 통해 여러 스레드에서 공유되는 데이터를 관리할 때 유용합니다.

##### Rc 예시 코드

```rust
use std::rc::Rc;
use std::thread;

fn main() {
    let shared_data = Rc::new(vec![1, 2, 3]); // 공유 가능한 벡터 생성

    let mut handles = vec![];

    for _ in 0..5 { // 5개의 스레드 생성
        let data = Rc::clone(&shared_data); // 공유 데이터 복사
        let handle = thread::spawn(move || {
            println!("스레드 데이터: {:?}", data); // 각 스레드에서 데이터 접근
        });
        handles.push(handle);
    }

    for handle in handles {
        handle.join().unwrap(); // 스레드 종료 대기
    }
}
```

**코드 분석:**
- `Rc::new(vec![1, 2, 3])`로 공유 가능한 벡터를 생성합니다.
- `Rc::clone()`을 사용하여 각 스레드에서 데이터에 대한 참조를 복사합니다.
- 참조 카운트가 증가하여 데이터가 유지되며, 스레드가 종료될 때 자동으로 메모리가 해제됩니다.

##### Arc 예시 코드

```rust
use std::sync::Arc;
use std::thread;

fn main() {
    let shared_data = Arc::new(vec![1, 2, 3]); // 공유 가능한 벡터 생성 (스레드 안전성)

    let mut handles = vec![];

    for _ in 0..5 { // 5개의 스레드 생성
        let data = Arc::clone(&shared_data); // 공유 데이터 복사 (스레드 안전성)
        let handle = thread::spawn(move || {
            println!("스레드 데이터: {:?}", data); // 각 스레드에서 데이터 접근
        });
        handles.push(handle);
    }

    for handle in handles {
        handle.join().unwrap(); // 스레드 종료 대기
    }
}
```

**코드 분석:**
- `Arc::new()`로 공유 데이터를 생성하여 멀티스레드 환경에서 안전성을 보장합니다.
- `Arc::clone()`을 통해 스레드 간 데이터 공유를 안전하게 관리합니다.

### 💡 초보자 폭풍 질문! 🚀
**질문 1:** 스마트 포인터(`Rc`, `Arc`)를 사용하지 않고도 메모리 누수를 방지할 수 있나요?
- **답변:** 네, 가능합니다! 소유권 규칙을 철저히 따르고, 필요 이상으로 변수를 유지하지 않는 것이 중요합니다. 예를 들어, 반복문 내에서 임시로 생성된 데이터는 루프 밖으로 이동시키거나 즉시 사용 후 해제하는 것이 좋습니다.

**질문 2:** 멀티스레드 환경에서 메모리 관리는 왜 더 복잡해지나요?
- **답변:** 멀티스레드 환경에서는 여러 스레드가 동시에 메모리에 접근할 수 있어 데이터 경쟁 조건이나 데드락이 발생할 위험이 있습니다. `Arc`와 같은 동기화된 스마트 포인터를 사용하면 이러한 문제를 완화할 수 있습니다.

### 3. 실전 팁: 메모리 프로파일링 도구 활용

메모리 누수를 찾아내려면 **프로파일링 도구**를 활용하는 것이 좋습니다. Rust에서는 `Valgrind`와 유사한 `AddressSanitizer`나 `LeakSanitizer`를 사용할 수 있습니다.

#### LeakSanitizer 예시

```rust
fn main() {
    // 잠재적인 메모리 누수를 확인하기 위한 코드
    let data = String::from("큰 데이터");
    // 의도적으로 데이터를 사용하지 않고 종료
    // ⚠️ 메모리 누수 검사 대상
}

// 컴파일 시 옵션 추가
// Cargo.toml에 추가
[profile.release]
opt-level = ["3"]
sanitize = ["address"]
```

**실행 방법:**
1. 프로젝트 디렉토리에서 `cargo build --release -- -Zsanity=leak` 명령어 실행.
2. 결과를 통해 메모리 누수 위치를 확인하고 수정합니다.
